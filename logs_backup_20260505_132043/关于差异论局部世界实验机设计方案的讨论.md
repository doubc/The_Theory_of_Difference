## 一、对原始施工图的可行性评估

### 1.1 确实可行的部分

- **公理→loss 的翻译路径成熟**：A2（离散性）、A4（变易成本）、A5（守恒）、A9（复杂度惩罚）可直接写成 PyTorch loss

### 1.2 关键风险点

| 风险 | 说明 |
|------|------|
| 公理冲突 | A4（变易最小）和 A7（稳定闭合）可能冲突；A5（守恒）可能和 A1（差异保持）冲突 |

### 1.3 规模问题

**结论**：项目的真正价值可能不是"造出能涌现的迷你宇宙"，而是"证明公理约束不会杀死涌现的可能性"——这已经是有意义的负结果。

### 1.4 规模与公理的关系

A4（最小变易）和 A3（局域性）本身就是对搜索空间的压缩。规模大时公理约束反而变得必要——否则搜索空间不可控。规模问题和公理约束可能是同一个问题的两面。

## 二、核心设计深化

### 2.1 A1 差异源（解决"差异从哪来"的问题）

**哲学问题**：初始差异是固定的还是持续涌入的？

**决策**：设定一个内在差异源，数值设为无穷大，不断向外 +1，类似地球能源来自太阳。

**实现**：边界条件的一端为注入端（source），持续向系统注入差异。

```python
def inject_difference(state, source_strength=1.0):
    mask = torch.rand_like(state) < 0.01  # 低概率注入
    state = state + mask.float() * source_strength
    return state.clamp(0, 1)
```

### 2.2 A6 差异流向（与 A1 结合）

A1 是源头，A6 是流向。合在一起构成"差异流"——从注入点出发，沿阻力最小的路径扩散、汇聚、分叉。A6 不需要单独的 loss，方向性由源-汇边界条件隐式保证。

### 2.3 A8 差异汇（对称偏好的重新理解）

A8 不是"强制对称"，而是差异的自然耗散。系统有一个源端（A1）和一个汇端（A8），差异从源流向汇，结构在中间自组织。对称性是结果，不是目标。

**工程实现**：格点一边是注入边界，另一边是吸收边界，中间是自由演化区。

### 2.4 A7 稳定闭合（活结构 vs 死结构）

A7 不能用"状态不变"定义，要用"模式持续"定义。

三种状态判定：

| pattern_persistence | material_turnover | 判定 |
|---|---|---|
| 高（模式持续） | 高（物质在换） | **活结构** → A7 通过 |
| 高（模式持续） | 低（物质不换） | **死结构** → A7 冻结惩罚 |
| 低（模式不持续） | 任意 | **噪声** → A7 模式损失 |

三个子指标：
- `_pattern_persistence`：差异场的时间相关性（余弦相似度）
- `_boundary_integrity`：梯度场的时间稳定性
- `_material_turnover`：状态值的平均变化量
- `_frozenness`：pattern 高 + turnover 低 → 冻结

### 2.5 A5 + A9 升维机制

**核心洞察（用户）**：A9 对应代数里的素数。不是说 A9 不允许其他状态存在，而是到了一个限度就必须再次分层。A9 的筛选提供了升维的可能。

**A9 不是压缩惩罚，而是升维触发器**。检测的不是"能不能压缩"，而是"当前层的表达能力是否已经耗尽"。

**升维机制**：
1. A1 注入差异，沿 A6 流动
2. 中途形成稳定结构（A7），结构成为流的障碍
3. A5 检测到障碍处守恒失衡（流入 ≠ 流出 + 内部变化）
4. A9 检测到失衡无法在当前层消解
5. 两件事同时成立 → 升维触发

**A9 实现思路**：不作为训练 loss，作为层级升级的准入测试。检测当前层是否用了比必要的更多自由度来维持结构。

### 2.6 A1-A8 完整流程与升维

```
A1(源) → A6(流向) → A3(只能局部走) → A4(走最小代价路径)
    → 中途形成稳定结构(A7)
    → 结构成为流的障碍
    → A5 检测到障碍处守恒失衡
    → A9 标记失衡无法在当前层消解
    → 升维
```

## 四、关于失败/死寂/爆炸的处理

**用户观点**：陷入死寂或爆炸不应停止系统，而应像现实中的黑洞或宇宙学现象一样，新产出一系列空间继续运作。

**设计决策**：死寂和爆炸不作为"实验失败"，而作为**结构事件**——系统的一部分发生相变，变成新的边界条件。

| 区域类型 | 物理类比 | 工程处理 |
|----------|----------|----------|
| 死寂区 | 黑洞/吸收体 | 转为强汇，吸收周围差异 |
| 爆炸区 | 奇点/辐射源 | 转为强源，向周围辐射差异 |
| 稳定区 | 正常演化区 | 按公理约束继续 |
| 边界区 | 相边界 | 梯度最大处，重点关注 |

```python
def classify_regions(self, state, history):
    """把状态空间分区，而不是整体判断死活"""
    recent = history[-8:]
    activity = torch.stack([s.mean() for s in recent]).mean()
    volatility = torch.stack([(recent[i+1] - recent[i]).abs().mean() 
                              for i in range(len(recent)-1)]).mean()
    persistence = self._pattern_persistence(recent)
    
    dead_mask = (activity < 0.05) & (volatility < 0.01)
    explode_mask = (activity > 0.95) | (volatility > 0.8)
    stable_mask = (persistence > 0.7) & ~dead_mask & ~explode_mask
    boundary_mask = ~dead_mask & ~explode_mask & ~stable_mask
    
    return {
        "dead": dead_mask,         # → 转为吸收体
        "explode": explode_mask,   # → 转为辐射源
        "stable": stable_mask,     # → 正常演化
        "boundary": boundary_mask, # → 重点关注区
    }
```

## 五、公理体系最终设计

### 公理分类

| 公理 | 类别 | 角色 |
|------|------|------|
| A1 差异源 | 观测 | 持续 +1 的外部注入，由 layer.inject_difference() 执行 |
| A2 离散编码 | 约束 | 限制状态空间为离散合法值 |
| A3 局域性 | 约束 | 优先由模型结构（CNN 小卷积核）保证 |
| A4 最小变易 | 约束 | 变化有代价，偏好较小代价路径 |
| A5 守恒 | 约合 | 追踪守恒残差，与 A9 联动检测升维 |
| A6 流向耦合 | 观测 | 方向性由源-汇边界条件隐式保证 |
| A7 稳定闭合 | 约束 | 区分活结构/死结构/噪声 |
| A8 差异汇 | 观测 | 持续 -1 的外部吸收 |
| A9 升维触发 | 触发 | 不作为 loss，作为层级升级判定条件 |

### 关键设计原则

1. **公理不写死在格点规则里**，而是写成可迁移的层级约束接口
2. **能用结构保证的公理，不要全部压到 loss 里**（如 A3 用 CNN 结构保证）
3. **A1 和 A8 作为边界条件**，而非 loss 项
4. **A9 不参与训练**，只做升维判定
5. **每一步演化都留下公理违背报告**

## 六、接口设计摘要

### LayerBase

所有层级必须实现的统一接口：
- 状态空间：initial_state, project_state, valid_state
- 差异度量：measure_difference, measure_invariant, transition_cost
- 源汇：inject_difference, absorb_difference
- 稳定性：detect_stable_structures, stability_violation
- 粗粒化：coarse_grain, measure_ascent_pressure
- 配置：get_axiom_weight

### AxiomBase

所有公理必须实现的接口：
- violation(state, next_state, layer, history) → AxiomReport

### WorldEngine

核心循环：
- run() 主循环，升维后自动切换新层继续运行
- classify_regions() 分区检测（死寂/爆炸/稳定/边界）
- _is_dead() 失败检测（但根据新设计，死寂区转为结构事件而非失败）
