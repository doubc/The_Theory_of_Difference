## 1. 核心发现：密封是一次性相变，不是渐进过程

exp_153 给出了一个极其干净的结论：

> **48/49 次密封运行中，密封发生在单次 cascade 事件中，一次性冻结所有 bits。**

### 数据支撑

| 指标 | 数值 |
|------|------|
| 密封率 | 49/50 (98%) |
| 平均密封步 | 19.1 +- 15.4 |
| 中位数密封步 | 16 |
| 密封步范围 | [6, 110] |
| 平均 cascade 事件数 | **1.0** |
| Cascade 大小: size=19 | 48/49 (98%) |
| Cascade 大小: size=15 | 1/49 (2%) |
| 快速密封 (<10步) 平均权重 | 27.6 |
| 慢速密封 (>30步) 平均权重 | 21.6 |

### 理论意义

这是一个 **一阶相变** (first-order phase transition)：

1. **积累阶段**：系统在时间中演化，激活模式逐渐变化，权重逐步增加
2. **临界点**：当活跃 bits 数达到阈值 (~35 bits out of 48)，系统到达临界状态
3. **瞬间密封**：在临界点，系统一次性做出所有 bits 的"保持/冻结"决策

这个过程不是：
- 逐个 bit 逐渐冻结
- 小 cascade 逐步扩散
- 反复的冻融循环

而是：
- 系统在临界密度的瞬间发生一次性分化

## 2. "共同反差"的操作性表达

这个结果直接对应差异论中的 **共同反差** 概念：

> 差异不是先后出现的，而是"共同被看见"的。

在模拟机中，bits 不是依次被判定为"应该冻结"或"应该保持"，而是**在同一个瞬间**，所有 bits 同时被一个共同的比较框架评判。这一评判的结果一次性决定了每个 bit 的命运。

**共同反差的操作性定义**：
- 条件：系统激活密度达到临界值 (~35/48 = ~73%)
- 触发：一次性分化决策
- 输出：N 维空间划分为"保持集" (~16 bits) 和"冻结集" (~19 bits)
- 冻结 bits 的差异位置在种子间变化 (无固定角色)

这解释了差异论的一个核心命题：**差异是同时性的，不是历时性的。**

---

## 3. 密封时序的稳健性

尽管密封是一次性的，密封发生的时间点变化较大 (6-110 步)：

- **初始激活数量影响**：初始 4-6 个 ones 时平均 12-15 步密封；初始 8-11 个 ones 时平均 18-25 步
- **权重与步数的反相关**：快速密封 (<10步) 的最终权重更高 (27.6 vs 21.6)
- **单个异常值**：run 23 在 110 步时才密封，且只冻结了 15 bits (阈值从 35 降至 31)
  - 这暗示可能存在"弱密封"的亚稳态
  - 权重=15 是异常低值，说明密封时系统尚未充分自组织

---

## 4. 无固定角色分配

> 没有哪个 bit 是永远冻结或永远保持的。

- 0/48 bits 永远不被冻结
- 0/48 bits 永远被冻结
- 所有 48 bits 在不同种子中交替出现在"保持"和"冻结"集

**分布梯度**：
- 最常冻结：bit 5 (55.1%)、bit 12 (53.1%)、bit 19 (51.0%)
- 最常保持：bit 39 (75.5%)、bit 33 (73.5%)、bit 40 (71.4%)

梯度暗示存在与拓扑/位置相关的偏好，但偏好不是确定性。

## 5. Phase 12 P2：N-Sweep 相变边界扫描 (exp_154)

P1 的发现提出了一个关键问题：**单次 cascade 的一阶相变是 N=48 的特例，还是所有 N 下的普适性质？**

### 5.1 实验设计

exp_154 跨越相变边界附近进行 N 值扫描：

| 参数 | 值 |
|------|------|
| N 值 | 24, 30, 36, 42, 48, 54, 60, 72, 84, 96 |
| 每 N 运行数 | 5-12 (总计 70 runs) |
| 步数 | 5000 |
| 记录 | 每步记录 cascade 事件与 seal 状态 |

### 5.2 关键发现

#### 发现 1：相变边界修正 — N0* ~ 34

| N | 密封率 | 结论 |
|---|--------|------|
| 24 | 0% | 无 L1 形成 |
| 30 | 0% | 无 L1 形成 |
| **36** | **40%** | **临界区！** |
| >=42 | 100% | 确定形成 L1 |

- Phase 9 测得的 N0* ~ 30.5 基于非精确密封指标 (L1 形成率)
- Phase 12 P2 的聚簇指标给出更精确临界点：**N0* ~ 34**
- 36 是第一个出现密封的 N 值，仅 40% runs 密封 — 临界涨落区域

**Revised phase boundary**: N0* = 34 (从 30.5 上调)

#### 发现 2：普适标度律 — cascade_size = 0.40 * N

| N | cascade 平均大小 | 密封比例 | cascade/N |
|---|-----------------|---------|-----------|
| 36 | 17.0 | 0.472 | 0.47 |
| 42 | 16.0 | 0.381 | **0.38** |
| 48 | 19.0 | 0.396 | **0.40** |
| 54 | 21.0 | 0.389 | **0.39** |
| 60 | 24.0 | 0.400 | 0.40 |
| 72 | 27.6 | 0.383 | 0.38 |
| 84 | 34.0 | 0.405 | **0.40** |
| 96 | 39.0 | 0.406 | **0.41** |

> **加权平均 cascade_size/N = 0.40** (范围 0.38-0.47)

这是一个极其干净的普适标度律：不论系统大小如何，密封 moment 总是恰好淘汰约 40% 的激活空间。

这对应差异论的一个深层结构命题：**共同反差的"分割比例"是系统的一个普适常数，不依赖于子空间的维度。**

#### 发现 3：100% 单次 Cascade — 一阶相变是普适性质

在所有产生密封的 N 值 (36-96) 和所有 runs (共 42 次密封运行) 中：

> cascade_events_mean = 1.0 且 single_event_ratio = 1.0

- 没有一次运行出现多次 cascade 事件
- 没有一次运行是渐进冻结
- cascade_size_std = 0.0 在 36/42/48/54/60/84/96 (N=72 除外, std=2.8)

**结论：一阶相变是聚簇时空动力学的普适性质，不是特定 N 的产物。**

#### 发现 4：密封时序与 N 的弱相关

密封步数随 N 增大而增加的总趋势存在，但方差极大：
- N=42: 11.8 +- 4.5 步
- N=72: 39.2 +- 45.5 步 (标准差大于均值！说明亚稳态多样性)
- N=96: 29.4 +- 4.7 步 (大 N 反而更快、更稳定)

密封步数与 N 不是单调关系 — 暗示密封时序更多取决于初始激活配置而非系统大小。

#### 发现 5：工程 Bug — N%3 必整除

实验中发现的 SpatialLongRangeEvolver 问题：
- 构造函数自动将 N 取整到 3 的倍数
- 但 initial_state 未相应调整大小
- 只有 N 能被 3 整除时结果才有效
- 这解释了 exp_154 为何只测试了 3 的倍数的 N 值
- 修复：自动 resize initial_state 或抛出异常

### 5.3 理论意义

| 发现 | 差异论对应 |
|------|-----------|
| 普适标度律 0.40*N | 共同反差的"分割比"是系统固有性质 |
| N0* ~ 34 (上调) | 聚簇阈值比早期估计 (基于 L1 形成率) 更高 |
| 100% 单次 cascade | 一阶相变是"差异压缩"的普适表达方式 |
| 密封比例稳定 0.39-0.41 | 淘汰比例不随维度变化 — 差异的"选择率"是常数 |

**核心结论**：相变不是 N=48 的巧合而是普适性质；cascade_size = 0.40 * N 是差异论世界的第一个普适量纲关系。
