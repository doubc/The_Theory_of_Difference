# 机制设计

## 转移引擎

1. 筛选候选通道（开放 + 类型匹配 + 有容量）
2. 按有效成本排序（最小变易原则）
3. 执行转移（容量足则全转，不足则部分转）
4. 剩余压力积累

## 守恒检查

initial_total = transferred + accumulated + transformed + cost

无记录消失即模型错误。

## 破缺机制

积累压力超阈值触发破缺事件：
- 价格跳变
- 基差异常
- 近月逼仓
- 流动性枯竭
- 保证金追缴
- 强制平仓
- 规则调整
- 交割失败
- 主体退出

破缺后差异压力部分释放（50%）。

## 锁定更新

lock_in += amount * 0.001（上限 1.0）

## 最近稳态判定

条件：总压力低于阈值 + 主要差异有承接 + 连续若干轮无破缺 + 通道选择稳定 + 主体退出放缓
