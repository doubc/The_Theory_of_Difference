# **Real_World 项目书（精简版）**

版本：V0.1 ｜ 日期：2026-05-07
定位：**差异结构推理机** ｜ 第一阶段落脚：商品期货

---

## **一、项目定义**
世界生成的中层语法概括为九个机制：聚簇、层级、守恒、先天完备性、最小变易、破缺、循环、锁定、自指。

Real_World 通过人为输入「差异源、承接体、转移通道、机制规则」，生成一个可观察、可调参、可复盘的玩具世界，用来检验差异如何**转移、守恒、积累、破缺、锁定**，并滑向**最近稳态**。

第一阶段以商品期货单品种为落脚地；社会系统（历史制度、国际政治、组织治理等）仅保留接口。

---

## **二、项目目标**

**总目标**：构建基于差异论的可运行推理系统，输入期货市场差异源，输出差异运动轨迹与稳态判断。

系统输出回答：当前主要差异源 → 经哪些通道转移 → 通道是否拥堵 → 谁在承接 → 谁可能退出 → 最可能滑向哪个稳态 → 参数变动后的轨迹变化。

| 阶段 | 范围 | 关键能力 |
|---|---|---|
| 一 | 单品种玩具模型 | YAML 输入差异/主体/通道，运行规则，输出轨迹与稳态报告 |
| 二 | 多差异、多主体、多通道 | 多地区、多月份合约、产业/投机/套利/交易所联动，参数与反事实测试 |
| 三 | 通用化 + 社会接口 | 差异/通道/承接体/规则通用化，建立 `domain_adapter` |

## **三、理论基础**

**核心判断**：世界由差异的区分、转移、承接、稳定化过程生成；价格不是起点，而是差异在制度中的显影点。期货市场为差异提供公共处理界面。

**第一阶段六个核心机制**：差异、转移、守恒、最小变易、最近稳态、锁定。

- **差异**：时间、空间、质量、库存、信用、预期、资金、流动性、规则、交割。
- **转移**：差异不能凭空消失，只能换位置、形式或承接体（如库存→基差、预期→波动、信用→保证金）。
- **守恒**：差异不能被无代价清零，须追问其是否被转移、压缩、隐藏、延期或回流。
- **最小变易**：优先选阻力小、成本低、可执行的路径（先调价，再调现货；先减仓，再改经营）。
- **最近稳态**：当前约束下最先能稳住的状态，**不是最优**（如基差扩大平衡、近月逼仓、低流动性平衡）。
- **锁定**：通道反复使用形成路径依赖，降低后续成本，可能积累长期脆弱性。

```python
    def __init__(self, name: str):
        self.name = name
        self.time = 0
        self.entities = {}
        self.differences = {}
        self.channels = {}
        self.rules = {}
        self.states = []
        self.traces = []
```

类型：现货运输、仓储、期货合约、基差、跨期价差、保证金、交割、仓单、资金流动、交易所规则。

```yaml

```

### **5.5 Rule**

差异选择、通道成本、容量约束、守恒、积累、破缺、锁定、稳态判定。

### **5.6 State / Trace**

`State`：每步系统快照（总压力、活跃差异/通道、阻塞通道、主导差异、最近稳态、压力等级）。
`Trace`：差异运动事件流（time、event_type、difference_id、from→to、amount、reason）。

## **六、核心机制设计**

**有效成本**：

$$
effective\_cost = base\_cost + congestion \times 10 + rule\_penalty - lock\_in \times 10
$$

**通道选择**：开放 + 类型匹配 + 有容量 → 取有效成本最低。

```python
def choose_channel(difference, channels):
    candidates = [c for c in channels
                  if c.status == "open"
                  and c.from_type == difference.type
                  and c.remaining_capacity > 0]
    return min(candidates, key=lambda c: c.effective_cost()) if candidates else None
```

**差异转移**：容量足则全转，不足则部分转，剩余积累。

```python
def transfer_difference(difference, channel):
    amount = min(difference.pressure, channel.remaining_capacity)
    difference.pressure -= amount
    channel.used_capacity += amount
    return amount
```

**守恒检查**：`initial_total = transferred + accumulated + transformed + cost`，无记录消失即模型错误。

**积累 / 破缺**：积累压力超阈值触发破缺事件（价格跳变、基差异常、近月逼仓、流动性枯竭、保证金追缴、强平、规则调整、交割失败、主体退出）。

```yaml
break_threshold: { inventory: 100, liquidity: 80, expectation: 90, margin: 70 }
```

**锁定更新**：`channel.lock_in += amount * 0.001`（上限 1.0）。

**最近稳态判定**：总压力低于阈值 + 主要差异有承接 + 连续若干轮无破缺 + 通道选择稳定 + 主体退出放缓。

## **十一、MVP 定义**

**Real_World Futures MVP** 必须能：读 YAML → 建世界 → 加载差异/主体/通道 → 跑 10\~30 步（最小变易、转移、积累、锁定、破缺、稳态判断）→ 输出轨迹/状态/Markdown 报告。

**MVP 不做**：真实行情、价格预测、交易建议、下单、数据库、前端、机器学习、多品种组合、宏观模型。
