## exp_001：库存→基差（变形链验证）

**目的**：验证变形链机制 inventory→basis→price→margin→liquidity。

**结果**：
- 差异经 `warehouse_receipt_channel` 转移后，正确生成 basis 类型变形差异
- 变形差异继续经 `basis_channel` 向 price 传递
- 链式传递完整，最大深度 5 未触发截断

**关键数据**：
- 变形效率：inventory→basis 约 30%（通道拥堵度影响）
- 无 recurrent，差异 1-2 步被消化

---

## exp_002：保证金压力（反馈循环验证）

**目的**：验证承接体承压后生成反馈差异。

**结果**：
- 承接体 capacity_ratio > 0.6 时，正确生成 margin 反馈差异
- 反馈差异注入世界后继续参与转移
- 反馈量 = `absorb_amount × 0.3 × leverage`，杠杆放大效应成立

**关键数据**：
- 稳态标签：margin_relief
- 反馈差异占总压力比例：约 15-20%

---

## exp_003：近月逼仓（破缺机制验证）

**目的**：验证高 recurrent 速率下的破缺行为。

**结果**：
- 34 次破缺事件，系统持续 unstable
- 破缺是内生事件：差异积累超过阈值 → 破缺释放压力 → 差异重新积累
- 与 Phase 1 结论一致：破缺不是外生冲击

**关键数据**：
- 破缺间隔：约 1 步（压力积累极快）
- 峰值压力：反复超过阈值

---

## exp_004：规则调整（干预验证）

**目的**：验证交易所干预机制。

**结果**：
- 交易所正确识别压力并执行干预
- 干预后稳态标签变化（liquidity_recovery）
- 干预有效但不消除差异本身——差异被重组，不是被清除

## exp_005：完整拓扑（参数调优）

**目的**：验证变形链+反馈循环+干预多动作的完整拓扑。

### 参数调优历程

**v1（初版）**：recurrent_rate 过高（0.35/0.4），系统永远无法收敛
- 问题：inventory_shortage 每步生成 42 压力，channel 容量只有 35
- 反馈放大：speculative_long_001 leverage=4.0，一步生成 36 的 margin 差异

**v2（当前版本）**：
- recurrent_rate 降低：0.35→0.15, 0.4→0.20, 0.2→0.15
- leverage 降低：4.0→2.0, 3.0→1.5, 2.5→2.0
- break_threshold 提高：inventory 70→90, liquidity 60→80

### 当前结果分析

**50 步运行关键数据**：
- 破缺事件：6 次 accumulation_overflow
- 变形次数：17 次（inventory→basis→price→margin→liquidity 完整链路）
- 反馈差异：10 个（全部已解决）
- 干预次数：9 次（3个时间步触发）
- 最终状态：unstable（持续振荡）

**压力流分析**：
```
输入压力/步：
- inventory_shortage: 18.0 (120×0.15)
- delivery_pressure: 20.0 (100×0.20)  
- expectation_bullish: 10.5 (70×0.15)
总输入：48.5/步

输出能力/步：
- warehouse_receipt_channel: 35
- delivery_channel: 45  
- 其他通道: 有限容量
理论输出：~80/步

净积累：~18.5/步（实际观测值）
```

**破缺模式**：
- 破缺间隔：6-13 步不等
- 破缺严重度：均为 1.0（阈值突破）
- 主导差异：inventory_shortage（持续活跃）

### 收敛问题诊断

**根本原因**：
1. **通道容量瓶颈**：warehouse_receipt_channel 容量35 < inventory_shortage生成18×3步=54
2. **变形链阻塞**：basis→price通道利用率0%，压力无法有效传导
3. **反馈循环延迟**：margin反馈虽生成但量级小，未能形成有效负反馈

**系统行为**：
- 压力积累 → 破缺释放 → 短暂缓解 → 重新积累
- 振荡幅度：60-100压力单位
- 活跃差异：持续9-11个

### 核心发现

1. **周期性破缺是结构内生的**：不是参数问题，是"结构性短缺+recurrent"的必然结果
2. **破缺间隔递增**：recurrent 衰减 → 生成速率下降 → 积累更慢 → 间隔更长
3. **反馈放大器需要约束**：高杠杆主体的 margin 反馈会指数放大压力
4. **收敛需要时间**：生成-排放交叉后，仍需 ~50 步消化存量压力
5. **稳态≠静止**：系统趋向的是"低频破缺+缓慢衰减"的动态平衡

### 参数调优建议（v3）

**方案A：进一步降低recurrent_rate**
```yaml
recurrent_rate调整：
- inventory_shortage: 0.15 → 0.08  # 120×0.08=9.6/步
- delivery_pressure: 0.20 → 0.12   # 100×0.12=12/步  
- expectation_bullish: 0.15 → 0.08 # 70×0.08=5.6/步
预期输入：27.2/步（降低44%）
```

**方案B：扩大关键通道容量**
```yaml
通道容量调整：
- warehouse_receipt_channel: 35 → 60  # +71%
- basis_channel: 80 → 120              # +50%
- margin_clearing_channel: 60 → 100     # +67%
预期输出：~120/步（提升50%）
```

**方案C：增加变形链连通性**
```yaml
新增通道：
- basis_to_price_direct: capacity=80, cost=15
- price_to_margin_direct: capacity=90, cost=10
目的：减少变形链中间环节，提高传导效率
```

**预期效果**：
- 破缺频率：6次/50步 → 2-3次/50步
- 最终稳态：unstable → margin_recovery 或 liquidity_recovery
- 收敛时间：50步内趋向稳定

## Phase 2 整体评估

### 已验证的机制

| 机制 | 验证状态 | 说明 |
|---|---|---|
| 变形链 | ✅ | inventory→basis→price→margin→liquidity 完整传递 |
| 反馈循环 | ✅ | 保证金/流动性/强平三层反馈正确生成 |
| 干预多动作 | ✅ | 降低生成率/扩大容量/释放承接力 + 副作用差异 |
| 守恒检查 | ⚠️ | 基本通过，通道损耗近似计算导致小偏差 |
| 破缺机制 | ✅ | 内生破缺，周期性释放压力 |

### 已知限制

1. **守恒检查偏差**：transform 效率用通道拥堵度近似，实际损耗更复杂
2. **收敛速度**：高 magnitude + 高 recurrent 的差异需要很长周期才能收敛
3. **反馈无上限**：margin 反馈可能在变形链的每一层递归生成，需要深度限制以外的约束

### 与 Phase 1 的关系

Phase 1 验证了 6 条核心机理（recurrent 必要性、通道瓶颈、内生破缺、干预有效性、稳态非唯一、路径依赖）。Phase 2 在此基础上证明：

- **差异不是被消耗，而是被转化**：变形链把一种差异变成另一种
- **承接体是差异的双重角色**：既承接差异，又生成新差异（反馈）
- **干预是结构重组**：交易所不消除差异，而是改变差异的运动路径
