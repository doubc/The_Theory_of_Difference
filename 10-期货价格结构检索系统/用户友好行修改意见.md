### **当前日报只展示了"命中了哪些结构"

，但没回答用户真正要看的三件事：该关心谁、方向与潜力多大、可以对照哪段历史或其他品种——需要从"面向算法输出"改成"面向研究决策"
，重写输出层并反向推动中下游字段补齐。**

你指出的体验问题不是前端美化的事，而是**整条信息流从一开始就没有围绕"用户决策动作"来组织**。现在的日报是一张"相似度匹配表"
，字段是"品种、相似度、模板日期、中枢、方向、幅度"
；但用户真正打开这个日报时，脑子里问的是：今天先看谁？它大概会往哪走？能走多远？我对不对照哪段历史来判断？哪一段跟它最像？这四个问题每一个都没有被日报直接回答。

所以这轮不是改样式，而是要做一次**输出层重构 + 向上反推字段补齐**。下面我按"用户视角 → 信息架构 → 工程落地"
三层给你一份完整方案，然后给出分层的代码改造路径，最后给你可以直接替换的关键文件。

### **第一层：重新定义"一条机会"的信息结构**

当前一条机会的数据结构，等价于一条"相似度记录"。它不足以支撑决策。我建议把"一条机会"升格为一个具备完整研究语义的对象，至少包含下面六块信息。

第一块是**身份与重要性**。品种代码、中文名称、当前价格、当前结构的 `confirmed_idx` 时间、以及一个"关注度评分"
——这个评分不是相似度本身，而是相似度、方向置信度、潜在幅度、模板样本数、活跃度的加权。它决定了日报里从上到下的排序，也就是"
我先看谁"。

第二块是**方向与潜力**。主方向（看涨/看跌/不明）、方向置信度（命中模板里同方向占比）、典型幅度区间（例如历史上同类模板 up_move
的中位数和分位数，而不是单一数字）、触发价位（当前中枢上下某个百分比的关键价）、预期兑现窗口（模板平均多少天内完成该幅度）。

第三块是**对照参考**。最相似的 3\~5 个历史模板，每条显示：品种、历史日期、结构图缩略图、当时后续 30 天走势图、与当前候选的分项差异。注意一定是
**多条**，不是一条，因为用户对照思考时必然要看"如果这是 2006 年铜的那种，或者是 2015 年焦煤的那种，风险/赔率是不同的"。

第四块是**相似性解构**
。不能只给总分，要拆成几何（价格形态）、关系（速度比/时间比）、家族（节奏分类）三项分数，以及每个关键不变量的差异明细，让用户判断"
像在哪、不像在哪"。

第五块是**证据链**。当前结构使用的编译器配置哈希、规则集版本、模板库构建日期、数据截止日、confirmation_lag、outcome
窗口。这是后续复盘和审计的必须。

第六块是**可操作的下一步建议**
。不是策略信号，而是研究建议：建议先调出哪张图、对比哪几个模板、关注什么触发条件。这一块文字化地告诉用户"下一步去做什么"。

把这六块抽象成一个 dataclass，整个系统围绕它组织：

```python
@dataclass
class Opportunity:
    # 身份
    symbol: str
    symbol_name: str
    current_price: float
    confirmed_at: str
    attention_score: float  # 综合排序分

    # 方向与潜力
    direction: str  # up / down / unclear
    direction_confidence: float  # 0~1
    potential_median: float  # 同类模板历史幅度中位数
    potential_p25: float
    potential_p75: float
    trigger_price: float  # 关键触发价
    expected_window_days: int  # 中位数兑现天数

    # 对照参考
    top_matches: list[dict]  # 3~5 个历史模板,每条含 diff/outcome

    # 相似性解构
    sim_total: float
    sim_geometry: float
    sim_relation: float
    sim_family: float
    diff_detail: dict[str, float]

    # 证据链
    evidence: dict  # config_hash, rule_set, template_date 等

    # 建议
    next_actions: list[str]  # 自动生成的研究建议
```

这个对象不是展示层的 ViewModel，而是业务层的统一接口。日报、CLI 输出、后端 API、未来的研究面板都消费它。现在的问题是整条管线根本就没有产出这样一个对象。

### **第二层：重新组织日报的信息架构**

有了上面的对象，日报的结构应该彻底从"一张表"改成"面向决策的三段式"。

**顶部是扫描简报**。显示扫描时间、扫描了多少品种、识别了多少结构、通过活跃度过滤多少、最终产出多少机会、按方向分布（看涨 X 个、看跌
Y 个、不明 Z 个）、按潜力分布（>15% 幅度 N 个、10\~15% M 个）。用户打开第一眼要知道"今天市场整体是什么氛围"。

**中间是今日重点关注**。只显示 attention_score 排名前 5 的机会，用卡片而不是表格。每张卡片左边是当前结构的缩略图（SVG，显示最近
6\~12 个月的价格 + Zone + Cycle），右边是关键信息：品种名、方向、潜力区间（用 p25\~p75
区间条而不是单个数字）、触发价、预期兑现窗口。下方是"最像的三段历史"的横向缩略图条，每张小图标注品种和年份，鼠标悬停或点击展开。

**底部是完整机会清单**。表格形式，但字段要改：品种、关注度评分、方向（带彩色图标）、潜力中位数 +
分位区间、最相似模板（品种+年份，不是光秃秃一个日期）、分项相似度（三个小条形）、研究建议摘要。允许按关注度、按方向、按品种筛选。

关键点不是页面排版，而是**用户眼动流**：打开 → 看顶部简报了解市场基调 → 看中部 5 张卡片确定今天要重点研究哪 1\~2 个 →
对着卡片里的"最像三段历史"去对照思考 → 需要更深入时，下翻看完整清单。现在的日报没有这条眼动流。

用户要看的东西，每一项都必须有对应的数据来源。你现在日报展示不出来，是因为中下游根本没这些字段。下面是反推的缺口清单。

#### **`src/retrieval/opportunity.py`（新建文件）**

```python
"""
机会聚合层 —— 把 top_k 模板匹配聚合成面向研究决策的 Opportunity 对象。
"""
from __future__ import annotations
from dataclasses import dataclass, field, asdict
from statistics import median
from typing import Sequence

FEATURE_SCALES = {
    "avg_speed_ratio": 2.0,
    "avg_time_ratio": 2.0,
    "zone_rel_bw": 0.03,
    "high_dispersion": 0.02,
    "low_dispersion": 0.02,
    "zone_strength": 5.0,
}
FEATURES = list(FEATURE_SCALES.keys())


@dataclass
class TemplateMatch:
    symbol: str
    symbol_name: str
    end_date: str
    outcome_start: str
    direction: str  # up / down / unclear
    up_move: float
    down_move: float
    days_to_peak: int
    days_to_trough: int
    bundle_id: str | None
    diff_detail: dict[str, float]
    distance: float
    similarity: float  # 1/(1+distance)


@dataclass
class Opportunity:
    symbol: str
    symbol_name: str
    current_price: float
    confirmed_at: str
    attention_score: float

    direction: str
    direction_confidence: float
    potential_median: float
    potential_p25: float
    potential_p75: float
    trigger_price: float
    expected_window_days: int

    top_matches: list[TemplateMatch]

    sim_total: float
    sim_geometry: float
    sim_relation: float
    sim_family: float
    diff_detail: dict[str, float]

    evidence: dict
    next_actions: list[str] = field(default_factory=list)

    def to_dict(self) -> dict:
        d = asdict(self)
        d["top_matches"] = [asdict(m) for m in self.top_matches]
        return d


# ── 聚合逻辑 ───────────────────────────────────────────────────────────
def _percentile(values: list[float], p: float) -> float:
    if not values:
        return 0.0
    s = sorted(values)
    k = (len(s) - 1) * p
    lo, hi = int(k), min(int(k) + 1, len(s) - 1)
    return s[lo] + (s[hi] - s[lo]) * (k - lo)


def _decompose_similarity(diff_detail: dict[str, float]) -> tuple[float, float, float]:
    """把分项差异拆成几何 / 关系 / 家族三类"""
    geo_keys = ["zone_rel_bw", "high_dispersion", "low_dispersion"]
    rel_keys = ["avg_speed_ratio", "avg_time_ratio"]
    fam_keys = ["zone_strength"]

    def _score(keys):
        vals = [diff_detail.get(k, 0) for k in keys if k in diff_detail]
        if not vals:
            return 0.0
        avg = sum(vals) / len(vals)
        return 1 / (1 + avg)

    return _score(geo_keys), _score(rel_keys), _score(fam_keys)


def compute_attention_score(
        similarity: float,
        direction_confidence: float,
        potential_median: float,
        match_count: int,
) -> float:
    """
    关注度评分: 综合相似度、方向一致性、潜力、样本充足度。
    输出 0~100 分。
    """
    sim_w = min(similarity, 1.0) * 40  # 相似度权重 40
    dir_w = direction_confidence * 25  # 方向置信度权重 25
    pot_w = min(potential_median / 0.20, 1.0) * 25  # 潜力权重 25 (20% 封顶)
    cnt_w = min(match_count / 5, 1.0) * 10  # 样本充足度权重 10
    return round(sim_w + dir_w + pot_w + cnt_w, 2)


def generate_next_actions(
        symbol: str,
        direction: str,
        trigger_price: float,
        top_matches: list[TemplateMatch],
) -> list[str]:
    """自动生成研究建议（不是交易信号，是研究动作）"""
    actions = []
    if top_matches:
        best = top_matches[0]
        actions.append(
            f"先对照 {best.symbol_name}({best.symbol}) {best.end_date[:7]} 的结构图，"
            f"确认高低点分布是否一致"
        )
        if len(top_matches) >= 3:
            other = top_matches[1:3]
            names = "、".join(f"{m.symbol_name} {m.end_date[:7]}" for m in other)
            actions.append(f"再对比 {names} 的后续走势差异，判断当前形态更接近哪种")

    if direction == "up":
        actions.append(f"重点观察价格突破 {trigger_price:.1f} 之后的放量情况")
    elif direction == "down":
        actions.append(f"重点观察价格跌破 {trigger_price:.1f} 之后的反抽力度")
    else:
        actions.append("方向尚不明朗，建议等待一根明确方向的 bar 再介入研究")

    actions.append(f"在数据库中调出 {symbol} 近 6 个月日线，人工核对 Zone 标注位置")
    return actions


def aggregate_opportunity(
        symbol: str,
        symbol_name: str,
        current_price: float,
        confirmed_at: str,
        current_inv: dict,
        top_matches: list[TemplateMatch],
        evidence: dict,
) -> Opportunity | None:
    """核心聚合函数"""
    if not top_matches:
        return None

    # 方向置信度
    up_count = sum(1 for m in top_matches if m.direction == "up")
    down_count = sum(1 for m in top_matches if m.direction == "down")
    total = len(top_matches)
    if up_count > down_count:
        direction = "up"
        direction_conf = up_count / total
    elif down_count > up_count:
        direction = "down"
        direction_conf = down_count / total
    else:
        direction = "unclear"
        direction_conf = 0.5

    # 潜力分布（取主方向那一侧的 move）
    if direction == "up":
        moves = [m.up_move for m in top_matches if m.direction == "up"]
        days = [m.days_to_peak for m in top_matches if m.direction == "up" and m.days_to_peak > 0]
    elif direction == "down":
        moves = [m.down_move for m in top_matches if m.direction == "down"]
        days = [m.days_to_trough for m in top_matches if m.direction == "down" and m.days_to_trough > 0]
    else:
        moves = [max(m.up_move, m.down_move) for m in top_matches]
        days = []

    if not moves:
        moves = [0.0]

    pot_median = median(moves)
    pot_p25 = _percentile(moves, 0.25)
    pot_p75 = _percentile(moves, 0.75)
    expected_days = int(median(days)) if days else 20

    # 触发价
    if direction == "up":
        trigger_price = current_price * 1.01
    elif direction == "down":
        trigger_price = current_price * 0.99
    else:
        trigger_price = current_price

    # 相似度
    sim_total = top_matches[0].similarity
    diff_detail = top_matches[0].diff_detail
    sim_geo, sim_rel, sim_fam = _decompose_similarity(diff_detail)

    attention = compute_attention_score(sim_total, direction_conf, pot_median, total)

    actions = generate_next_actions(symbol, direction, trigger_price, top_matches)

    return Opportunity(
        symbol=symbol,
        symbol_name=symbol_name,
        current_price=current_price,
        confirmed_at=confirmed_at,
        attention_score=attention,
        direction=direction,
        direction_confidence=round(direction_conf, 3),
        potential_median=round(pot_median, 4),
        potential_p25=round(pot_p25, 4),
        potential_p75=round(pot_p75, 4),
        trigger_price=round(trigger_price, 2),
        expected_window_days=expected_days,
        top_matches=top_matches[:5],
        sim_total=round(sim_total, 3),
        sim_geometry=round(sim_geo, 3),
        sim_relation=round(sim_rel, 3),
        sim_family=round(sim_fam, 3),
        diff_detail=diff_detail,
        evidence=evidence,
        next_actions=actions,
    )
```

```python
"""
高潜力模板构建 v2 —— 扩充前瞻字段，支撑聚合层计算。

每条模板现在记录：
- up_move / down_move
- days_to_peak / days_to_trough
- max_drawdown
- bundle_id / direction
- invariants（沿用旧版）
- outcome_start_date / outcome_end_date
以及品种中文名，方便日报直接使用。
"""
import sys
import os
import json
import pandas as pd
from datetime import timedelta

sys.path.insert(0, os.path.join(os.path.dirname(__file__), ".."))

from src.data.loader import MySQLLoader
from src.data.symbol_meta import symbol_name

CONFIRMATION_LAG = 3
LOOKFORWARD_DAYS = 30
MIN_MOVE_THRESHOLD = 0.10


def analyze_post_performance(loader, symbol, end_date_str,
                             confirmation_lag=CONFIRMATION_LAG,
                             lookforward_days=LOOKFORWARD_DAYS):
    end_date = pd.to_datetime(end_date_str)
    outcome_start = end_date + timedelta(days=confirmation_lag)
    outcome_end = outcome_start + timedelta(days=lookforward_days)

    bars = loader.get(
        symbol=symbol,
        start=outcome_start.strftime('%Y-%m-%d'),
        end=outcome_end.strftime('%Y-%m-%d'),
        freq='1d',
    )
    if not bars:
        return None

    start_price = bars[0].open
    if start_price <= 0:
        return None

    # 逐 bar 追踪峰/谷位置
    peak_price, peak_idx = start_price, 0
    trough_price, trough_idx = start_price, 0
    for i, b in enumerate(bars):
        if b.high > peak_price:
            peak_price, peak_idx = b.high, i
        if b.low < trough_price:
            trough_price, trough_idx = b.low, i

    up_move = (peak_price - start_price) / start_price
    down_move = (start_price - trough_price) / start_price

    # 最大回撤（从峰到谷，若峰在谷之前）
    max_dd = 0.0
    running_max = start_price
    for b in bars:
        if b.high > running_max:
            running_max = b.high
        dd = (running_max - b.low) / running_max if running_max > 0 else 0
        if dd > max_dd:
            max_dd = dd

    direction = "up" if up_move >= down_move else "down"

    return {
        "up_move": round(up_move, 4),
        "down_move": round(down_move, 4),
        "max_move": round(max(up_move, down_move), 4),
        "max_drawdown": round(max_dd, 4),
        "days_to_peak": peak_idx,
        "days_to_trough": trough_idx,
        "outcome_start_date": outcome_start.strftime('%Y-%m-%d'),
        "outcome_end_date": outcome_end.strftime('%Y-%m-%d'),
        "bars_count": len(bars),
        "direction": direction,
    }


def identify_representative_structures():
    pool_path = os.path.join(
        os.path.dirname(__file__), "..", "data", "library", "full_structure_pool.jsonl"
    )
    loader = MySQLLoader(host='localhost', user='root', password='root', db='sina')
    templates = []
    total = errors = 0

    with open(pool_path, "r", encoding="utf-8") as f:
        for line in f:
            total += 1
            rec = json.loads(line)
            try:
                oc = analyze_post_performance(
                    loader, rec["symbol"], rec["end_date"]
                )
                if oc is None or oc["max_move"] < MIN_MOVE_THRESHOLD:
                    continue
                rec["outcome"] = oc
                rec["primary_direction"] = oc["direction"]
                rec["symbol_name"] = symbol_name(rec["symbol"])
                templates.append(rec)
            except Exception:
                errors += 1

    print(
        f"从 {total} 条结构中筛出 {len(templates)} 个高潜力模板"
        f"（错误 {errors}，lag={CONFIRMATION_LAG}d，阈值={MIN_MOVE_THRESHOLD:.0%}）"
    )

    out_path = os.path.join(
        os.path.dirname(__file__), "..", "data", "library", "high_potential_templates.jsonl"
    )
    os.makedirs(os.path.dirname(out_path), exist_ok=True)
    with open(out_path, "w", encoding="utf-8") as f:
        for t in templates:
            f.write(json.dumps(t, ensure_ascii=False) + "\n")
    print(f"已写入 {out_path}")


if __name__ == "__main__":
    identify_representative_structures()
```

```python
流程:

3. 对活跃模板计算归一化距离,取 top_k
4. 委托 opportunity.aggregate_opportunity() 生成 Opportunity
5. 委托 daily_report.render_daily_report() 渲染 HTML
6. 同步落一份 JSON 快照供复盘
"""
import sys
import os
import json
import math
import hashlib
from datetime import datetime

sys.path.insert(0, os.path.join(os.path.dirname(__file__), ".."))

from sqlalchemy import create_engine, inspect
from src.data.loader import MySQLLoader
from src.data.symbol_meta import symbol_name
from src.compiler.pipeline import compile_full, CompilerConfig
from src.retrieval.opportunity import (
    TemplateMatch, aggregate_opportunity, FEATURE_SCALES, FEATURES,
)
from src.workbench.daily_report import render_daily_report

MAX_DIST_THRESHOLD = 1.5
TOP_K = 5


def _distance(inv1: dict, inv2: dict) -> tuple[float, dict]:
    diff = {}
    sq = 0.0
    for f in FEATURES:
        scale = FEATURE_SCALES[f]
        v1 = (inv1.get(f) or 0) / scale
        v2 = (inv2.get(f) or 0) / scale
        d = abs(v1 - v2)
        diff[f] = round(d, 3)
        sq += d ** 2
    return math.sqrt(sq), diff


def _is_active(bars, min_atr_pct: float = 0.005) -> bool:
    if len(bars) < 20:
        return False
    recent = bars[-20:]
    atrs = []
    for i in range(1, len(recent)):
        b = recent[i]
        prev_close = recent[i - 1].close
        tr = max(b.high - b.low, abs(b.high - prev_close), abs(b.low - prev_close))
        atrs.append(tr / prev_close if prev_close > 0 else 0)
    return (sum(atrs) / len(atrs) if atrs else 0) >= min_atr_pct


def _config_hash(cfg: CompilerConfig) -> str:
    s = json.dumps(cfg.__dict__, sort_keys=True, default=str)
    return hashlib.md5(s.encode()).hexdigest()


def _load_templates() -> list[dict]:
    path = os.path.join(
        os.path.dirname(__file__), "..", "data", "library", "high_potential_templates.jsonl"
    )
    out = []
    with open(path, "r", encoding="utf-8") as f:
        for line in f:
            out.append(json.loads(line))
    return out


def _get_symbols() -> list[str]:
    try:
        engine = create_engine("mysql+pymysql://root:root@localhost/sina?charset=utf8")
        tables = inspect(engine).get_table_names()
        return [t for t in tables if not t.endswith("m5") and not t.startswith("test")]
    except Exception:
        return []


def _build_template_match(tmpl: dict, distance: float, diff: dict) -> TemplateMatch:
    oc = tmpl.get("outcome", {})
    return TemplateMatch(
        symbol=tmpl["symbol"],
        symbol_name=tmpl.get("symbol_name") or symbol_name(tmpl["symbol"]),
        end_date=tmpl["end_date"],
        outcome_start=oc.get("outcome_start_date", "N/A"),
        direction=tmpl.get("primary_direction", oc.get("direction", "unclear")),
        up_move=oc.get("up_move", 0.0),
        down_move=oc.get("down_move", 0.0),
        days_to_peak=oc.get("days_to_peak", 0),
        days_to_trough=oc.get("days_to_trough", 0),
        bundle_id=tmpl.get("bundle_id"),
        diff_detail=diff,
        distance=round(distance, 4),
        similarity=round(1 / (1 + distance), 4),
    )


def daily_scan(scan_window_years: int = 3):
    print(f"=== 每日结构机会扫描  [{datetime.now():%Y-%m-%d %H:%M}] ===")

    # 1. 加载活跃参照系
    all_templates = _load_templates()
    cur_year = datetime.now().year
    active_templates = [
        t for t in all_templates
        if cur_year - int(t["end_date"][:4]) <= scan_window_years
    ]
    print(f"模板库 {len(all_templates)} 条，活跃参照系 {len(active_templates)} 条。")
    if not active_templates:
        print("活跃模板为空，请先运行 identify_high_potential_structures.py")
        return

    # 2. 扫描全市场
    loader = MySQLLoader(host="localhost", user="root", password="root", db="sina")
    config = CompilerConfig(min_amplitude=0.02, min_duration=3, zone_bandwidth=0.01)
    cfg_hash = _config_hash(config)
    symbols = _get_symbols()

    structures_found = 0
    opportunities = []
    data_cutoff = datetime.now().strftime("%Y-%m-%d")

    for sym in symbols:
        try:
            bars = loader.get(symbol=sym, freq="1d")
            if len(bars) < 50:
                continue
            if not _is_active(bars):
                continue

            result = compile_full(bars, config)
            if not result.structures:
                continue
            structures_found += 1

            current_st = result.structures[-1]
            current_inv = current_st.invariants or {}
            current_price = bars[-1].close
            confirmed_at = getattr(current_st, "t_end", None) or str(bars[-1].timestamp)

            # 3. top_k 匹配（不再只留 best_match）
            scored = []
            for tmpl in active_templates:
                dist, diff = _distance(current_inv, tmpl.get("invariants", {}))
                if dist < MAX_DIST_THRESHOLD:
                    scored.append((dist, diff, tmpl))
            if not scored:
                continue
            scored.sort(key=lambda x: x[0])
            top = scored[:TOP_K]
            top_matches = [_build_template_match(t, d, diff) for d, diff, t in top]

            # 4. 委托聚合层
            evidence = {
                "config_hash": cfg_hash,
                "data_cutoff": data_cutoff,
                "template_pool_size": len(active_templates),
                "scan_window_years": scan_window_years,
                "top_k": TOP_K,
            }
            opp = aggregate_opportunity(
                symbol=sym,
                symbol_name=symbol_name(sym),
                current_price=round(current_price, 2),
                confirmed_at=str(confirmed_at),
                current_inv=current_inv,
                top_matches=top_matches,
                evidence=evidence,
            )
            if opp is not None:
                opportunities.append(opp)

        except Exception:
            continue

    # 5. 排序 + 渲染 + 落快照
    opportunities.sort(key=lambda o: o.attention_score, reverse=True)
    scan_meta = {
        "total_symbols": len(symbols),
        "structures_found": structures_found,
        "template_count": len(active_templates),
        "data_cutoff": data_cutoff,
        "config_hash": cfg_hash,
    }

    print(
        f"扫描 {len(symbols)} 品种 / 识别 {structures_found} 结构 / "
        f"产出 {len(opportunities)} 机会"
    )

    html_path = render_daily_report(opportunities, scan_meta)
    print(f"HTML 日报: {html_path}")

    snap_path = os.path.join(
        os.path.dirname(__file__), "..", "output",
        f"daily_scan_{datetime.now():%Y%m%d_%H%M}.json",
    )
    with open(snap_path, "w", encoding="utf-8") as f:
        json.dump(
            {"scan_meta": scan_meta,
             "opportunities": [o.to_dict() for o in opportunities]},
            f, ensure_ascii=False, indent=2,
        )
    print(f"JSON 快照: {snap_path}")


if __name__ == "__main__":
    daily_scan()
```

#### **`src/workbench/mini_chart.py`（新建，缩略图渲染，可后续增强）**

这个文件是预留给"重点关注卡片"和"最像三段历史"里的缩略图。当前日报里我用了渐变色块作为占位，真正上线时把下面函数嵌进去替换即可。

```python
"""
紧凑缩略图 SVG 渲染 —— 200×80 级别，嵌入 HTML 日报。
"""
from __future__ import annotations


def mini_price_svg(bars, zones=None, width: int = 200, height: int = 80) -> str:
    if not bars:
        return ""
    zones = zones or []
    p_min = min(b.low for b in bars)
    p_max = max(b.high for b in bars)
    p_range = p_max - p_min or 1

    def tx(i: int) -> float:
        return (i / max(len(bars) - 1, 1)) * width

    def ty(p: float) -> float:
        return height - ((p - p_min) / p_range) * height

    # 价格折线
    pts = " ".join(f"{tx(i):.1f},{ty(b.close):.1f}" for i, b in enumerate(bars))
    parts = [
        f'<svg xmlns="http://www.w3.org/2000/svg" width="{width}" height="{height}" '
        f'viewBox="0 0 {width} {height}" style="background:#12132a">'
    ]
    # Zone 横带
    for z in zones:
        y1, y2 = ty(z.upper), ty(z.lower)
        color = "#ff980044" if z.source.value == "high_cluster" else "#2196f344"
        parts.append(
            f'<rect x="0" y="{y1:.1f}" width="{width}" '
            f'height="{max(y2 - y1, 1):.1f}" fill="{color}"/>'
        )
    # 走势折线
    parts.append(
        f'<polyline points="{pts}" fill="none" stroke="#90caf9" stroke-width="1.2"/>'
    )
    # 起止点
    parts.append(
        f'<circle cx="{tx(0):.1f}" cy="{ty(bars[0].close):.1f}" r="2" fill="#aaa"/>'
    )
    parts.append(
        f'<circle cx="{tx(len(bars) - 1):.1f}" cy="{ty(bars[-1].close):.1f}" '
        f'r="2.5" fill="#fff"/>'
    )
    parts.append("</svg>")
    return "".join(parts)
```

在 `daily_report.py` 的 `_match_thumbnails` 里，想把占位色块换成真图时，把 `match-thumb` 的 div 内容替换为
`mini_price_svg(...)` 的返回值即可——缩略图所需的 bars 在模板里保存的时间区间，扫描阶段按模板 `end_date ± N` 去 MySQL
拉一段塞到 `TemplateMatch` 里就行，属于第二轮增强。

### **落地校验清单（按步执行）**

第一步（数据层）：运行新版 `identify_high_potential_structures.py`，打开 `high_potential_templates.jsonl` 任取一条，检查
`outcome.up_move / down_move / days_to_peak / days_to_trough / outcome_start_date` 五个字段都在，并且
`outcome_start_date` 严格晚于 `end_date` 3 天以上。这条通过代表前瞻泄漏被封死。

第二步（聚合层）：写一个最小测试，用一条人工构造的 current_inv 加 3 条方向不同的 TemplateMatch，喂给 `aggregate_opportunity`
，确认输出的 Opportunity 里方向是多数投票结果、`direction_confidence` 等于多数占比、潜力分位是同方向 move
的分位数而不是全体混用。这条通过代表聚合逻辑对。

第三步（扫描层）：跑 `daily_scan.py`，打开生成的 JSON 快照，任取一条 opportunity，确认 `top_matches` 是 5 条而不是 1 条、每条都有
`diff_detail`、`evidence.config_hash` 与脚本里的一致、`attention_score` 在 0\~100 之间。这条通过代表信息流完整。

第四步（输出层）：打开 HTML 日报，按下面这条眼动流走一遍——顶部简报一眼看懂今日市场氛围 → 中部 5 张卡片每张都能回答"
该关心吗、涨跌、幅度区间、对照谁、下一步做什么"五个问题 →
底部表格按关注度排序且支持快速浏览。如果任意一张卡片上的任意一个问题答不上来，就是渲染或字段缺口，回到对应层补齐。这条通过代表用户体验达标。

---

### **这一轮整改的核心理念复述**

你这次提出的体验问题，本质不是"日报好不好看"，而是"整条管线的输出语义和用户的决策语义对不上"
。现在的整改方案做了三件事：一是把模板的前瞻统计补全，让"潜力"不再是一个数字而是一个分布；二是把 `best_match` 升级为
`top_k_matches`，并独立出聚合层，让"方向""幅度""像什么"都从多样本里统计而不是从单一最近邻里猜；三是把渲染层从"算法输出表"
改造成"研究决策三段式"，让用户的眼动流和信息结构对齐。

做完这四步以后，日报不再是一张扁平的匹配表，而是一份研究简报。用户打开它，从上到下就能自然回答"
先看谁、往哪走、走多远、对照谁、下一步做什么"这五个问题，这才是你想要的"指引我去对照、思考"的体验。
