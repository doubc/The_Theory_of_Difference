# **价格结构形式系统 工程设计书 v2.0**

> v1.0 → v2.0：对齐《差异论 V1.6》，引入投影框架、运动态、守恒、最近稳态。
> 理论→定义→工程，三层逐条映射。

## **一、项目定义**

构建一个**价格结构形式语言 + 结构编译器 + 历史检索引擎 + 结构学习系统**。

系统每日读取本地行情，将价格序列编译为结构对象，识别其类型与阶段与**运动态**，检索历史相似结构，输出后验统计。

**不做**：自动交易、短期价格预测、K线形态库、端到端深度学习黑箱。
**做**：结构本体定义、结构对象化、相似性检索、条件分布研究。

**核心认识（V1.6 D0）**：
价格是现实差异的低维压缩投影（影子）。
系统分析的是影子，不是实体。
系统必须知道自己在看影子——投影觉知是第一等公民。

---

## **二、核心理论链（项目宪法）**

1. **价格是影子**：高维现实差异 Π 投影为低维价格序列。Π 有损、不可逆、非线性、时变。 *(D0.1)*
2. **信息沿迟滞、分层、反馈的时序过程进入价格**。
3. **关系不变量在投影下存活**：速度比、时间比等无量纲比率可能是投影不变量。 *(D0.4)*
4. **结构 = 差异被组织后的骨架**：差异在共同反差下聚簇，形成 Zone、Cycle、Structure。 *(D2.1, D2.2)*
5. **系统 = 结构 × 运动**：没有运动的结构是死骨架，没有结构的运动是噪声。 *(D9.1)*
6. **守恒：差异不能被无代价清零**：被压缩的差异沿低阻通道转移。 *(D4.1, D4.2)*
7. **最近稳态**：系统先滑向最近能稳住的安排，不是最优解。 *(D3.2, D3.4)*
8. **研究对象是投影不变量**，不是K线图形。

### **阶段 2:对象系统(Week 6–7)**

**任务**:只定义对象,不实现逻辑。

**对象模型(伪代码)**

```
Point {
  t, x                      # 时间、价格位置
}

Segment {
  start: Point, end: Point
  duration, delta, avg_rate
  direction ∈ {+1, -1, 0}
  noise_level
}

Zone {
  price_center, bandwidth
  source ∈ {high_cluster, low_cluster, pivot, volume}
  strength
  context_contrast: ContrastType  # 共同反差 (D2.2)
  contrast_label: str
}

ContrastType {              # V1.6 D2.2: 驱动聚簇的外部差异类型
  PANIC | OVERSUPPLY | POLICY | LIQUIDITY | SPECULATION | UNKNOWN
}

NearestStableState {        # V1.6 D3.2: 差异转移后最先停驻的位置
  zone: Zone | None
  arrival_point: Point
  duration_to_arrive: float
  resistance_level: float   # 越低越"最近"
}

Cycle {
  entry: Segment            # 进入关键区
  exit:  Segment            # 离开关键区
  zone:  Zone
  next_stable: NearestStableState | None  # 最近稳态 (D3.2)
  speed_ratio = |v_exit| / |v_entry|
  time_ratio  = Δt_entry / Δt_exit
}

Structure {
  zone: Zone
  cycles: List[Cycle]
  phases: List[Phase]       # 生成/确认/破坏/反演
  invariants: Dict          # 关系不变量
  narrative_context: str    # 叙事标签 (D2.3)
  motion: MotionState       # 运动态 (D9.2)
  projection: ProjectionAwareness  # 投影觉知 (D0)
}

MotionState {               # V1.6 D9.2: 差异正在如何运动
  phase_tendency: str       # →breakdown / →confirmation / stable
  conservation_flux: float  # 守恒通量（正=释放，负=压缩）
  stable_distance: float    # 距最近稳态的距离
  stable_velocity: float    # 趋近稳态的速度
  transfer_source/target: str  # 差异转移路径
  structural_age: int       # 结构已存活的 cycle 数
}

ProjectionAwareness {       # V1.6 D0: 系统对自身认知边界的觉知
  compression_level: float  # 投影压缩度 [0,1]
  blind_channels: List[str] # 差异可能藏在的通道
  projection_confidence: float
  is_blind: bool            # 高压缩时=True，系统在看假象
}

Bundle {
  structures: List[Structure]
  generator_constraint      # 共同生成约束
}

Relation (算子,不存储)
  first_diff(p1, p2)
  second_diff(p1, p2, p3)
  distance_to_zone(p, z)
  extrema_similarity(points)
  compute_motion(structure)     # 计算运动态
  compute_projection(structure) # 计算投影觉知
  check_conservation(structure) # 守恒检查
```

**交付物**:类图 + Pydantic/dataclass 定义 + 存储格式。
**验收**:能用对象手工描述一个真实历史案例。

### **阶段 3:结构编译器(Week 8–12)**

**任务**:实现 价格序列 → 结构候选 的四层流水线。

**流水线伪代码**

```
def compile(bars) -> List[Structure]:
    # 3.1 极值提取（强制交替）
    pivots = extract_pivots(bars, 
                            min_amplitude=θ_a, 
                            min_duration=θ_t,
                            noise_filter=θ_n)
    
    # 3.2 段生成
    segments = build_segments(pivots)
    
    # 3.3 关键区识别（含共同反差推断）
    zones = detect_zones(pivots, segments,
                         methods=[cluster_highs, cluster_lows, 
                                  pivot_center, volume_profile])
    
    # 3.4 结构候选（含最近稳态检测）
    cycles = build_cycles(segments, zones)
    structures = assemble_structures(cycles, zones,
                                     min_cycles=2,
                                     tolerance=η)
    
    # 3.5 运动态 + 投影觉知 + 守恒检查
    for st in structures:
        st.invariants["conservation"] = check_conservation(st)
        st.motion = compute_motion(st)          # 系统 = 结构 × 运动
        st.projection = compute_projection(st)  # 价格 = Π(差异)
        st.narrative_context = infer_narrative_context(st)
    
    return structures
```

**关键参数必须可配置**:最小摆动幅度、最小持续时间、zone 带宽、容差 η。

**交付物**:4 个子模块 + 单元测试 + 可视化验证工具。
**验收**:对你熟悉的历史行情,输出与人眼判断大致一致。

### **阶段 4:规则引擎与 DSL(Week 13–16)**

**任务**:把主观结构翻译成可执行规则。

**DSL 示例**

```
STRUCT SlowUpFastDown_TopReversal:
    window:        [60d, 240d]         # 尺度弹性
    zone:          detect(method=high_cluster, tol=2%)
    cycles:        >= 3
    entry_pattern: slow_up    # v_entry < θ1
    exit_pattern:  fast_down  # v_exit > θ2
    speed_ratio:   v_exit / v_entry > 1.5
    time_ratio:    Δt_entry / Δt_exit > 1.8
    highs_cluster: stddev < η
    resolution:    break_below(support) AND follow_through(5d)

STRUCT FastUpSlowDown_TopDistribution:
    # 镜像参数,结构同态
    ...
```

**第一批只做 3–5 个结构**:

- 慢涨急跌顶部反转
- 急涨慢跌顶部派发
- 底部多次试探突破
- 压缩后单边释放
- 趋势中继整理

**交付物**:DSL 解析器 + 规则执行器 + 5 个结构定义 + 单测。
**验收**:一个结构可以"文字描述 → DSL → 程序扫出"全链路通。

### **阶段 6:相似性检索(Week 21–26)**

**任务**:最可能早期产出价值的模块。

**三层相似度**

```
sim_total = w1·sim_geometric + w2·sim_relational + w3·sim_family

sim_geometric   : 幅度、持续、极值位置  (欧氏/DTW)
sim_relational  : 试探次数、方向序列、速度比、zone 关系
sim_family      : 是否同一 Bundle、是否镜像变体
```

**检索流程**

```
def retrieve(current_structure, top_k=10):
    candidates = sample_library.filter_by_type()
    scored = [(s, similarity(current_structure, s)) 
              for s in candidates]
    top = sorted(scored)[:top_k]
    
    outcomes = [s.forward_outcome for s, _ in top]
    return {
        'neighbors': top,
        'posterior': aggregate_distribution(outcomes),
        'matched_invariants': diff_invariants(current, top),
        'sample_size': len(top)
    }
```

**输出**:相似案例清单 + 后验收益/回撤分布 + 约束匹配报告。

**交付物**:相似度定义文档 + 检索引擎 + 每日扫描脚本。
**验收**:能稳定回答"今天哪些品种的结构像历史上哪些案例"。

### **阶段 7:结构学习模型(Week 27–36)**

**任务**:用模型补足规则系统的柔性,不是替代。

**三类任务**

```
T1  Structure Embedding
    input:  compiled Structure (or feature vector)
    output: vector ∈ R^d
    loss:   contrastive / triplet
    goal:   同类近、镜像可映射、异类远

T2  Structure Classification  
    input:  Structure
    output: {type, phase, confirmed/broken/failed}

T3  Conditional Distribution
    input:  Structure + context
    output: P(future_return | structure, regime)
```

**技术路线(由简到繁)**

```
Stage A: 特征工程 + XGBoost/LightGBM
Stage B: 1D CNN / LSTM on segment sequence
Stage C: Transformer encoder + contrastive pretraining
Stage D: 向量检索 (FAISS) 集成回 M7
```

**验证铁律**

- 时间顺序切分,禁止随机打乱
- walk-forward 回测
- 跨品种验证
- 报告样本内/外差距

**交付物**:训练数据生成器 + 3 类模型 + 验证报告。
**验收**:样本外相似性排序或分类优于规则基线。

- 参数调试器(修改阈值重新回扫)
