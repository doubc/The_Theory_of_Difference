# **价格结构形式系统 工程设计书 v1.0**

## **一、项目定义**

构建一个**价格结构形式语言 + 结构编译器 + 历史检索引擎 + 结构学习系统**。

系统每日读取本地行情，将价格序列编译为结构对象，识别其类型与阶段，检索历史相似结构，输出后验统计。

**不做**：自动交易、短期价格预测、K线形态库、端到端深度学习黑箱。
**做**：结构本体定义、结构对象化、相似性检索、条件分布研究。

---

## **二、核心理论链（作为项目宪法）**

1. 价格是高维信息与约束状态的低维压缩投影。
2. 信息沿迟滞、分层、反馈的时序过程进入价格。
3. 组织关系相似 → 价格路径保留结构同态。
4. 传播效率变化 → 结构发生时间压缩与表观变形，但关系不变量可能保留。
5. 宏观与制度约束 → 结构演化具阶段性稳定性。
6. 研究对象是**关系不变量**，不是K线图形。

```
A5  结构通过阶段转换生成、确认、破坏、反演

```

### **阶段 2:对象系统(Week 6–7)**

**任务**:只定义对象,不实现逻辑。

**对象模型(伪代码)**

```
Point {
  t, x                      # 时间、价格位置
}

Segment {
  start: Point, end: Point
  duration, delta, avg_rate
  direction ∈ {+1, -1, 0}
  noise_level
}

Zone {
  price_center, bandwidth
  source ∈ {high_cluster, low_cluster, pivot, volume}
  strength
}

Cycle {
  entry: Segment            # 进入关键区
  exit:  Segment            # 离开关键区
  zone:  Zone
  speed_ratio = |v_entry| / |v_exit|
  time_ratio  = Δt_entry / Δt_exit
}

Structure {
  zone: Zone
  cycles: List[Cycle]
  phases: List[Phase]       # 生成/确认/破坏/反演
  invariants: Dict          # 关系不变量
}

Bundle {
  structures: List[Structure]
  generator_constraint      # 共同生成约束
}

Relation (算子,不存储)
  first_diff(p1, p2)
  second_diff(p1, p2, p3)
  distance_to_zone(p, z)
  extrema_similarity(points)
```

**交付物**:类图 + Pydantic/dataclass 定义 + 存储格式。
**验收**:能用对象手工描述一个真实历史案例。

### **阶段 3:结构编译器(Week 8–12)**

**任务**:实现 价格序列 → 结构候选 的四层流水线。

**流水线伪代码**

```
def compile(bars) -> List[Structure]:
    # 3.1 极值提取
    pivots = extract_pivots(bars, 
                            min_amplitude=θ_a, 
                            min_duration=θ_t,
                            noise_filter=θ_n)
    
    # 3.2 段生成
    segments = build_segments(pivots)
    
    # 3.3 关键区识别
    zones = detect_zones(pivots, segments,
                         methods=[cluster_highs, cluster_lows, 
                                  pivot_center, volume_profile])
    
    # 3.4 结构候选
    cycles = build_cycles(segments, zones)
    structures = assemble_structures(cycles, zones,
                                     min_cycles=2,
                                     tolerance=η)
    return structures
```

**关键参数必须可配置**:最小摆动幅度、最小持续时间、zone 带宽、容差 η。

**交付物**:4 个子模块 + 单元测试 + 可视化验证工具。
**验收**:对你熟悉的历史行情,输出与人眼判断大致一致。

### **阶段 4:规则引擎与 DSL(Week 13–16)**

**任务**:把主观结构翻译成可执行规则。

**DSL 示例**

```
STRUCT SlowUpFastDown_TopReversal:
    window:        [60d, 240d]         # 尺度弹性
    zone:          detect(method=high_cluster, tol=2%)
    cycles:        >= 3
    entry_pattern: slow_up    # v_entry < θ1
    exit_pattern:  fast_down  # v_exit > θ2
    speed_ratio:   v_exit / v_entry > 1.5
    time_ratio:    Δt_entry / Δt_exit > 1.8
    highs_cluster: stddev < η
    resolution:    break_below(support) AND follow_through(5d)

STRUCT FastUpSlowDown_TopDistribution:
    # 镜像参数,结构同态
    ...
```

**第一批只做 3–5 个结构**:

- 慢涨急跌顶部反转
- 急涨慢跌顶部派发
- 底部多次试探突破
- 压缩后单边释放
- 趋势中继整理

**交付物**:DSL 解析器 + 规则执行器 + 5 个结构定义 + 单测。
**验收**:一个结构可以"文字描述 → DSL → 程序扫出"全链路通。

### **阶段 6:相似性检索(Week 21–26)**

**任务**:最可能早期产出价值的模块。

**三层相似度**

```
sim_total = w1·sim_geometric + w2·sim_relational + w3·sim_family

sim_geometric   : 幅度、持续、极值位置  (欧氏/DTW)
sim_relational  : 试探次数、方向序列、速度比、zone 关系
sim_family      : 是否同一 Bundle、是否镜像变体
```

**检索流程**

```
def retrieve(current_structure, top_k=10):
    candidates = sample_library.filter_by_type()
    scored = [(s, similarity(current_structure, s)) 
              for s in candidates]
    top = sorted(scored)[:top_k]
    
    outcomes = [s.forward_outcome for s, _ in top]
    return {
        'neighbors': top,
        'posterior': aggregate_distribution(outcomes),
        'matched_invariants': diff_invariants(current, top),
        'sample_size': len(top)
    }
```

**输出**:相似案例清单 + 后验收益/回撤分布 + 约束匹配报告。

**交付物**:相似度定义文档 + 检索引擎 + 每日扫描脚本。
**验收**:能稳定回答"今天哪些品种的结构像历史上哪些案例"。

### **阶段 7:结构学习模型(Week 27–36)**

**任务**:用模型补足规则系统的柔性,不是替代。

**三类任务**

```
T1  Structure Embedding
    input:  compiled Structure (or feature vector)
    output: vector ∈ R^d
    loss:   contrastive / triplet
    goal:   同类近、镜像可映射、异类远

T2  Structure Classification  
    input:  Structure
    output: {type, phase, confirmed/broken/failed}

T3  Conditional Distribution
    input:  Structure + context
    output: P(future_return | structure, regime)
```

**技术路线(由简到繁)**

```
Stage A: 特征工程 + XGBoost/LightGBM
Stage B: 1D CNN / LSTM on segment sequence
Stage C: Transformer encoder + contrastive pretraining
Stage D: 向量检索 (FAISS) 集成回 M7
```

**验证铁律**

- 时间顺序切分,禁止随机打乱
- walk-forward 回测
- 跨品种验证
- 报告样本内/外差距

**交付物**:训练数据生成器 + 3 类模型 + 验证报告。
**验收**:样本外相似性排序或分类优于规则基线。

**交付物**:日更脚本 + Web 面板。
**验收**:你每天真的在用,而不是零散跑 notebook。

## **五、关键风险与对策**

| 风险       | 对策                   |
|----------|----------------------|
| 理论写成绝对真理 | 用弱公理 + 可证伪假设         |

| 模式进化失效   | 结构半衰期监控 + 定期重训       |
| 过早做交易系统  | 第一年只做研究底座            |
