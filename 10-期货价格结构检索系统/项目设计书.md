## **〇、理论地基：差异论 V1.6 + 象界**

本系统同时建立在两部理论之上：

**《差异论 V1.6》** 提供：
- 价格是差异的临时显影，不是现实本身 (D13.1)
- 守恒：差异不能无代价清零，只能转移 (D4.1)
- 最近稳态：系统先滑向最近能稳住的安排 (D3.4)
- 系统 = 结构 × 运动 (D9.1)

**《象界》** 提供前主体态生成链——**价格结构的本体论**：
- 象 = 差异达到可辨认、可维持、可延续的组织密度时的状态显现
- 边界→界面：Zone 不是支撑阻力线，是调节价格与区交互的**界面**
- 闭合→自维持：Cycle 不是简单的进出配对，是结构在开放市场中**自我维持**的方式
- 痕迹→记忆：不变量跨 Cycle 保留 = **结构记忆**
- 再现→复制：同一 Zone 反复生成类似结构 = **结构复制**（带偏差的延续）
- 并存→筛选：市场自然选择——能自我维持的结构存活，不能的消散
- 耦合→功能：Zone 内部不同部分对结构持存的贡献不对称
- **前主体态**：所有机制汇聚 = Structure 的三重态（结构×运动×投影）

**核心洞察**：本系统识别的"价格结构"，就是价格差异在市场中达到**前主体态密度**时的组织显现。
系统不是在做"K线形态匹配"，而是在识别**差异已经跨过象界门槛、具备自我维持能力的组织状态**。

## **一、项目定义（不变）**

构建一个**价格结构形式语言 + 结构编译器 + 历史检索引擎 + 结构学习系统**。

系统每日读取本地行情，将价格序列编译为结构对象，识别其类型与阶段与**运动态**，检索历史相似结构，输出后验统计。

**不做**：自动交易、短期价格预测、K线形态库、端到端深度学习黑箱。
**做**：结构本体定义、结构对象化、相似性检索、条件分布研究。

**核心认识（V1.6 D0）**：
价格是现实差异的低维压缩投影（影子）。
系统分析的是影子，不是实体。
系统必须知道自己在看影子——投影觉知是第一等公民。

## **二、核心理论链（项目宪法）**

### **差异论 V1.6 命题**

1. **价格是影子**：高维现实差异 Π 投影为低维价格序列。Π 有损、不可逆、非线性、时变。 *(D0.1)*
2. **信息沿迟滞、分层、反馈的时序过程进入价格**。
3. **关系不变量在投影下存活**：速度比、时间比等无量纲比率可能是投影不变量。 *(D0.4)*
4. **结构 = 差异被组织后的骨架**：差异在共同反差下聚簇，形成 Zone、Cycle、Structure。 *(D2.1, D2.2)*
5. **系统 = 结构 × 运动**：没有运动的结构是死骨架，没有结构的运动是噪声。 *(D9.1)*
6. **守恒：差异不能被无代价清零**：被压缩的差异沿低阻通道转移。 *(D4.1, D4.2)*
7. **最近稳态**：系统先滑向最近能稳住的安排，不是最优解。 *(D3.2, D3.4)*
8. **研究对象是投影不变量**，不是K线图形。

### **象界前主体态命题**

9. **Zone 是界面，不是围墙**：Zone 调节价格与区的交互（哪些通过、哪些阻断、哪些延迟），不是简单的支撑阻力。 *(象界 Ch2)*
10. **Cycle 是自维持，不是重复**：一个 Cycle 如果只能在封闭条件中存在，它只是闭合；只有在开放市场波动中仍能保持进出配对，才叫自维持。 *(象界 Ch3)*
11. **不变量是结构记忆**：跨 Cycle 保留的 speed_ratio / time_ratio 不是统计量，是结构的路径偏置——过去对未来的约束。 *(象界 Ch4)*
12. **结构复制带偏差**：同一 Zone 反复生成的结构不是原样复制，而是关键关系的再构造 + 偏差保留。 *(象界 Ch5)*
13. **市场筛选是延续能力的分流**：不是谁在决定取舍，而是不同结构样式在开放条件中的持存能力自然不同。 *(象界 Ch6)*
14. **反身性**：系统的输出会改变被观测对象，方法会压缩自身有效性。 *(Ch14)* — v2.5 新增

## **四、v2.5 理论驱动升级详解**

### **升级 1：跨维度守恒验证（D4.2 命题 4.3）**

**理论根据**：
> 定义 4.2：转移，是差异在原有通道受阻后，沿着其他较低阻力路径继续存在和积累的过程。
> 命题 4.3：差异转移通常沿着低阻通道发生。

**v2.0 的问题**：`check_conservation()` 只在日线维度内检查速度比和带宽变化。但理论明确说差异会**转移到其他维度**——如果只看一个维度，守恒检查永远只能说"可能在转移"，永远无法确认。

**v2.5 设计**：

```
DEF D4.2-v2.5 跨维度守恒验证

  低阻通道排序（理论 D4.2 + D0.3）：
    1. shorter_timeframe  — 转到更短周期（阻力最低）
    2. volume             — 转到成交量异动
    3. open_interest      — 转到持仓变化
    4. cross_symbol       — 转到相关品种

  验证流程：
    当 check_conservation() 检测到日线差异被压缩时：
    1. 加载同品种 5 分钟线数据
    2. 计算 5min 线的波动率变化
    3. 如果 5min 波动率上升而日线波动率下降
       → 确认差异转移到 shorter_timeframe
    4. 检查 volume 变异系数是否放大
       → 确认差异转移到 volume

  输出：
    conservation_result = {
      daily_compressed: bool,
      transfer_confirmed: bool,
      transfer_target: str,        # "5min" | "volume" | "both" | "unconfirmed"
      transfer_strength: float,     # 转移强度 [0, 1]
    }
```

**代码改动**：
- `relations.py`: `check_conservation()` 增加 `bars_5min` 参数
- `pipeline.py`: `compile_full()` 增加可选的 `bars_5min` 输入
- `models.py`: `Structure` 增加 `cross_freq_transfer` 字段

### **升级 2：反身性追踪（Ch14 命题 14.4）**

**理论根据**：
> 命题 14.4：方法失效的根本原因，不是市场突然无规律，而是方法在被广泛采用后，会逐步压缩支撑它有效的差异。

**v2.0 的问题**：系统输出结构匹配和规则命中，但不追踪这些输出是否反过来改变了市场。一个规则如果被很多人采用，它的有效性会衰减——但系统对此一无所知。

**v2.5 设计**：

```
DEF Ch14-v2.5 反身性追踪

  ReflexivityRecord = {
    rule_name: str,              # 哪条规则
    match_date: str,             # 匹配日期
    structure_snapshot: dict,    # 匹配时的结构快照
    forward_outcome: {           # 后续结果（5/10/20日）
      ret_5d: float,
      ret_10d: float,
      ret_20d: float,
      max_dd: float,
    },
    prediction_correct: bool,    # 规则预测是否正确
  }

  ReflexivityTracker = {
    records: List[ReflexivityRecord],

    # 核心方法
    record_match(rule, structure, date):
      → 记录一次规则匹配

    backfill_outcome(record_id, bars_after):
      → 回填后续结果

    decay_detection(rule_name, window=20):
      → 检测规则准确率是否在衰减
      → 如果近 window 次匹配的准确率 < 历史平均 - 2σ
      → 标记为 "可能正在经历反身性衰减"

    rule_health_report():
      → 每条规则的准确率趋势、衰减状态、建议权重
  }
```

**代码改动**：
- `src/reflexivity.py`: 从骨架升级为完整实现
- `src/dsl/rule.py`: `scan()` 输出附加反身性追踪
- `src/workbench/app.py`: 增加"规则健康度"面板

### **升级 3：叙事递归追踪（Ch9 命题 9.4）**

**理论根据**：
> 命题 9.4：叙事与现实是递归关系——叙事改变行为，行为改变现实，现实又改变叙事。

**v2.0 的问题**：`narrative_context` 是每次编译时静态生成的。同一 zone 在不同日期可能有不同的叙事（比如从"恐慌性结构"变成"供需失衡结构"），但系统不追踪这种变化。

**v2.5 设计**：

```
DEF Ch9-v2.5 叙事递归

  每个 Zone 的叙事不是固定的，而是随时间演化的：
    narrative_t0 = "恐慌性结构 · 急跌型 · 3次密集试探"
    narrative_t1 = "恐慌性结构 · 均衡型 · 5次试探 · 趋向确认"
    narrative_t2 = "供需失衡结构 · 慢跌型 · 4次试探 · 差异压缩中"

  叙事变化本身是信号：
    - 从"恐慌"变成"均衡" → 市场在去恐慌化
    - 从"趋向确认"变成"趋向破坏" → 结构在恶化
    - 叙事中出现"⚠️高压缩" → 投影觉知在报警

  知识图谱中已有 narrative_evolves 边 → 需要激活并展示
```

**代码改动**：
- `src/graph/store.py`: `daily_ingest()` 已有叙事递归边逻辑 → 需要确保 narrative 变化时正确建立
- `src/workbench/app.py`: 增加"叙事演化"时间线视图

### **升级 4：投影觉知深化（D0.3 代码的系统性盲区）**

**理论根据**：
> D0.3：因为代码只处理 Π(P)，存在结构性盲区。
> D0.2：投影压缩时，被压缩的维度不会消失，只会变得不可见。

**v2.0 的问题**：`ProjectionAwareness` 只标记"高压缩"和列出盲区通道名称，但不给出**具体的可操作信息**——比如"在高压缩期间，你应该关注什么"。

**v2.5 设计**：

```
DEF D0.3-v2.5 投影觉知增强

  ProjectionAwareness 增加：
    observation: str           # 人可读（已有）
    recommended_actions: list  # v2.5: 系统建议研究者做什么
    blind_evidence: dict       # v2.5: 每个盲区通道的具体证据

  示例输出：
    {
      compression_level: 0.85,
      is_blind: true,
      blind_channels: ["5min", "volume"],
      recommended_actions: [
        "⚠️ 当前日线 Zone 带宽极窄(0.3%)，差异可能在 5min 线积累",
        "建议：检查 5min 线波动率是否在上升",
        "建议：检查成交量变异系数是否在放大",
      ],
      blind_evidence: {
        "5min": "日线波动率下降40%，待验证5min线",
        "volume": "Zone内外成交量CV比=0.8，流动性偏枯竭",
      }
    }
```

**代码改动**：
- `src/relations.py`: `compute_projection()` 增加 `recommended_actions` 和 `blind_evidence`
- `src/models.py`: `ProjectionAwareness` 增加两个新字段

### **升级 5：算法层优化（非理论驱动，但提升系统能力）**

以下改进**不直接对应 V1.6 命题**，但提升编译器和检索器的工程能力：

**5a. 自适应极值窗口**（pivots.py）
- 动机：固定窗口在急涨急跌中丢失极值，在慢震荡中产生噪声
- 方法：局部波动率驱动窗口大小 + 分形一致性验证
- 理论关联：间接支持 D3.1（事件是离散的）——更好的极值检测 = 更好的事件识别

**5b. DTW 相似度**（similarity.py）
- 动机：`extrema_similarity` 是 stub
- 方法：Dynamic Time Warping 比较 Cycle 速度比序列
- 理论关联：间接支持 D0.4（关系不变量在投影下存活）——DTW 捕捉的是"形状不变量"

**5c. 段形状编辑距离**（similarity.py）
- 动机：两个结构的 Cycle 节奏是否相似
- 方法：将 Cycle 编码为离散符号，用编辑距离度量
- 理论关联：同上
