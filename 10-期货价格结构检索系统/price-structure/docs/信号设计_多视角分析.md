# 信号设计：多视角交集分析

> 方法论：采用**多视角交集法**——每个视角对同一问题独立给出判断（子集），取所有子集的**交集**（所有视角共同认可的部分）作为最终高置信度信号设计方案。
> 
> 四个视角：量化交易员 / 技术分析师 / 风险管理师 / 做市商

---

## 一、问题定义

**目标**：把当前扫描器输出的"结构描述卡片"升级为"可辅助交易决策的信号"，在现有 `Structure` / `MotionState` / `conservation_flux` / `stability_verdict` 等字段基础上，补充信号层设计。

---

## 二、各视角分析

### 2.1 量化交易员视角（Quant）

**核心关切**：信号可回测、可量化、胜率/赔率可验证

| 信号 | 量化定义 | 触发条件 |
|------|----------|---------|
| Zone突破确认 | 5维评分 ≥ 0.8 | `close_breakout_depth > 0.3×bandwidth AND volume_ratio > 1.5 AND flux_consistency = True` |
| 假突破识别 | 5种模式之一命中 | `FAKE_PIN OR FAKE_DSPIKE OR FAKE_VOLDIV OR FAKE_BLIND_WHIP OR FAKE_GAP` |
| 回踩确认入场 | 突破后回踩不破 | `breakout_confirmed AND pullback_low > zone_upper (long) OR pullback_high < zone_lower (short)` |
| 结构老化失效 | 超时未触发 | `days_since_formation > 20 AND motion_state == 'forming'` |

**特别强调**：`conservation_flux` 必须参与所有信号的过滤条件，量价背离是最可靠的否定信号。

### 2.2 技术分析师视角（TA）

**核心关切**：价格行为的市场微观结构，形态识别的实战有效性

| 信号 | 技术定义 | 关键判定 |
|------|----------|---------|
| Zone突破确认 | 收盘站稳 + 量能扩张 | 收盘突破且下一根K线不回破，量能 ≥ 近20根中位数×1.5 |
| 假突破折返 | 探针后立即收回 | 盘中突破但收盘回到Zone内，且`conservation_flux`与突破方向相反 |
| 突破回踩确认 | 高胜率最佳入场点 | 突破确认后，价格回踩Zone边界不破，形成次低点/次高点 |
| 多时间框架共振 | 日线+1h同向 | `daily_bias == 1h_bias AND 5m_bias != opposite`，且日线质量层 ≥ B |

**额外贡献**：提出"突破回踩确认"是实战中胜率最高的入场形态，应作为信号输出的优先推荐点位。

### 2.3 风险管理师视角（Risk）

**核心关切**：最大回撤可控、信号失效早知道、仓位与结构质量挂钩

| 信号 | 风险定义 | 风控规则 |
|------|----------|---------|
| Zone突破确认 | 需叠加失效条件 | 突破失效 = 收盘回到Zone内超过2根K线 → 立即平仓/不入场 |
| 假突破识别 | 是风险也是机会 | 假突破确认后，反向入场，止损设在假突破极端点外 |
| 结构质量过滤 | 低质量结构不交易 | 仅 `quality_tier == 'A' OR ('B' AND stability_verdict != 'red')` 可下单 |
| 盲区交易禁止 | `is_blind == True` 时禁止方向性交易 | 盲区只做跨品种对冲，不做单边 |
| 仓位与质量绑定 | 质量层直接映射仓位系数 | A=1.0, B=0.6, C=0.3, D=0 |

**特别强调**：`stability_verdict`（错觉检测）是独特价值，红灯信号应直接覆盖方向性判断。

### 2.4 做市商视角（Market Maker）

**核心关切**：流动性供给、订单流、盲区引爆、区间边界的订单堆积

| 信号 | 做市定义 | 订单流逻辑 |
|------|----------|-----------|
| Zone突破确认 | 是流动性吞噬事件 | 突破伴随大单吃掉Zone边界挂单 → 确认度最高；若突破无量，可能是诱导 |
| 假突破 | 是流动性猎杀 | 主动扫单诱导止损盘后反向吃单 → 假突破后反向是做市商标准操作 |
| 盲区引爆 | `is_blind == True` 时的突破 | 盲区突破 = 无共识区域的价格暴力寻路 → 突破幅度往往超预期 |
| 区间边界订单堆积 | Zone边界是限价单密集区 | 多次试探后在Zone边界形成大量限价单 → 突破时是流动性洪峰 |
| 跨品种引流 | 相关品种突破带动 | 板块龙头突破 → 跟风品种滞后突破 → 可提前布局跟风品种 |

## 三、交集结论

| 信号 | Quant | TA | Risk | MM | 交集结论 |
|------|-------|-----|------|----|---------|
| Zone突破确认 | ✅ | ✅ | ✅（需加失效条件） | ✅（需看流动性） | **强共识**：4/4认可，Risk和MM各提了一个必要条件 |
| 假突破识别/交易 | ✅ | ✅ | ✅（是机会） | ✅（是猎杀） | **强共识**：4/4认可，逻辑完全自洽 |
| 突破回踩确认 | — | ✅ | — | — | **单视角**：仅TA提出，但实战价值高，保留为TA专属信号 |
| 结构质量过滤 | — | — | ✅ | — | **部分共识**：Risk明确提出，Quant隐含在评分里，需显式化 |
| 盲区交易策略 | — | — | ❌（禁止） | ✅（引爆突破） | **核心分歧**：Risk禁止 vs MM引爆突破 |
| 多时间框架共振 | — | ✅ | — | — | **单视角**：TA提出，符合差异论多尺度精神，建议保留 |

### 强共识要点

**共识1**：Zone突破确认信号必须包含 `conservation_flux` 同向验证
- Quant: 量价背离是最可靠否定信号
- TA: 量能扩张必须与通量方向一致
- Risk: 通量反向时方向性判断直接作废
- MM: 无量突破是诱导，通量揭示订单流真相

**共识2**：假突破是核心风险也是核心机会，判定关键是 `flux方向与价格突破方向相反`

**共识3**：`conservation_flux` 是最核心的判别字段，应参与所有信号的过滤条件

**共识4**：`stability_verdict`（错觉检测/红绿灯）是系统的独特价值，应作为信号准入的第一道门槛

**共识5**：结构老化/失效需要显式建模（3/4认可，MM未提及但也不反对）

### 核心分歧：盲区交易策略

| 视角 | 立场 | 理由 |
|------|------|------|
| Risk | **禁止**单边方向性交易 | 盲区=无共识，方向性判断期望值≈0，止损难以设定 |
| MM | **主动参与**引爆突破 | 盲区突破=流动性事件，突破幅度往往超预期 |
| Quant | 未明确表态 | 现有数据不足以回测盲区突破的期望收益 |
| TA | 未明确表态 | 技术形态在盲区同样有效，但样本少 |

**暂定方案**：盲区突破信号保留，但标记为 `⚠️ 盲区·低样本`，在回测积累到 N≥30 之前仅作观察、不推荐下单。

## 四、落地建议

### 4.1 信号层叠加

在 `src/workbench/tab_scan.py` 的结构卡片下方新增「信号」区块：

```python
signal = generate_signal(structure)
# signal 可能值：
# - "突破确认·做多/做空"  (5维评分≥0.8 + flux同向)
# - "假突破·反向做多/做空"  (假突破模式命中 + flux反向)
# - "回踩确认·观察入场"  (TA专属，突破后回踩不破)
# - "结构老化·移除观察"  (days_since_formation > threshold)
# - "盲区突破·观察"  (is_blind=True，仅观察)
# - None  (无信号)
```

信号优先级：`假突破反向` > `突破确认` > `回踩确认` > `结构老化`

### 4.2 信号输出结构

```python
@dataclass
class Signal:
    kind: str          # 'breakout_confirm' | 'fake_breakout' | 'pullback_confirm' | 'structure_expired'
    direction: str     # 'long' | 'short' | 'neutral'
    confidence: float  # 0-1，基于质量层+视角共识数
    flux_aligned: bool # conservation_flux是否与信号方向一致
    stability_ok: bool # stability_verdict != 'red'
    entry_note: str    # 自然语言入场说明
```

### 4.3 5维突破评分权重

| 维度 | 权重 | 计算方式 |
|------|------|---------|
| 收盘穿透深度 | 0.25 | `\|close - zone_boundary\| / bandwidth` |
| 成交量扩张比 | 0.25 | `volume / median(volume, 20d)` |
| 守恒通量一致性 | 0.15 | `flux_direction == breakout_direction` → 1 else 0 |
| 压缩蓄势度 | 0.20 | `n_tests / max_tests` |
| 驻留比 | 0.15 | `time_in_zone / total_time` |

评分 ≥ 0.8 → 强突破确认；0.55-0.8 → 待确认；< 0.55 → 弱/假突破

### 4.4 假突破5种模式

```
FAKE_PIN       = 盘中突破幅度 > 1.5×bandwidth AND 收盘回到Zone内 AND flux方向与突破方向相反
FAKE_DSPIKE    = 单根K线极端价格 AND 其余K线均在Zone内 AND volume_climax BUT flux_weak
FAKE_VOLDIV    = 价格突破Zone AND volume < median×0.8（量能背离）
FAKE_BLIND_WHIP = is_blind=True AND 价格快速突破Zone BUT 无follow_through
FAKE_GAP       = 跳空突破Zone BUT 缺口当日回补 AND flux_direction != gap_direction
```
