# 知识注入系统设计

> 系统目前有图谱的拓扑（谁连谁），缺图谱的语义（为什么连、意味着什么）。
> 本文档定义知识注入的架构、存储、查询、应用方式。

## 一、问题

当前系统的能力边界：

| 有 | 没有 |
|----|------|
| 结构编译（价格→结构对象） | 什么样的结构更可靠 |
| 相似度检索（几何/关系/运动/族） | 检索结果意味着什么 |
| 运动类型判定（趋势/震荡/反转） | 什么时候该信、什么时候该疑 |
| 知识图谱（结构间的关系） | 关系背后的因果逻辑 |
| 叙事生成（模板化的描述） | 基于经验的深度解读 |

核心矛盾：**系统能描述「是什么」，但不能回答「所以呢」。**

## 二、知识的三层

### L1: 判定知识（Condition → Verdict）

回答「这个结构现在处于什么状态，我该信多少」。

```yaml
# 示例
- name: 趋势可靠性递增
  when:
    - movement_type == "trend_up"
    - stable_count >= 3
    - flux > 0
  verdict: "连续 3 个稳态递增且通量释放，趋势可靠性高"
  confidence: 0.85
  weight_adjust: +0.15    # 检索时加权
  source: "历史回测"
```

**作用点**：质量评估（quality.py）的第五维度、检索排序。

### L2: 失效知识（Condition → Invalidate）

回答「什么条件下当前判断作废」。

```yaml
- name: 趋势通量背离
  when:
    - movement_type in ("trend_up", "trend_down")
    - flux 与 movement_type 方向相反
  invalidate: "通量方向与趋势方向背离，趋势可能中断"
  severity: high
  action: "降低关注度，等待通量方向确认"
```

**作用点**：每日扫描的失效条件提示、信号过滤。

### L3: 市场知识（Context → Wisdom）

回答「结合品种特征和市场环境，有什么值得注意的」。

```yaml
- name: 铜 Zone 带宽常识
  symbol: CU000
  normal_bandwidth: [0.01, 0.02]
  alert_bandwidth: 0.03
  wisdom: "铜的 Zone 带宽通常 1-2%，超过 3% 说明市场状态异常"

- name: 政策驱动结构特征
  contrast_type: POLICY
  wisdom: "政策驱动型结构试探间隔均匀，反弹快但持续性差，适合短线不适合趋势跟踪"
```

**作用点**：叙事生成、品种描述、研究建议。

## 三、存储架构

```
knowledge/
├── L1_conditions.yaml      # 判定知识（L1）
├── L2_invalidation.yaml    # 失效知识（L2）
├── L3_wisdom.yaml          # 市场知识（L3）
├── rules/                  # 可选：按品种/类型拆分
│   ├── metals.yaml
│   ├── energy.yaml
│   └── agriculture.yaml
└── README.md
```

**不引入数据库**。YAML 文件就是知识库，版本控制（git）就是知识的变更历史。

### YAML Schema

```yaml
# L1_conditions.yaml
rules:
  - id: "L1-001"
    name: "趋势可靠性递增"
    description: "连续多个稳态递增的趋势更可靠"
    when:
      - field: "movement_type"
        op: "in"
        value: ["trend_up", "trend_down"]
      - field: "stable_count"
        op: ">="
        value: 3
      - field: "flux"
        op: ">"
        value: 0
    verdict: "趋势可靠性高"
    confidence: 0.85
    weight_adjust: 0.15
    source: "历史回测 + 原理推导"
    note: "稳态越多，趋势的统计基础越扎实"
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | str | 唯一标识，L1/L2/L3 前缀 |
| `name` | str | 规则名称 |
| `when` | list | 条件列表（AND 关系） |
| `field` | str | 可用字段见下方 |
| `op` | str | `==`, `!=`, `>`, `<`, `>=`, `<=`, `in`, `not_in`, `contains` |
| `value` | any | 比较值 |
| `verdict` / `invalidate` / `wisdom` | str | 输出文本 |
| `confidence` | float | 置信度 [0, 1] |
| `weight_adjust` | float | 检索排序时的权重调整 |
| `source` | str | 知识来源（回测/原理/经验） |

**可用字段（`when.field`）：**

| 字段 | 来源 | 说明 |
|------|------|------|
| `movement_type` | MotionState | trend_up / trend_down / oscillation / reversal |
| `phase_tendency` | MotionState | →breakout / →confirmation / stable / forming |
| `flux` | MotionState.conservation_flux | 守恒通量 [-1, 1] |
| `stable_distance` | MotionState | 距最近稳态距离 |
| `stable_count` | Structure.cycle_count | 稳态/循环数 |
| `compression_level` | ProjectionAwareness | 投影压缩度 [0, 1] |
| `contrast_type` | Zone.context_contrast | PANIC / OVERSUPPLY / POLICY / ... |
| `zone_rel_bw` | Zone | 相对带宽 |
| `tier` | QualityAssessment | A / B / C / D |
| `label` | Structure | 规则引擎标签 |
| `symbol` | Structure | 品种代码 |

## 四、查询与应用

### 4.1 KnowledgeEngine

```python
class KnowledgeEngine:
    """知识引擎：加载 YAML 知识库，对结构进行知识匹配"""

    def __init__(self, knowledge_dir: str = "knowledge"):
        self.conditions = load_yaml(f"{knowledge_dir}/L1_conditions.yaml")
        self.invalidation = load_yaml(f"{knowledge_dir}/L2_invalidation.yaml")
        self.wisdom = load_yaml(f"{knowledge_dir}/L3_wisdom.yaml")

    def evaluate(self, structure: Structure, motion: MotionState) -> KnowledgeResult:
        """对一个结构执行全部知识规则，返回匹配结果"""
        matched_l1 = self._match_rules(self.conditions, structure, motion)
        matched_l2 = self._match_rules(self.invalidation, structure, motion)
        matched_l3 = self._match_wisdom(structure)
        return KnowledgeResult(
            conditions=matched_l1,      # 判定知识
            invalidations=matched_l2,   # 失效知识
            wisdoms=matched_l3,         # 市场知识
            confidence_boost=sum(r.weight_adjust for r in matched_l1),
        )
```

## 六、与图谱的关系

知识不替代图谱，图谱承载拓扑，知识承载语义：

```
价格序列
  → 编译器（结构对象）
    → 图谱（结构间的关系：演化/转移/共享）
    → 知识（结构的语义：可靠/失效/智慧）
      → 输出（质量分 + 剧本 + 失效条件 + 叙事）
```

图谱回答「这个结构和哪些结构有关」。
知识回答「这个结构意味着什么、该不该信」。

### 7.2 导入器：ProductKnowledgeIngester

`src/graph/product_ingester.py` 实现了统一导入引擎：

```python

store = GraphStore("data/graph")
ingester = ProductKnowledgeIngester(store)

# 全量导入所有 active 品种

# 增量导入（仅更新有变化的品种，通过文件 hash 判断）
```

**节点类型映射**：

| JSON 实体前缀 | GraphStore 节点类型 | 存储位置 |
|---------------|-------------------|----------|
| GEO_ | zone（地理节点） | zones.jsonl |
| POW_ | structure（权力机构） | structures.jsonl |
| RUL_ | structure（规则框架） | structures.jsonl |
| CUL_ | narrative（共识叙事） | narratives.jsonl |
| VAR_ | structure（核心变量） | structures.jsonl |
| 关系 | edge | edges.jsonl |
| 传导链 | narrative + edges | narratives.jsonl + edges.jsonl |
| 极值 | structure（polarity_rule） | structures.jsonl |
| 定价模型 | narrative（pricing_model） | narratives.jsonl |

**命名空间**：所有节点 ID 带品种前缀，如 `copper:GEO_066`、`_shared:R_001`。跨品种关系使用 `copper:LME铜价 → _shared:原油价格` 格式。
