# 差异即世界 → 价格结构系统：形式化定义层 v2.5

> 本文档是理论与代码之间的**唯一中间层**。
> 九个机制来自《差异即世界》的统一生成语法，逐条映射到价格结构系统。
> 修改代码前，必须先过这一层。
>
> 核心原则：一个语法，九次展开，价格结构是这套语法在市场差异上的显影。

## 总纲：统一语法的一句话

**差异先行，关系由此生成，结构由此沉淀，对象由此显现，主体由此诞生，世界由此成立。**

价格结构系统做的事情：从价格序列中识别差异走到了生成链的哪一步。

## 机制一：聚簇

**语法**：分散差异在共同约束下发生聚集，形成局部结构密度。

**价格映射**：

```
DEF 1.1 聚簇（Zone）

  输入：一组极值点 P = {p_1, ..., p_k}
  聚簇条件：
    ① 价格邻近性：max(|x_i - x_j|) / mean(P) < ε
    ② 时间跨度：span(P) > 0（非瞬时）
    ③ 密度：|P| ≥ min_points

  聚簇不是"价格接近所以放在一起"，
  聚簇是"这些极值点被同一个共同反差驱动，反复出现在同一价位"。

  共同反差（ContrastType）：
    PANIC      — span < 30天 ∧ |P| ≥ 3（恐慌性密集试探）
    OVERSUPPLY — span > 180天（供需失衡的长期堆积）
    POLICY     — 30 ≤ span ≤ 180 ∧ gap_CV < 0.5（政策驱动，间隔均匀）
    UNKNOWN    — 不满足以上条件

  同一价位、不同反差类型的 Zone 不归为同一 Structure。
```

**当前状态**：✅ 已实现（zones.py `_infer_contrast` + `detect_zones`）

## 机制二：层级

**语法**：聚簇的结果作为新单位进入更高层组织。

**价格映射**：

```
DEF 2.1 层级链

  Point（最小可区分单位）
    → first_diff(p_i, p_{i+1}) = x_{i+1} - x_i
    → 对数差分：log(x_{i+1}) - log(x_i)（消除量纲漂移）

  Segment（两点间连线）
    → 方向、速率、持续时间

  Zone（聚簇）
    → 界面：调节价格与区的交互

  Cycle（进出配对）
    → 自维持检验

  Structure（Zone + Cycles + Phases + Motion + Projection）
    → 前主体态

  Bundle（共享生成约束的 Structure 集合）
    → 跨结构关系

  每一层是前一层的凝聚态，不是简单的容器嵌套。
```

**当前状态**：✅ 已实现（models.py 全部对象）

## 机制三：守恒

**语法**：差异不能被无代价清零，只能在不同形态之间转化。

**投影层注意**：
  价格 = Π(现实差异)，Π 有损、不可逆、非线性、时变。
  系统在影子层工作，不能从影子反推实体。
  守恒在影子层的正确表述不是"差异在社会差异之间转化"，
  而是：当价格结构的某些特征被压缩时，
  另一些特征会补偿性变化。

**价格层守恒（System = Structure × Motion）**：

```
DEF 3.1 影子层守恒

  不从理论做直接对应，只在影子层观察：
    当 Structure 的某些不变量被压缩时，
    Structure 的另一些不变量是否在补偿性变化。

  可观测的价格特征：
    ① Zone 相对带宽（价格差异的密度）
    ② 速度比（进出节奏的不对称性）
    ③ 时间比（进出耗时的不对称性）
    ④ 试探密度（Cycle 在时间轴上的密集度）
    ⑤ 振幅趋势（Cycle 幅度的变化方向）

  守恒检测：
    如果 ① 被压缩（带宽收窄），
    而 ②③④⑤ 中有补偿性变化，
    → 结构正在重新分配差异，不是真正变平

  如果 ① 被压缩，且 ②③④⑤ 都没有补偿性变化，
    → 可能真正稳定，也可能差异转到了影子层看不到的地方

  守恒通量 flux_score ∈ [-1, 1]：
    正 = 价格差异在释放
    负 = 价格差异在压缩
    绝对值 > 0.3 = 结构正在重新分配差异

  关键区别：
    系统不说"差异从价格转到了时间差异维度"（这是从影子反推实体）
    系统说"Zone 带宽收窄 + 速度比在扩张"（这是影子层的观测）
    理论在后台解释为什么这种模式可能存在
    系统只输出模式 + 历史后验
```

**当前状态**：✅ 已实现（check_conservation 检测价格特征的补偿性变化）

## 机制四：先天完备性

**语法**：在破缺发生前，所有候选路径逻辑平等地并存。

**价格映射**：

```
DEF 4.1 候选结构并存

  给定一个编译结果 CompileResult，
  其中 structures = [S_1, S_2, ..., S_n]

  每个 S_i 都是一个候选结构——
  它们在破缺发生前具有逻辑平等的地位。
  系统不预判哪个会"赢"。

  后验统计（forward_outcome）记录破缺后哪个路径被选定：
    ret_5d, ret_10d, ret_20d, max_dd, max_rise

  系统不预测，只检索历史相似 + 输出后验分布。
```

**当前状态**：✅ 已实现（sample/store.py + retrieval/engine.py）

## 机制五：最小变易

**语法**：变化通过局部、离散、可传递的单步推进。系统先滑向最近稳态，不是最优。

**价格映射**：

```
DEF 5.1 最小变易 = 极值点提取

  极值点是价格路径上的离散事件点。
  段生成是事件间的局部连线。
  编译器只处理离散事件，不处理连续过程。

DEF 5.2 最近稳态（NearestStableState）

  给定 Cycle c = (entry, exit, zone)
  exit 之后，价格最先停驻的位置。

  多信号评分：
    ① Zone 触及（权重 0.3）— 价格到达已知支撑/阻力区
    ② 减速（权重 0.4）— 连续两段幅度递减
    ③ 波动率收缩（权重 0.2）— 近期幅度 CV 下降

  阻力评分 ∈ [0, 1]：越低 = 越容易到达 = 越"最近"
  系统先找最近稳态，不是最优稳态。
```

**当前状态**：✅ 已实现（pivots.py 自适应窗口 + cycles.py `_find_nearest_stable` 多信号评分）

## 机制六：破缺

**语法**：候选路径失去平衡，某一条获得现实优先权。

**价格映射**：

```
DEF 6.1 结构阶段转换

  FORMATION → CONFIRMATION：
    avg_speed_ratio 的 CV < 0.5（速度比稳定）
    结构的进出节奏开始自我维持
    不是外部强加的稳定性，而是结构内部节奏的一致性

  * → BREAKDOWN：
    最后一个 exit.end 距离 zone > bandwidth × 2
    价格越过结构边界，阈值被突破

  * → INVERSION：
    前半段 avg(speed_ratio) 和后半段符号相反
    结构内部方向断裂

  破缺 = 从多种候选结构中，某一条路径被市场选定的时刻。
```

**当前状态**：✅ 已实现（cycles.py `_infer_phases`）

## 机制七：循环

**语法**：被选中的路径通过反复返回关键差异关系来维持自身。

**价格映射**：

```
DEF 7.1 自维持

  Cycle 不是简单的进出配对。
  一个 Cycle 只有在开放市场波动中仍能保持进出配对，才叫自维持。

  自维持检验：
    - entry.end 触及 zone
    - entry.start 不在 zone 内
    - exit.end 离开 zone
    - exit 方向与 entry 反向

  如果 Cycle 只能在封闭条件中存在（没有后续波动考验），
  它只是闭合，不是自维持。

DEF 7.2 结构复制（带偏差）

  同一 Zone 反复生成的 Structure 不是原样复制。
  关键关系（speed_ratio, time_ratio）被再次构造，
  但每次都有偏差。

  偏差通过 invariants 追踪：
    avg_speed_ratio, avg_time_ratio, high_dispersion, ...
  偏差通过相似度检索比较：
    DTW 相似度捕捉"形状相似但时间错位"的复制
    段形状编辑距离捕捉"节奏相似"的复制
```

**当前状态**：✅ Cycle 组装已实现（cycles.py）
           ✅ DTW + 编辑距离已实现（similarity.py）
           ⚠️ 跨周期复制追踪待做

## 机制八：锁定

**语法**：循环持续后压缩替代路径，形成路径依赖。

**价格映射**：

```
DEF 8.1 结构锁定

  结构的 invariants = 路径偏置的记录：
    - avg_speed_ratio：速度比的稳定值
    - avg_time_ratio：时间比的稳定值
    - zone_strength：Zone 被试探的累积强度
    - stable_state_ratio：已识别稳态的 cycle 占比

  当这些 invariants 跨多个 Cycle 保持一致时，
  结构进入锁定——替代路径的成本越来越高。

  锁定的结构更难被打破，
  但一旦打破（BREAKDOWN），后果更剧烈。
```

**当前状态**：✅ 已实现（structure_invariants）

## 机制九：自指

**语法**：系统把自身纳入自身的运作中。

**价格映射**：

```
DEF 9.1 反身性

  系统输出结构匹配 → 研究者据此行动 → 市场受影响 → 结构变化

  反身性追踪（ReflexivityTracker）：
    记录每次规则匹配 → 回填后续结果 → 检测准确率衰减

  当规则准确率在近 window 次匹配中低于历史平均 - 2σ
  → 标记为"可能正在经历反身性衰减"

  系统知道自己在看影子（投影觉知），
  也知道自己在改变影子（反身性）。
  这两层觉知加起来 = 自指。

DEF 9.2 投影觉知 = 系统对自身认知边界的觉知

  compression_level：投影压缩度 [0,1]
  blind_channels：差异可能藏在的通道
  recommended_actions：系统建议研究者做什么
  blind_evidence：每个盲区通道的具体证据

  高压缩时系统说"我可能在看假象"——这是自指的第一层。
  反身性追踪说"我的输出在改变对象"——这是自指的第二层。
```

**当前状态**：✅ 投影觉知已实现（ProjectionAwareness）
           ⚠️ 反身性追踪骨架完成（reflexivity.py），待完整实现
