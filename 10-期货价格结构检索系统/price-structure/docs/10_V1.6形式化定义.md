> 这是理论与代码之间的**中间层**。
> 每一条定义先有理论出处，再有系统级形式化，最后才进代码。

## 第一章 差异的本体地位（Ch1）

### 1.1 差异是成立条件

**理论（定义 1.1）**：
> 差异不是事物的附属特征，而是事物得以被成立、被区分、被识别的前提。

**系统定义**：
```
DEF D1.1 最小可区分单位（Point）
  给定价格序列 P = {p_0, p_1, ..., p_n}
  最小可区分单位 = 一对相邻极值点 (p_i, p_{i+1}) 之间的关系
  不是单点本身，而是点间差分 first_diff(p_i, p_{i+1})
  
  形式化：
    Point = (t, x, idx)          -- 点本身不含结构信息
    差异算子 = Δx = x_{i+1} - x_i -- 差异在算子中，不在点中
    对数差异算子 = log(x_{i+1}) - log(x_i) -- 消除量纲漂移
```

**检验**：如果系统只输出单点坐标而不输出点间关系，则违反 D1.1。

### 1.2 无差异则无世界

**理论（命题 1.1）**：
> 无差异则无区分，无区分则无世界。

**系统定义**：
```
DEF D1.2 结构存在的最小条件
  一个 Structure 至少需要：
    - 2 个 Point（构成最小差异）
    - 1 个 Segment（差异被组织为一段）
    - 段内包含可度量的方向、速率、持续时间
  
  如果一个"结构"只有一个点，它在系统中不存在。
```

## 第二章 聚簇与结构的形成（Ch2）

### 2.1 聚簇不是堆在一起

**理论（定义 2.1）**：
> 聚簇不是差异的简单并置，而是差异在某种共同约束下发生稳定关联，从而形成更高层级结构的过程。

**系统定义**：
```
DEF D2.1 聚簇判定条件
  给定一组极值点 C = {p_1, ..., p_k}，C 构成聚簇当且仅当：
    1. 价格邻近性：max(|x_i - x_j|) / mean(C) < ε  （可识别性）
    2. 时间跨度约束：span(C) = t_last - t_first > 0  （非瞬时）
    3. 强度阈值：|C| ≥ min_points  （最小密度）
  
  不满足 → 只是堆积，不生成 Zone
```

### 2.2 共同反差

**理论（定义 2.2）**：
> 共同反差，是多个差异单元共同面对的外部差异，它为聚簇提供方向、压力和边界。

**系统定义**：
```
DEF D2.2 共同反差（ContrastType）
  反差 = 驱动极值点在同一价位反复出现的外部压力类型
  
  判定依据（极值点时序特征 → 反差类型）：
  ┌─────────────┬────────────────────────────────────┐
  │ PANIC       │ span < 30天 ∧ |C| ≥ 3              │
  │             │ 恐慌性密集试探，短时间内反复冲击     │
  ├─────────────┼────────────────────────────────────┤
  │ OVERSUPPLY  │ span > 180天                        │
  │             │ 供需失衡导致的长期价格底部堆积       │
  ├─────────────┼────────────────────────────────────┤
  │ POLICY      │ 30 ≤ span ≤ 180 ∧ |C| ≥ 3         │
  │             │ ∧ gap_CV < 0.5                      │
  │             │ 政策驱动，试探间隔均匀               │
  ├─────────────┼────────────────────────────────────┤
  │ LIQUIDITY   │ 待实现（需 volume 数据）            │
  ├─────────────┼────────────────────────────────────┤
  │ SPECULATION │ 待实现（需 volume + OI 数据）       │
  ├─────────────┼────────────────────────────────────┤
  │ UNKNOWN     │ 不满足以上任何条件                  │
  └─────────────┴────────────────────────────────────┘
  
  结构影响：
    同一价位、不同反差类型的 Zone 不应归为同一 Structure
    反差类型进入 Structure.invariants 用于相似性检索过滤
```

### 2.3 可叙事性

**理论（命题 2.3）**：
> 能够形成结构的差异，通常同时具备可识别性、可利用性和可叙事性。

**系统定义**：
```
DEF D2.3 叙事标签（narrative_context）
  每个 Structure 必须携带一个人可读的叙事标签：
    narrative_context = f(contrast_type, speed_ratio, cycle_count)
  
  生成规则：
    [反差类型描述] · [速度特征] · [试探频率]
    
    示例：
    "恐慌性结构 · 急跌型 · 4次密集试探"
    "供需失衡结构 · 均衡型 · 3次试探"
    "未分类结构"
  
  用途：
    - 人类研究者快速理解结构性质
    - 检索结果展示
    - 不进入数值计算，纯描述性
```

## 第三章 时间、事件与结构断裂（Ch3）

### 3.1 事件是可能性的关门

**理论（定义 3.1）**：
> 事件，是使原有可能性结构发生不可忽略变化的离散性发生。

**系统定义**：
```
DEF D3.1 价格事件
  事件 = 一个极值点或段转换，使得后续路径条件明显改变
  
  形式化：
    给定 Segment s_i = (p_i, p_{i+1})
    s_i 是事件段 ⟺ |log_rate(s_i)| > θ_event
                 ∧ 方向与前一段相反
                 ∧ 后续结构模式与前段不同
  
  事件 vs 过程：
    极值点提取 = 找事件点
    段生成 = 连接事件点
    编译器只处理事件，不处理连续过程
```

### 3.2 最近稳态

**理论（命题 3.4）**：
> 社会变化通常先沿阻力最小的方向发生，并首先停驻在一个最近的、能够暂时稳住局面的平衡点上。

**理论（命题 3.5）**：
> 最近稳态的意义不在于它最优，而在于它最先可用。

**系统定义**：
```
DEF D3.2 最近稳态（NearestStableState）
  给定 Cycle c = (entry, exit, zone)
  最近稳态 = exit 段之后，价格最先停驻的位置
  
  形式化：
    NearestStableState = {
      zone: Zone | None,           -- 稳态价位区
      arrival_point: Point,         -- 到达点
      duration_to_arrive: float,    -- 从 exit.end 到到达点的天数
      resistance_level: float,      -- 阻力评分 ∈ [0, 1]
    }
  
  判定规则（按优先级）：
    规则1（触及已知 zone）：
      扫描 exit 后 k 段 (k ≤ max_lookahead=8)
      若某段终点 p 满足 touches_zone(p, z) 对某个已知 z
      → nearest_stable = (z, p, duration, k/max_lookahead)
    
    规则2（减速判定）：
      若连续两段 s_{j-1}, s_j 满足 |Δ(s_j)| < 0.5 · |Δ(s_{j-1})|
      → 价格在减速，构造临时 Zone 作为稳态候选
      nearest_stable = (Zone(s_j.end), s_j.end, duration, j/max_lookahead)
    
    规则3（无稳态）：
      扫描完 max_lookahead 段仍未命中 → next_stable = None
  
  阻力评分语义：
    resistance_level = 到达稳态所需段数 / max_lookahead
    越低 = 越容易到达 = 越可能是"最近"的稳态
    越高 = 越远 = 越可能是被迫到达的
  
  检验标准：
    - 如果一个 Cycle 的 next_stable = None，不一定说明没有稳态，
      只说明在 max_lookahead 范围内没找到
    - resistance_level < 0.2 的稳态需要额外检查（可能是假稳态）
```

## 第四章 守恒、转移与最近稳态（Ch4）

### 4.1 守恒

**理论（定义 4.1）**：
> 社会守恒，不是差异数值的严格等量，而是差异不可能被无代价清零的结构性约束。

**系统定义**：
```
DEF D4.1 价格守恒
  守恒 = 价格差异不能凭空消失，只能转移
  
  形式化：
    给定维度集合 D = {price_daily, price_5min, volume, OI}
    若 diff(daily) 下降 → ∃ d' ∈ D 使得 diff(d') 上升
  
  系统级守恒检查：
    check_conservation(structure, bars) → {
      conservation_violated: bool,    -- 是否检测到守恒可能被违反
      transfer_channels: List[str],   -- 差异可能转移到的通道
      notes: List[str],              -- 人可读说明
    }
  
  当前可检测项：
    ① 速度比变化显著（前半 vs 后半差异 > 30%）
       → 差异可能在结构内部转移
    ② Zone 相对带宽极窄（< 0.01）
       → 差异可能转移到短周期或成交量维度
    ③ 最近稳态阻力异常低（< 0.2）
       → 可能是假稳态，差异在隐性积累
  
  待实现（需多维度数据）：
    ④ 日线波动率下降时，检查 5 分钟线波动率是否上升
    ⑤ 持仓量变化与价格变化的背离
    ⑥ 成交量变异系数的异常放大
```

### 4.2 转移

**理论（定义 4.2）**：
> 转移，是差异在原有通道受阻后，沿着其他较低阻力路径继续存在和积累的过程。

**理论（命题 4.3）**：
> 差异转移通常沿着低阻通道发生。

**系统定义**：
```
DEF D4.2 低阻通道
  低阻通道 = 差异更容易转移去的维度
  
  价格系统中的低阻通道排序（阻力由低到高）：
    1. shorter_timeframe  -- 转到更短周期（最容易）
    2. volume             -- 转到成交量异动
    3. open_interest      -- 转到持仓变化
    4. cross_symbol       -- 转到相关品种
    5. term_structure     -- 转到期现价差
  
  判定逻辑：
    当主维度（日线价格）的差异被压缩时，
    按通道阻力从低到高依次检查：
    → 找到第一个差异明显上升的通道 = 转移目标
  
  形式化：
    transfer_path = argmin_{d ∈ D \ {source}} resistance(d)
    使得 diff(d, window) > diff(d, prev_window) × threshold
```

### 4.3 最近稳态在守恒中的角色

**理论（命题 4.4）**：
> 差异转移之后，系统通常不会直接抵达最优状态，而会优先收敛到最近的可稳住安排。

**系统定义**：
```
DEF D4.3 守恒-转移-稳态 完整链
  
  完整逻辑链：
    1. diff(zone) 被压缩
    2. diff → 转移到低阻通道 d'
    3. 系统滑向 nearest_stable（不是最优，是最先可用）
    4. nearest_stable 可能是临时的 → 后续继续失衡
  
  系统级追踪：
    Structure 守恒链 = {
      source: "zone_bandwidth_compressed",
      transfer_to: ["shorter_timeframe", "volume"],
      stable_state: NearestStableState,
      is_temporary: bool,  -- 基于 resistance_level 判断
    }
```

## 第五章 主体、边界与差异的自我维持（Ch5）

### 5.1 主体不是起点

**理论（命题 5.1）**：
> 主体不是差异之外的起点，而是差异被稳定组织之后形成的结果。

**系统定义**：
```
DEF D5.1 市场主体
  在价格系统中，"主体"不是预设的，而是从结构中涌现的
  
  形式化：
    一个 Structure 之所以被识别为"这个结构"，
    不是因为它预先存在，而是因为：
    - 它的 Zone 维持了价格边界
    - 它的 Cycle 形成了可重复的进入/退出模式
    - 它的 invariants 在检索中被反复匹配
    
    主体 = 被系统反复识别出来的稳定结构
  
  这意味着：
    同一价格区间在不同时期可能被识别为不同的结构主体
    结构的身份由其当前的 cycle 组织方式决定，不由历史决定
```

### 5.2 边界维持

**理论（命题 5.2）**：
> 边界不是主体之外的附属物，而是主体得以持续存在的条件之一。

**系统定义**：
```
DEF D5.2 Zone 边界强度
  Zone 的存在 = 它的价格边界被反复试探并维持
  
  形式化：
    zone_strength = Σ decay^i × compactness
    其中 compactness = 1 / (1 + price_range / center)
  
  边界被打破 = Zone 失效
    当价格持续突破 zone.upper 或 zone.lower 超过 bandwidth × 2
    → 该 Zone 对应的结构主体进入 BREAKDOWN 阶段
```

## 第六章 社会差异的三种基本形态（Ch6）

### 6.1 三种差异在价格中的映射

**理论（Ch6 三种基本形态）**：
> 权力差异、财富差异、身份差异是社会世界中最基本的三种差异形态。

**系统定义**：
```
DEF D6.1 价格中的三种差异分层

  ┌──────────────────────────────────────────────────────┐
  │ 社会差异          │ 价格映射          │ 数据来源     │
  ├───────────────────┼───────────────────┼──────────────┤
  │ 权力差异          │ 时间差异          │              │
  │ (谁能定义规则)    │ (谁能等得起)      │ OI, 换手率   │
  │                   │ 持仓结构 = 时间    │ 期限结构     │
  │                   │ 耐受力的代理变量   │              │
  ├───────────────────┼───────────────────┼──────────────┤
  │ 财富差异          │ 流动性差异        │              │
  │ (谁能拿到什么)    │ (能不能变成钱)    │ volume       │
  │                   │ 成交量 = 财富     │ 买卖盘深度    │
  │                   │ 流动性的代理变量   │              │
  ├───────────────────┼───────────────────┼──────────────┤
  │ 身份差异          │ 边界恐惧          │              │
  │ (谁算自己人)      │ (怕被踢出局)      │ 价格跳空     │
  │                   │ 止损线 = 存在     │ 波动率突变   │
  │                   │ 边界的代理变量     │ 持仓集中度   │
  └──────────────────────────────────────────────────────┘

  当前代码状态：只显影了价格维度（几何分析）
  P1 目标：至少引入 volume 维度，让流动性差异进入编译器
```

### 6.2 时间差异

**系统定义**：
```
DEF D6.2 时间差异度量
  时间差异 = 谁能等、谁不能等
  
  形式化：
    time_compression_ratio = avg_entry_duration / avg_exit_duration
    
    若 time_compression_ratio > 1.5：
      → 入场慢、出场快 = 恐慌性退出 = 时间差异剧烈
    若 time_compression_ratio < 0.67：
      → 入场快、出场慢 = 缓慢消化 = 时间差异平缓
  
  数据需求：已有（entry/exit duration 已计算）
  实现状态：Cycle.time_ratio 已覆盖，但未显式标记为"时间差异"
```

### 6.3 流动性差异

**系统定义**：
```
DEF D6.3 流动性差异度量
  流动性差异 = 能不能把价格变成钱
  
  形式化：
    liquidity_stress = volume_cv_in_zone / volume_cv_outside_zone
    
    若 liquidity_stress < 0.5：
      → Zone 内成交量萎缩 = 流动性枯竭 = 差异被压缩
    若 liquidity_stress > 2.0：
      → Zone 内成交量放大 = 多空激烈交锋 = 差异在释放
  
  数据需求：volume（已有但未被编译器使用）
  实现状态：❌ 未实现
  P1 工程项：
    1. extract_pivots() 增加 volume_weighted 模式
    2. Zone 增加 volume_profile 相关字段
    3. Structure 增加 liquidity_stress 指标
```

### 6.4 边界恐惧

**系统定义**：
```
DEF D6.4 边界恐惧度量
  边界恐惧 = 怕被踢出局的紧张程度
  
  形式化：
    fear_index = 
      w1 × price_gap_frequency     -- 跳空频率
      + w2 × volatility_spike      -- 波动率突变
      + w3 × zone_touch_intensity  -- 试探密集度
    
    其中 zone_touch_intensity = |touches| / span_days
  
  数据需求：价格（已有）、可能需要 OI
  实现状态：❌ 未实现
  P1 工程项：
    1. Structure 增加 fear_index 计算
    2. 将 fear_index 纳入 invariants 用于检索
```

## 第七章 社会差异守恒与转化（Ch7）

### 7.1 转化是守恒的常态

**理论（命题 7.1）**：
> 转化不是守恒的例外，而是守恒在社会世界中的常态形式。

**系统定义**：
```
DEF D7.1 差异转化追踪
  
  价格系统中的四条高频转化通道：
  
  通道1：显性 → 隐性
    日线波动率下降 → 5 分钟线波动率上升
    检测：compare_volatility(daily_bars, m5_bars, window)
  
  通道2：价格 → 成交量
    价格波动收敛 → 成交量变异放大
    检测：price_volume_divergence(structure, bars)
  
  通道3：波动率 → 持仓
    价格平稳 → 持仓量异常变化
    检测：price_oi_divergence(structure, bars_with_oi)
  
  通道4：当前品种 → 相关品种
    主力合约波动收敛 → 近月/远月合约波动放大
    检测：cross_symbol_transfer(main_symbol, related_symbols)
  
  形式化：
    TransferEvent = {
      source_dim: str,
      target_dim: str,
      source_diff_before: float,
      source_diff_after: float,
      target_diff_before: float,
      target_diff_after: float,
      transfer_strength: float,  # (target_after - target_before) / 
                                 # (source_before - source_after)
    }
```

### 7.2 时间差制造错觉

**理论（命题 7.4）**：
> 社会守恒最常通过时间差制造错觉——旧差异先被看见地缩小，新差异后被延迟地显现。

**系统定义**：
```
DEF D7.2 错觉检测
  
  系统不应在波动率下降时立即报告"结构稳定"
  而应标记为"表面稳定，待验证"
  
  形式化：
    stability_verdict = {
      surface: "stable" | "unstable",
      verified: bool,
      verification_window: int,  -- 需要等待多少根 bar 才能确认
      pending_channels: List[str],  -- 待检查的转移通道
    }
  
  规则：
    IF zone.relative_bandwidth < threshold
    THEN stability_verdict.surface = "stable"
         stability_verdict.verified = false
         stability_verdict.verification_window = 20  -- 20 天后复查
         stability_verdict.pending_channels = ["5min", "volume"]
```

## 第八章 变迁的离散事件结构（Ch8）

### 8.1 过程不等于结构变化

**理论（命题 8.1）**：
> 过程会改变条件，但条件本身并不自动等于结构变化。

**系统定义**：
```
DEF D8.1 编译器的离散性原则
  
  编译器只处理离散事件，不处理连续过程：
    - 极值点 = 事件点（摆动触发）
    - 段 = 事件之间的连线
    - Zone = 事件反复发生的价位
    - Cycle = 进出事件的配对
  
  连续过程（均线、趋势线、通道）不是编译器的处理对象
  它们是"条件"，不是"结构"
  
  这意味着：
    编译器不会输出"趋势向上"这样的连续判断
    它只会输出"在 Zone z 处形成了 n 个 Cycle，
    速度比为 sr，最近稳态在 ns"
```

### 8.2 阈值触发

**理论**：
> 很多结构不是一点点坏一点点崩，它们常常在某个范围内还能维持，一旦越过某条线，性质就变了。

**系统定义**：
```
DEF D8.2 结构阶段转换阈值
  
  Phase 转换规则（已有，形式化补充）：
  
  FORMATION → CONFIRMATION：
    条件：avg_speed_ratio 的 CV < 0.5（速度比稳定）
    含义：结构开始自我维持
  
  * → BREAKDOWN：
    条件：最后一个 exit.end 距离 zone > bandwidth × 2
    含义：价格越过了结构边界，阈值被突破
  
  * → INVERSION：
    条件：前半段 avg(speed_ratio) 和后半段符号相反
    含义：结构内部方向发生了断裂
```
