  - 段方向严格交替 ✅
  - Cycle 入口/出口边界正确 ✅

### [1] → [2] 编译 → 结构态

```
状态: ✅ 完整

新增的三层全部计算:
  ✅ motion = compute_motion(st)       → phase_tendency, flux, stable
  ✅ projection = compute_projection(st) → compression, blind_channels
  ✅ conservation = check_conservation(st) → transfer_channels
  ✅ narrative = infer_narrative_context(st) → 人可读标签

输出示例:
  S0: zone=98755 cycles=6 contrast=unknown
      motion=→breakdown flux=+0.50 proj=40%
```

### [2] → [3] 结构态 → 规则引擎

```
状态: ⚠️ 部分连接

已连接:
  ✅ 规则匹配使用 invariants 中的 avg_speed_ratio, avg_time_ratio 等
  ✅ 匹配后 label 回写到 Structure.label
  ✅ 典型度回写到 Structure.typicality

未连接:
  ⚠️ 规则引擎不使用 motion 字段
     → 规则无法表达"速度比正在加速上升"这样的动态条件
     → 无法区分"已确认的 breakdown"和"正在走向 breakdown"
  
  ⚠️ 规则引擎不使用 projection 字段
     → 无法过滤高压缩度的"假信号"
     → 无法表达"只在可信度>70%时匹配"
  
  ⚠️ 规则引擎不使用 zone.context_contrast
     → 无法表达"只匹配恐慌型底部"或"排除政策驱动型"

影响: 规则系统只做静态匹配，丢失了运动态信息
```

### [3] → [4] 规则 → 相似性检索

```
状态: ⚠️ 部分连接

已连接:
  ✅ label 传播到 family_similarity (相同label=1.0, 镜像=0.6, 不同=0.0)
  ✅ relational_similarity 使用 invariants keys
  ✅ geometric_similarity 使用结构几何特征

未连接:
  ⚠️ 相似性不考虑 motion 相似度
     → 两个结构几何相似但运动方向相反（一个→breakdown，一个→confirmation）
     → 它们会被判为高相似，但后验行为可能完全不同
  
  ⚠️ 相似性不考虑 contrast_type
     → 恐慌型底部 vs 过剩型底部，几何可能相似
     → 但驱动逻辑完全不同，后验应该不同

  ⚠️ 相似性不考虑 projection 置信度
     → 高压缩结构的相似性结果应该打折
```

### [4] → [5] 相似性 → 样本库/后验

```
状态: ⚠️ 部分连接

已连接:
  ✅ library.jsonl 有 46 条样本
  ✅ full_structure_pool 有 1221 条结构记录
  ✅ active_match.py 实现了带观点检索
  
未连接:
  ⚠️ daily_pipeline 不自动调用样本库检索
     → 每日扫描只用规则引擎，不查历史相似案例
     → 需要手动调用 active_match
  
  ⚠️ 后验统计从 sample.forward_outcome 中提取
     → 但 daily_scan 没有自动聚合后验分布
     → 用户看不到"历史上类似结构之后发生了什么"
  
  ⚠️ 最近稳态分析完全缺失
     → active_match 的 comparison_guide 没有分析"崩塌后先到哪"
     → 这是 V1.6 最核心的方法论支点，但流程中没有体现
```

### [5] → [6] 后验 → 输出

```
状态: ❌ 断裂

问题:
  ❌ daily_scan.py 的输出没有包含运动态和投影觉知
     → 只输出 zone, cycles, label, typicality
     → 用户看不到"在怎么动"和"可信度"
  
  ❌ daily_report 没有最近稳态分析
     → 只说"历史上类似结构涨了 X%"
     → 不说"如果崩塌，最先停在 Y 价位"
  
  ❌ comparison_guide 是静态模板
     → 不根据当前结构的 motion/projection 调整建议
     → 高压缩结构和低压缩结构给出相同建议
```

### [6] → [7] 输出 → active_match

```
状态: ✅ 基本完整

active_match.py 实现了:

  ✅ comparison_guide 生成对比建议
  ✅ 输出 HTML 报告

未连接:
  ⚠️ comparison_guide 没有使用 motion 和 projection
  ⚠️ 没有"最近稳态"分析环节
```

## 断裂点汇总

| # | 断裂位置 | 严重度 | 影响 | 修复方向 |
|---|---------|--------|------|---------|
| 1 | 规则引擎不使用 motion/projection | 🟡 P1 | 规则只能做静态匹配 | DSL 增加 motion 条件 |
| 2 | 相似性不考虑 motion/contrast | 🟡 P1 | 几何相似≠行为相似 | similarity 加 motion 维度 |
| 3 | daily_scan 不连接样本库 | 🟡 P1 | 每日不输出后验分析 | pipeline 集成检索 |
| 4 | 输出缺少运动态/投影觉知 | 🟡 P1 | 用户看不到动态信息 | daily_report 更新 |
| 5 | comparison_guide 缺最近稳态 | 🔴 P0 | 核心方法论缺失 | 增加稳态分析 |
| 6 | comparison_guide 静态模板 | 🟢 P2 | 建议不随结构变化 | 增加动态调整 |
| 7 | 稳态验证窗口缺失 | 🟢 P2 | "稳定"可能是假象 | 增加20天复查 |

## v3.0 流水线扩展 (2026-04-24)

v2.5 的流水线在结构编译后结束。v3.0 新增了质量分层、共振检测、生命周期追踪和交易信号四个后处理环节。

### 扩展流程图

```
[0] 行情数据        [1] 编译器          [2] 结构态          [3] 质量分层
  Bar[] ───→ compile_full() ───→ Structure[] ───→ stratify()
  Parquet      pivots→segs→zones    zone,cycles       ├─ A层: 检索+统计
  65品种       →cycles→structures   motion,proj       ├─ B层: 降权检索
                                                   ├─ C层: 仅参考
                                                   └─ D层: 丢弃
                            │
                            ▼
[4] 共振检测          [5] 生命周期          [6] 日内节奏        [7] 交易信号
  ResonanceDetector    LifecycleTracker    IntradayRhythm    PythonGO策略
  板块分组→共振评分    每日快照→JSONL       5min时段对比       on_bar→信号→提示
  同板块≥2品种→共振    Zone匹配→轨迹        开盘/盘中/收盘     假突破反转
  同板块≥3品种→强共振  质量趋势追踪          最佳时段检测       跨品种共振
```

### 新增数据流

| 环节 | 输入 | 输出 | 文件 |
|------|------|------|------|
| 质量分层 | Structure[], SystemState[] | StratificationResult | `src/quality.py` |
| 共振检测 | {symbol: (structures, states)} | ResonanceResult | `src/resonance.py` |
| 生命周期 | Structure[], date | LifecycleRecord[] | `src/lifecycle.py` |
| 日内节奏 | 5min bars, structures | RhythmReport | `src/intraday_rhythm.py` |
| 交易信号 | on_bar(KLineData) | 信号提示 | `trading/pythongo_signal.py` |
