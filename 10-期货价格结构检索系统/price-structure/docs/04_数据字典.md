# 数据字典

## Bar（K线）

核心字段，从 MySQL 读取后统一为此格式。

```
Bar {
    symbol:        str          # 品种代码，如 "CU000"（铜连续）
    timestamp:     datetime     # Bar 时间戳（UTC 或约定时区）
    open:          float        # 开盘价
    high:          float        # 最高价
    low:           float        # 最低价
    close:         float        # 收盘价
    volume:        float        # 成交量
    open_interest: float?       # 持仓量（可选）
    adj_factor:    float?       # 复权因子（可选）
}
```

## MySQL 表结构（兼容约定）

系统不强制要求原始表结构，通过 `config.yaml` 中的映射配置适配：

```yaml
data:
  source: mysql
  connection:
    host: localhost
    port: 3306
    database: futures
    user: ${MYSQL_USER}
    password: ${MYSQL_PASSWORD}
  tables:
    daily_bars:
      table_name: cu_continuous
      column_mapping:
        symbol: instrument
        timestamp: trade_date
        open: open_price
        high: high_price
        low: low_price
        close: close_price
        volume: vol
        open_interest: oi
```

## 数据清洗规则

1. **缺失值**: 连续缺失 > 3 天标记警告，不自动插值
2. **异常值**: 单日涨跌幅 > 15% 标记审查（可配置阈值）
3. **时间对齐**: 确保交易日连续，非交易日跳过
4. **复权**: 连续合约使用复权因子统一价格序列
5. **类型校验**: 所有数值字段强制 float，时间字段强制 datetime

## 存储格式

- **原始数据**: 保留在 MySQL，通过 DataLoader 读取
- **缓存**: Parquet 格式本地缓存，DuckDB 查询
- **中间对象**: Python dataclass，运行时内存

## 数据质量报告

每次加载数据时生成：
- 总 Bar 数 / 时间跨度
- 缺失率
- 异常标记数
- 量仓数据完整性

### LocalStore 接口

```python
from src.data.local_store import LocalStore, open_store

store = open_store()
store.save_bars("CU0", bars, freq="1d")       # 写入 Parquet
bars = store.load_bars("CU0", freq="1d")       # 读取
```
