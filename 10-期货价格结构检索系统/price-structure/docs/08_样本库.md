# 样本库

本文档定义样本库的样本结构、标注流程、质量标准和存储方式。

## 样本定义

一个样本 = 一个被规则引擎命中（或人工确认）的结构 + 标签 + 前向演化结果。

## Sample Schema

| 字段 | 类型 | 说明 |
|------|------|------|
| id | str | 唯一标识（例: CU_0001） |
| symbol | str | 品种代码 |
| t_start | datetime | 结构起始时间 |
| t_end | datetime | 结构结束时间 |
| structure | dict | Structure.to_dict() 完整序列化 |
| label_type | str | 结构类型（规则名） |
| label_phase | str | 生命周期阶段 |
| typicality | float | 典型度 [0, 1] |
| annotation | str | 人工备注 |
| forward_outcome | dict | 前向演化结果 |

## 前向演化标签

| 字段 | 说明 |
|------|------|
| ret_5d | 结构结束后 5 日收益 |
| ret_10d | 10 日收益 |
| ret_20d | 20 日收益 |
| max_rise_20d | 20 日内最大涨幅 |
| max_dd_20d | 20 日内最大回撤 |
| regime_tag | 市场状态标签（预留） |

## 标注流程

```
规则引擎扫描 → 候选结构 → 自动标注 → 人工复核 → 写入样本库
```

规则引擎扫描所有结构，自动标注并写入样本库。人工复核对典型度进行调整，并记录复核原因。

## 质量标准

- 金标准案例：20-50 个人工挑选的典型样本
- 规则扫出样本：人工复核后使用
- 目标：≥ 100 个高质量案例，人机判断一致率 > 80%

## 存储

JSONL（每行一个 JSON 对象），位于 `data/samples/library.jsonl`。支持增量追加，不支持删除（需要时重新生成）。
