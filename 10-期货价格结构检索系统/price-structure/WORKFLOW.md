# 价格结构检索系统 — 作业流程

> 标准化的工作循环，每次迭代都按此流程执行。

---

## 流程总览

```
理论 → 文本 → 工程设计 → 代码实现 → 测试 → 复核 → 去重去冗余 → 优化 → 推送 → 新建议
```

---

## 各阶段详细说明

### 1. 理论
**输入**：《差异即世界》九机制 / 研究直觉 / 市场观察
**输出**：明确的理论命题或假设
**检查点**：
- [ ] 命题是否可证伪？
- [ ] 是否对应九机制之一？
- [ ] 是否与现有理论冲突？

### 2. 文本
**输入**：理论命题
**输出**：`下一步有趣的想法.md` 中的新条目
**格式**：
```markdown
### N. 标题
**对应机制**：XXX
- 核心思路
- 已具备条件
- 实现量：~XXX 行
```
**检查点**：
- [ ] 优先级标注（P0/P1/P2）
- [ ] 与现有功能的关系明确
- [ ] 实现可行性评估

### 3. 工程设计
**输入**：文本方案
**输出**：模块设计文档（可内联在代码文件头部）
**内容**：
- 模块职责、接口定义
- 与现有代码的关系
- 数据结构设计
- 性能考量
**检查点**：
- [ ] 接口是否与现有模块兼容？
- [ ] 是否引入了不必要的依赖？
- [ ] 是否有向后兼容性保证？

### 4. 代码实现
**输入**：工程设计
**输出**：`.py` / `.c` 源文件
**规范**：
- 文件头必须有 docstring 说明模块职责
- 所有公共函数有类型注解
- 关键逻辑有注释
- 不修改现有文件（独立模块化）
**检查点**：
- [ ] AST 语法检查通过
- [ ] C 文件 gcc -fsyntax-only 通过
- [ ] 导入路径正确

### 5. 测试
**输入**：代码实现
**输出**：测试通过的证据
**层级**：
1. **语法检查**：`python3 -c "import ast; ast.parse(open('file.py').read())"`
2. **逻辑验证**：构造 mock 数据，验证核心逻辑
3. **集成测试**：与真实数据端到端测试
**检查点**：
- [ ] 正常路径测试通过
- [ ] 边界条件测试通过
- [ ] 异常路径有容错

### 6. 复核
**输入**：测试通过的代码
**输出**：复核意见
**维度**：
- [ ] 逻辑正确性：评分/分层/检测逻辑是否符合设计意图
- [ ] 边界安全：空数据、极端值、类型错误的处理
- [ ] 性能：是否有明显的 O(n²) 或内存泄漏
- [ ] 一致性：与项目整体风格是否一致

### 7. 去重去冗余
**输入**：复核后的代码
**输出**：精简后的代码
**检查点**：
- [ ] 是否有与现有模块重复的逻辑？
- [ ] 是否有未使用的函数/变量/导入？
- [ ] 是否可以合并相似的函数？
- [ ] 是否有可以简化的条件分支？

### 8. 优化
**输入**：精简后的代码
**输出**：优化后的代码
**方向**：
- 热路径是否可以向量化（NumPy）？
- 是否可以用 C 扩展加速？
- 缓存是否可以减少重复计算？
- 数据结构是否最优？

### 9. 推送
**输入**：优化后的代码
**输出**：GitHub 上的新文件/更新
**流程**：
1. 确认目标路径（`src/` / `scripts/` / `docs/`）
2. 使用 GitHub API 创建/更新文件
3. 验证推送结果
**检查点**：
- [ ] 文件路径正确
- [ ] 提交信息清晰（`v3.0: add XXX`）
- [ ] 推送后验证文件存在

### 10. 新建议
**输入**：本次迭代的经验
**输出**：更新 `下一步有趣的想法.md`
**内容**：
- 标记已实现的条目为 ✅
- 根据实现经验调整 P0/P1/P2 优先级
- 新增发现的改进方向

## 文件命名规范

| 类型 | 命名规范 | 示例 |
|------|---------|------|
| 核心模块 | `snake_case.py` | `quality.py`, `resonance.py` |
| C 扩展 | `_name.c` | `_pivots.c`, `_dtw.c` |
| Streamlit 页 | `xxx_page.py` | `v3_page.py` |
| 脚本 | `action_target.py` | `batch_compile.py` |
| 文档 | `中文名.md` | `下一步有趣的想法.md` |
| 集成补丁 | `xxx_integration.py` | `quality_integration.py` |
