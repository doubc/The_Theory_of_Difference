## 二、核心升级详解

### 2.1 自适应极值点提取 (pivots.py)

**问题**：v2.0 使用固定窗口 `min_duration` 检测 swing high/low。
- 急涨急跌行情：窗口太长，错过快速转折
- 缓慢震荡行情：窗口太短，产生大量噪声极值

**v2.5 方案**：

```
局部波动率 → 自适应窗口 → 分形一致性验证
```

1. **自适应窗口** (`_adaptive_window`)
   - 计算每个位置的局部波动率（20 bar 收益率标准差）
   - 高波动 → 缩短窗口 (捕捉急转折)
   - 低波动 → 延长窗口 (过滤慢震荡)
   - 范围约束: `[base_n // 2, base_n * 3]`

2. **分形一致性** (`_fractal_consistency`)
   - 用 3 个不同窗口 (0.5x, 1x, 2x) 验证同一位置是否都是极值
   - 返回 [0, 1] 的一致性分数
   - 低于阈值 `fractal_threshold=0.34` 的极值点被过滤
   - 核心思想：**真正的极值在多个时间尺度上都应该是极值**

3. **新配置参数**
   - `adaptive=True`: 是否启用自适应窗口
   - `fractal_threshold=0.34`: 分形一致性阈值

### 2.2 DTW 时间序列相似度 (similarity.py)

**问题**：v2.0 的 `extrema_similarity` 返回 0.0（stub），几何相似只有不变量欧氏距离。

**v2.5 方案**：新增两个相似度维度：

1. **DTW 相似度** (`dtw_similarity`)
   - 从结构的 Cycle 速度比序列中提取"形状指纹"
   - 用 Dynamic Time Warping 允许时间轴非线性拉伸
   - 捕捉"形状相似但时间错位"的结构
   - Sakoe-Chiba 带宽约束控制计算复杂度

2. **段形状相似度** (`segment_shape_similarity`)
   - 将每个 Cycle 编码为离散符号 (方向×速度×时间)
   - 用编辑距离 (Levenshtein) 度量序列差异
   - 捕捉"整体节奏相似"的结构

3. **使用方式**
   ```python
   from src.retrieval.similarity import dtw_similarity, segment_shape_similarity

   score = dtw_similarity(structure1, structure2)        # [0, 1]
   shape = segment_shape_similarity(structure1, structure2)  # [0, 1]
   ```

### 2.3 五通道守恒检查 (relations.py)

**问题**：v2.0 的 `check_conservation` 只有 3 个检查，返回 bool。

**v2.5 方案**：5 个通道 + 连续通量值

| 通道 | 检查内容 | 信号 |
|------|---------|------|
| 速度比 | 前半/后半 cycle 的速度比变化 | 收窄/扩张 |
| Zone 带宽 | 相对带宽是否极窄 | 压缩 |
| 时间压缩 | 入场/出场时间比 | 加速释放/缓慢消化 |
| 振幅衰减 | 近期 cycle 的振幅是否在衰减 | 差异转移 |
| 试探密度 | 前半/后半的试探频率比 | 突破前兆 |

**输出升级**：
```python
{
    "conservation_violated": True/False,  # 综合判断
    "flux_score": -0.35,                  # v2.5: 守恒通量 [-1, 1]
    "channel_scores": {                   # v2.5: 每通道评分
        "speed_ratio": -0.3,
        "zone_bandwidth": -1.0,
        "time_compression": 1.5,
        "amplitude_decay": 0.6,
        "touch_density": 1.8,
    },
    "transfer_channels": ["volume", "shorter_timeframe"],
    "notes": ["Zone 相对带宽极窄...", "试探密度加速..."],
}
```

### 2.4 六维叙事推断 (relations.py)

**问题**：v2.0 只有 3 个维度 (反差类型、速度特征、试探次数)。

**v2.5 方案**：6 个维度

| 维度 | 来源 | 示例输出 |
|------|------|---------|
| 反差类型 | zone.context_contrast | 恐慌性结构 |
| 速度特征 | avg_speed_ratio | 急跌/急涨型 |
| 试探密度 | cycle_count | 5次密集试探 |
| 运动态 | motion.phase_tendency | 趋向破坏 |
| 守恒通量 | motion.conservation_flux | 差异压缩中 |
| 稳定性 | stability_verdict | 表面稳定待验证 |
| 投影觉知 | projection.is_blind | ⚠️高压缩 |

**示例输出**：
```
恐慌性结构 · 急跌/急涨型 · 5次密集试探 · 趋向破坏 · 差异压缩中 · ⚠️高压缩
```

### 2.5 多信号稳态检测 (cycles.py)

**问题**：v2.0 的 `_find_nearest_stable` 只看 zone 触及，容易产生假稳态。

**v2.5 方案**：3 个信号加权评分

| 信号 | 权重 | 含义 |
|------|------|------|
| Zone 触及 | 0.3 | 价格到达已知支撑/阻力区 |
| 减速 | 0.4 | 连续两段幅度递减 |
| 波动率收缩 | 0.2 | 近期幅度 CV 下降 |

**改进**：
- 不再"找到第一个就返回"，而是扫描全部候选取最优
- 阻力评分从粗略的 `k/max_lookahead` 改为多因子加权
- 减少"一碰 zone 就认定稳态"的假阳性

### 2.6 极值点相似度 (models.py)

**修复**：`extrema_similarity` 从 `return 0.0` 改为 DTW 实现。

- 归一化两组极值点的价格序列
- 用 DTW 计算最优对齐距离
- Sigmoid 转换为 [0, 1] 相似度

## 三、配置变更

### CompilerConfig 新增参数

```python
@dataclass
class CompilerConfig:
    # ... v2.0 参数不变 ...
    
    # ── v2.5 新增 ──
    adaptive_pivots: bool = True     # 自适应极值窗口
    fractal_threshold: float = 0.34  # 分形一致性阈值 [0, 1]
```

- `adaptive_pivots=True`: 启用自适应窗口（默认开启）
- `fractal_threshold=0.34`: 至少 1/3 窗口确认才保留极值点
