# 编译器：九机制形式化定义

本文档是理论与代码之间的唯一中间层。九个机制来自统一生成语法，逐条映射到价格结构系统。修改代码前，必须先过这一层。

核心原则：一个语法，九次展开，价格结构是这套语法在市场差异上的显影。

## 机制一：聚簇

**语法**：分散差异在共同约束下发生聚集，形成局部结构密度。

**价格映射**：一组极值点 P 在价格邻近性（max|x_i-x_j|/mean < ε）、时间跨度（span > 0）、密度（|P| ≥ min_points）条件下聚簇为关键区（Zone）。Zone 按共同反差（ContrastType）分类：PANIC（恐慌性密集）、OVERSUPPLY（供需失衡长期堆积）、POLICY（政策驱动均匀间隔）。同一价位、不同共同反差类型的 Zone 不归为同一结构。

**代码**：`zones.py` `_infer_contrast` + `detect_zones`

## 机制二：层级

**语法**：聚簇的结果作为新单位进入更高层组织。

**价格映射**：Point → Segment → Zone → Cycle → Structure → Bundle，每一层是前一层的凝聚态，不是简单的容器嵌套。

**代码**：`models.py` 全部对象

## 机制三：守恒

**语法**：差异不能被无代价清零，只能在不同形态之间转化。

**价格映射**：系统在影子层工作（价格 = Π(现实差异)），不从影子反推实体。当 Zone 相对带宽被压缩时，检查速度比、时间比、试探密度、振幅趋势是否有补偿性变化。守恒通量（conservation_flux）flux_score ∈ [-1, 1]：正=释放，负=压缩，绝对值>0.3=结构正在重新分配差异。

**代码**：`relations.py` 五通道守恒 + `check_conservation`

## 机制四：先天完备性

**语法**：在破缺发生前，所有候选路径逻辑平等地并存。

**价格映射**：编译结果中的每个结构都是候选结构，系统不预判哪个会"赢"。后验统计（forward_outcome）记录破缺后哪个路径被选定：ret_5d, ret_10d, ret_20d, max_dd, max_rise。

**代码**：`sample/store.py` + `retrieval/engine.py`

## 机制五：最小变易

**语法**：变化通过局部、离散、可传递的单步推进。系统先滑向最近稳态，不是最优。

**价格映射**：极值点是价格路径上的离散事件点，段生成是事件间的局部连线。最近稳态（NearestStableState）通过多信号评分（Zone 触及 0.3 + 减速 0.4 + 波动率收缩 0.2）寻找 exit 后最先停驻的位置。

**代码**：`pivots.py` 自适应窗口 + `cycles.py` `_find_nearest_stable`

## 机制六：破缺

**语法**：候选路径失去平衡，某一条获得现实优先权。

**价格映射**：结构阶段转换——生成→确认（avg_speed_ratio 的 CV < 0.5）；任意→破坏（exit.end 距离 zone > bandwidth × 2）；任意→反演（前后半段 avg(speed_ratio) 符号相反）。

**代码**：`cycles.py` `_infer_phases`

## 机制七：循环

**语法**：被选中的路径通过反复返回关键差异关系来维持自身。

**价格映射**：Cycle 的自维持检验（entry.end 触及 zone，exit 方向与 entry 反向）。同一 Zone 反复生成的结构不是原样复制，关键关系（speed_ratio, time_ratio）被再次构造但每次有偏差，通过 DTW 相似度和段形状编辑距离追踪。

**代码**：`cycles.py` Cycle 组装 + `similarity.py` DTW + 编辑距离

## 机制八：锁定

**语法**：循环持续后压缩替代路径，形成路径依赖。

**价格映射**：结构的不变量（invariants）= 路径偏置的记录。当 avg_speed_ratio、avg_time_ratio、zone_strength、stable_state_ratio 跨多个 Cycle 保持一致时，结构进入锁定——替代路径的成本越来越高。锁定的结构更难被打破，但一旦破坏后果更剧烈。

**代码**：`structure_invariants`

## 机制九：自指

**语法**：系统把自身纳入自身的运作中。

**价格映射**：投影觉知（ProjectionAwareness）——compression_level 投影压缩度、blind_channels 盲区通道、recommended_actions 建议。反身性追踪（ReflexivityTracker）——记录每次规则匹配，回填后续结果，检测准确率衰减。高压缩时系统说"我可能在看假象"，反身性追踪说"我的输出在改变对象"。

**代码**：`ProjectionAwareness` + `reflexivity.py`
