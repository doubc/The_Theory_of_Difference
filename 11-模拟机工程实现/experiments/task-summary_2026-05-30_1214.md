### 1. P0 修复：降低系统密封率（方案 B）
**文件**: `acl/axioms_v2.py`

**修改**: `min_active_bits` 从 `max(3, N // 4)` 改为 `max(N // 3, 12)`

| 参数 | 修改前 | 修改后 |
|------|--------|--------|
| N=72 min_active_bits | 18 | 24 |
| 密封率 | 75% (54/72冻结) | 66.67% (48/72冻结) |
| N=48 min_active_bits | 12 | 16 |

**理论依据**: 诊断报告指出 75% 密封率导致结构多样性被严重压制，ODI 无法稳定超过 0.5。释放额外 8 个比特（N=72）可增加差异场的结构不对称性，提升六阈值达标概率。

### 2. P0 修复：回流通道 baseline_shift 锚定增强
**文件**: `engine/hierarchical_evolver.py`（第 ~1250 行）

**修改前**: `baseline_shift = len(anchored) * 0.01`（仅锚定数量，无锚定时为 0）

**修改后**: 
- 锚定强度加权平均：`baseline_shift = mean(anchor_strengths) * (1.0 + 0.1 * count)`
- 叠加历史锚定成功率修正：`baseline_shift += 0.02 * success_rate`
- 上限约束：`min(baseline_shift, 1.0)`

**理论依据**: 诊断报告指出回流通道为空（baseline_shift=0）导致 MSI 自参照条件无法满足。新的计算方式即使锚定数量为 0 时，也能从历史锚定事件中获取累积信号。

### 3. P1 修复：p3_active 滑动窗口门控
**文件**: `engine/hierarchical_evolver.py`

**修改**:
- 新增 `self._odi_window: List[float]` 记录最近 100 步 ODI 值
- 单步门控：`p3_active = current_odi >= 0.5`
- 窗口门控补充：`p3_active_window = mean(odi_window[-10:]) >= 0.35`
- 宽松模式：`p3_active = p3_active or p3_active_window`

**理论依据**: 诊断报告指出 ODI 仅在末端短暂超过 0.5，单步门控导致 Phase 3 组件几乎从未真正开启。滑动窗口确保近期平均密度足够时，Phase 3 能持续收集数据。
