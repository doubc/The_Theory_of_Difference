## 实验概述

**实验名称**: exp_192_phase20_p2_competition_synergy  
**实验时间**: 2026-06-12 01:15 CST  
**实验目的**: 测试多条并行自指链对有限"认知资源"（活跃比特数）的竞争，测量霸权链出现、深度差异、密封时间延迟

**假设**:
- **H20-P2a**: 多条链对有限资源竞争时，会出现"霸权链"（一条链消耗大部分比特）
- **H20-P2b**: 霸权链的涌现深度显著高于其他链（depth difference > 1）
- **H20-P2c**: 资源竞争会延迟所有链的密封时间（相比独立运行）

## 核心发现

### 1. 资源阈值效应
- **N_allocated ≥ 32**: 可以出现霸权链（某些链消耗更多比特）
- **N_allocated = 24**: 所有链达到相同深度（4或5），无霸权链
- **理论解释**: 24比特可能处于"临界阈值"以下 — 所有链都只能达到深度4，无法进一步区分

### 2. 当前实现的问题
**重要**: 当前 exp_192 实现**没有真正实现动态资源竞争**！

代码中的问题：
```python
# 当前实现：每条链分配固定 N_allocated
N_allocated = N_total // n_chains

mw = MultiWorld(
    n_worlds=n_chains,
    N0=N_allocated,  # ← 固定值，不动态减少
    ...
)
```

**真正的资源竞争应该是**:
- 当链A的L1封装消耗比特时，链B/C/D的可用比特应该减少
- 需要实现"资源池"机制：总比特数固定，链之间动态分配

### 3. 假设通过情况

| 假设 | 状态 | 说明 |
|------|------|------|
| H20-P2a | ⚠️ 部分通过 | 需要 N_allocated ≥ 32，且定义需澄清（多个链深度相同时算不算霸权） |
| H20-P2b | ❌ 大部分失败 | 霸权链深度并不显著更高 |

### 下一步修正
需要实现真正的资源竞争机制：

1. **资源池类** (`ResourcePool`):
   ```python
   class ResourcePool:
       def __init__(self, total_bits: int):
           self.total = total_bits
           self.allocated = {}  # world_id -> allocated_bits
       
       def allocate(self, world_id: int, request: int) -> int:
           """为链分配比特，返回实际分配数量"""
           available = self.total - sum(self.allocated.values())
           actual = min(request, available)
           self.allocated[world_id] = actual
           return actual
       
       def release(self, world_id: int, n_bits: int):
           """链释放比特（封装完成时）"""
           self.allocated[world_id] -= n_bits
   ```

2. **修改 MultiWorld.run_all()**: 在每步检查资源池，动态减少其他链的比特

## 理论意义

### 当前结果的理论解释

1. **无真正竞争时，链之间独立**:
   - 当前实现中，链之间不竞争资源
   - 结果反映的是"独立链在固定N下的涌现深度分布"
   - 这不是真正的"竞争与协同"

2. **N_allocated 的阈值效应**:
   - N=24 时，所有链深度=4（系统下限）
   - N=32 时，链可以分化（某些链达到深度5）
   - N=48 时，链可以达到深度6

3. **霸权需要资源优势**:
   - 2chains_N96（N=48）出现了深度差=3的霸权
   - 3chains_N96（N=32）无深度差
   - **假设**: 霸权需要"资源优势"（N_allocated 的不平等分配）

### 对差异论的意义

**如果实现真正的资源竞争**:
- 可能发现：霸权链会"挤压"其他链，使它们停在深度3-4
- 可能发现：协同效应（链之间共享封装结果）可以提高总深度
- **核心问题**: 自指链的封装是"消耗性"的还是"生产性"的？

**当前实验的启示**:
- 即使无竞争，多条链并行也会增加系统的总涌现深度（相比单链）
- 这是"并行计算"效应，不是"竞争与协同"效应
