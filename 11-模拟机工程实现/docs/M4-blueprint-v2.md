# M4 深度构型：从"单世界检测"到"种群生成链"

## 核心诊断

### 当前引擎的根本限制
当前引擎是**单世界、单结构、无种群**系统：
- 一个state tensor → 一套公理损失 → 检测"这个结构是否满足象界门槛"
- 但象界生成链描述的是**种群层次**的涌现——多个结构竞争、复制、筛选

### 生成链各步的工程充要条件

| 生成章 | 数学条件 | 工程对应 | 当前状态 |
|--------|---------|---------|---------|
| **边界** | 约束→划分→转换生成序列 | 状态空间被划分为内外区域，边界处有调节交换 | ❌ 无边界形成动力学 |
| **自维持** | SS1+SS2+SS3'：内部循环+外部开放+净回归 | 结构在开放环境中保持内部循环，统计偏好回归 | ⚠️ A7只是损失项，无循环检测 |
| **记忆** | M1+M2+M3：多稳态+跨盆抑制+外部可区分 | 结构有多个稳定态，历史路径决定当前态 | ⚠️ 有lifetime但无多稳态追踪 |
| **复制** | R1+R2+R3：模板耦合+态对应+不可逆 | 一个结构能产生相似的新结构 | ❌ 完全缺失 |
| **筛选** | F1+F2+F3：差异化耦合+存活差异+不可逆累积 | 多个结构竞争，某些存活某些消亡 | ❌ 完全缺失 |
| **功能** | FN1+FN2+FN3：定向影响+稳定性+特异性 | 结构主动改变环境，同时保持自身 | ❌ 完全缺失 |
| **前主体态** | PS1+PS2+PS3：内外区分+预期驱动+整合涌现 | 多个功能结构整合为统一行为体 | ❌ 完全缺失 |

## 架构升级方案

### 当前架构（M3及之前）
```
[State Tensor] → [Reactor] → [Axiom Loss] → [Structure Detector] → [Xiangjie Checker]
     单世界         单步演化      单套损失         单结构检测          事后门槛检测
```

### 目标架构（M4）
```
[Population of Worlds] → [Parallel Reactors] → [Selection Pressure] → [Emergence Tracker]
     多世界种群            并行演化              选择压力              涌现追踪
         ↓                      ↓                     ↓                    ↓
   [Replication Mechanism]  [Mutation/Variation]  [Differential Survival]  [Chain Stage Detection]
     复制机制                变异机制                差异化存活              生成链阶段检测
```

### 关键新增组件

#### 1. PopulationEngine（种群引擎）
- 维护 N 个并行世界（state tensors）
- 每个世界独立演化（reactor.step）
- 世界间可通过"复制"产生新世界
- 世界可通过"筛选"被淘汰

#### 2. ReplicationMechanism（复制机制）
- 输入：一个"父代"世界 + 其结构签名
- 输出：一个"子代"世界（带有变异的初始状态）
- 关键：子代继承父代的模型参数（知识）+ 状态模式（结构）
- 对应 R1+R2+R3：模板耦合 + 态对应 + 不可逆

#### 3. SelectionPressure（选择压力）
- 评估每个世界的"适应度"（基于象界门槛通过数）
- 低适应度世界被淘汰（资源重分配）
- 对应 F1+F2+F3

#### 4. EmergenceTracker（涌现追踪器）
- 实时追踪种群中各世界的生成链阶段
- 记录：最高到达阶段、阶段转换时刻、种群阶段分布
- 不再是事后检测，而是演化中的实时状态
