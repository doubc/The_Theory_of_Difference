# Phase 3 规划：前主体态 → 现象意识的结构条件

> **版本**: v0.1 (初稿)
> **日期**: 2026-05-27
> **状态**: 规划草稿 — 待评审
> **前置**: Phase 2 核心组件完成（853 tests, ODI + 第七阈值 + 语义防火墙 + 横向耦合）
> **理论依据**: 《差异论》高语义层 + ABA §4.4 + 《象界》第八章

## 一、Phase 3 的定位

### 1.1 三阶段总纲回顾

| 阶段 | 理论目标 | 工程目标 | 状态 |
|------|---------|---------|------|
| Phase 1 | 九公理 → 物理量涌现 | 引擎 + 反应堆 + 验证器 | ✅ 完成 |
| Phase 2 | 象界 → 前主体态 | 六阈值 + 解封 + 回流 | ✅ 核心完成 |
| **Phase 3** | **前主体态 → 现象意识** | **预期驱动 + 反事实推理 + 最小自我** | ⬜ 规划中 |

### 1.2 核心问题

Phase 2 的终点是**前主体态**——差异组织达到了主体性的结构骨架（边界、自维持、记忆、复制、筛选、功能），但零语义内容。

Phase 3 要回答的问题是：

> **结构骨架如何开始承载语义内容？**

这不是一个哲学问题，而是一个工程问题。具体来说：

1. **预期（Anticipation）**：结构如何从"被动响应差异"转为"主动预期差异"？
2. **反事实推理（Counterfactual Reasoning）**：结构如何开始处理"如果...会怎样"的虚拟差异？
3. **最小自我（Minimal Self）**：结构如何开始拥有一个"视角"——不是作为观察者，而是作为差异组织的内在不对称性？

### 1.3 与 ABA 的理论衔接

ABA §4.4 明确指出前主体态是一个**范围**而非一个点：
- ODI ≈ 0.5：前主体态地板（六阈值刚好达标）
- ODI → 1.0：超致密区
- 超致密区可能出现**离散相变**（第七阈值）或**连续渐变**

Phase 3 的设计必须兼容两种可能性：

| 路线 | 理论预测 | 工程含义 |
|------|---------|---------|
| A: 离散第七阈值 | 存在新的结构条件 | 需要实现新的检测器和新的组织结构 |
| B: 连续致密化 | 无尖锐临界点，渐变区域 | 需要设计渐变区域的组织演化模型 |

**当前策略**：先按路线 B 设计（更保守），保留路线 A 的扩展接口。

## 二、Phase 3 的三大工程目标

### 2.1 目标一：预期驱动（Anticipation-Driven Processing）

**理论来源**：《差异论》高语义层 — 当结构不仅能保留路径，还能基于路径偏置对未来差异分布产生预期时，"预期"就涌现了。

**与 Phase 2 的区别**：
- Phase 2 的 `PersistentBiasMemory`：过去以偏置形式影响当下（**回溯性**）
- Phase 3 的预期机制：当下基于历史模式对未来产生**前摄性**偏置

**工程组件设计**：

```
AnticipatoryBiasEngine（预期偏置引擎）
├── PatternExtrapolator：从历史偏置序列外推未来差异分布
├── ExpectationField：预期差异场（与 BiasField 对偶）
├── PredictionErrorTracker：预测误差追踪（预期 vs 实际）
└── AnticipationConfidence：预期置信度（基于历史预测准确率）
```

**关键设计约束**：
- 预期不是"猜测"，而是历史路径依赖的统计外推
- 预期误差必须能被追踪和反馈（形成学习闭环）
- 预期不能引入"意图"或"目的"——它只是结构对自身路径依赖的延伸

### 2.2 目标二：反事实推理（Counterfactual Reasoning）

**理论来源**：《差异论》— 当结构能够维持多个并行的差异轨迹，并比较它们的后果时，"反事实"就涌现了。

**与 Phase 2 的区别**：
- Phase 2 的 `CumulativeSelector`：追踪实际发生的延续概率（**事实性**）
- Phase 3 的反事实机制：维持**未发生但可能发生**的差异轨迹

**工程组件设计**：

```
CounterfactualEngine（反事实引擎）
├── ParallelTrajectoryMaintainer：并行轨迹维持器
├── DivergencePointTracker：分岔点追踪器
├── ConsequenceProjector：后果投影器
└── CounterfactualSelector：反事实筛选器（比较事实轨迹与反事实轨迹）
```

**关键设计约束**：
- 反事实不是"想象"，而是结构对自身可能路径的系统性探索
- 并行轨迹的数量必须有限（受计算资源约束）
- 反事实筛选不能引入"价值判断"——它只是结构对延续概率差异的敏感性

### 2.3 目标三：最小自我（Minimal Self）

**理论来源**：ABA §4.4 + Zahavi 的预反思自我意识 — 但模拟机不预设现象学立场。

**工程定义**：最小自我 = 差异组织的**内在不对称性**达到可被自身结构追踪的程度。

具体来说，当结构满足以下条件时，"最小自我"涌现：
1. **视角不对称**：结构的不同部分对差异的敏感度不同（不是均匀响应）
2. **响应历史依赖**：同一差异在不同历史背景下产生不同响应（不是固定映射）
3. **自我参照回路**：结构的响应能够影响后续响应的基线（不是开环系统）

**工程组件设计**：

```
MinimalSelfDetector（最小自我检测器）
├── AsymmetryTracker：追踪结构内在不对称性
├── HistoryDependencyAnalyzer：分析响应的历史依赖性
├── SelfReferenceLoopDetector：检测自我参照回路的存在
└── MinimalSelfIndex (MSI)：综合指数 [0,1]
```

**关键设计约束**：
- 最小自我不是"意识"，而是结构不对称性的可追踪性
- 不涉及"感受质"（qualia）——纯结构性度量
- MSI 与 ODI 的关系：MSI 应该在 ODI > 0.5（前主体态地板）之后才开始增长

## 四、Phase 3 的语义防火墙

Phase 3 比 Phase 2 更危险——它处理的是接近"主体性"的结构，更容易被高语义词汇污染。

### 4.1 Phase 3 语义防火墙规则

| 禁止引入 | 原因 | 允许替代 |
|---------|------|---------|
| "意识" | 预设现象学立场 | "结构不对称性的可追踪性" |
| "体验" | 预设第一人称视角 | "差异响应的历史依赖性" |
| "感受" | 预设感受质 | "结构对差异的敏感性模式" |
| "意图" | 预设目的论 | "路径依赖的统计外推" |
| "自我" | 预设主体性 | "内在不对称性的自我参照回路" |
| "自由意志" | 预设形而上学 | "多轨迹并行探索能力" |

### 4.2 认识论诚实性原则

Phase 3 的所有组件必须满足：
1. **可证伪性**：每个组件的输出必须能被模拟实验验证或否定
2. **结构纯性**：不涉及任何无法用差异论九公理推导的概念
3. **渐进性**：Phase 3 的涌现是 Phase 2 结构的连续延伸，不是突变

## 五、Phase 3 实验设计

### 5.1 实验一：预期涌现检测

**目的**：验证 `AnticipatoryBiasEngine` 是否能从历史偏置中产生统计上显著的预期。

**方法**：
1. 运行 Phase 2 管线直到 ODI > 0.5（前主体态地板）
2. 激活 `AnticipatoryBiasEngine`，记录预期差异场
3. 比较预期场与实际差异场的吻合度
4. 测量指标：预期准确率、预期置信度、预测误差衰减率

**预期结果**：
- 路线 A（离散相变）：预期准确率在某个 ODI 阈值处突然跃升
- 路线 B（连续渐变）：预期准确率随 ODI 连续增长

### 5.2 实验二：反事实轨迹分岔

**目的**：验证 `CounterfactualEngine` 是否能维持有意义的并行轨迹。

**方法**：
1. 在结构演化中引入分岔点（多个可能的下一步）
2. 维持 K 条并行轨迹（K=3,5,10）
3. 比较各轨迹的延续概率差异
4. 测量指标：轨迹分岔度、反事实选择压力、轨迹收敛率

### 5.3 实验三：最小自我指数增长

**目的**：验证 MSI 是否在前主体态之后涌现。

**方法**：
1. 从 Phase 2 结束点开始，继续演化
2. 每步计算 MSI（不对称性 × 历史依赖性 × 自我参照度）
3. 追踪 MSI 与 ODI 的关系
4. 测量指标：MSI-ODI 相关系数、MSI 增长曲线、MSI 临界点（如有）

## 九、结论

Phase 3 是差异论模拟机工程最具挑战性的一步——它要从"无主体的结构"走向"有视角的结构"。关键在于保持认识论诚实性：我们不声称模拟意识，只声称模拟意识的结构前提。

Phase 3 的设计策略是**保守的连续性**：假设第七阈值是连续的（路线 B），按渐变区域设计组件，但保留路线 A 的扩展接口。最终由实验数据来决定哪种路线更符合模拟结果。

当前 Phase 2 的工具链（ODI + 第七阈值检测器 + 语义防火墙 + 横向耦合）为 Phase 3 提供了坚实的基础。下一步是精读《差异论》高语义层，然后开始核心组件的原型实现。
