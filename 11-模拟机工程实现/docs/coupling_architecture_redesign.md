## 一、问题陈述

经过 7 个实验（exp_61~exp_67）的系统性诊断，收敛瓶颈已精确锁定：

**不是六阈值**（封口后 6/6 全部达标，达标率 92.5%~100%）
**不是稳定性**（稳定性率 93%~98%）
**不是语义防火墙**（始终通过）
**是耦合强度测量本身**。

当前耦合密度子指数（CD）均值 0.55-0.66，是六个子指数中最低的。但这不是"耦合真的弱"，而是**测量方法有结构性缺陷**。

## 二、当前耦合计算的结构性问题

### 2.1 机制分组：bit_id % 6

```python
# hierarchical_evolver.py 第 545-568 行
mechanism_signals = {name: [] for name in mech_names}
for bit_id in active_idx:
    group = bit_id % 6
    name = mech_names[group]
    mechanism_signals[name].append(dir_vals[bit_id].item())
```

**问题**：按比特索引的模 6 余数分组，将物理位置相邻的比特归入同一机制。这不是功能分组，而是位置分组。

**后果**：
- 机制间的"耦合"实际上测量的是"位置相近比特的方向值相似度"
- 这与理论上的"机制功能依赖"完全无关
- 封口后仅 12 个活跃比特，每组约 2 个比特，信号极弱且噪声大

### 2.2 耦合强度计算：几何平均归一化

```python
geo_mean = (abs_a * abs_b) ** 0.5
max_abs = max(abs_a, abs_b)
coupling_matrix[ma][mb] = abs(sign * geo_mean / (max_abs + 0.01))
```

**问题**：几何平均/最大值归一化在信号微弱时（小组均值接近 0）产生系统性低估。

**数学分析**：
- 设两个机制的均值方向分别为 μ_a 和 μ_b
- 耦合强度 = |√(|μ_a|·|μ_b|) / max(|μ_a|, |μ_b|)|
- 当 |μ_a| ≈ |μ_b| 时，值 ≈ 1.0（强耦合）
- 当 |μ_a| ≪ |μ_b| 时，值 ≈ √(|μ_a|/|μ_b|)（被抑制）
- 当两者都接近 0 时，值 ≈ 0.0（被压制到基线 0.05）

这意味着耦合强度实际上测量的是"两个机制信号的相对大小"，而不是"两个机制是否功能耦合"。

### 2.3 全对制 vs 多数制的虚假对立

exp_65 发现多数制（≥12/15）在 PSC bug 修复前有 2.5x 提升。exp_67 发现 PSC bug 修复后全对制（15/15）反而更好（76.2% vs 21.9%）。

**解释**：这不是"哪种模式更好"的问题，而是"耦合测量本身不准确"的问题。当测量有噪声时：
- 全对制：噪声导致假阴性（该耦合的被判为不耦合）
- 多数制：噪声导致假阳性（不该耦合的被判为耦合）

PSC bug 修复后，六阈值达标率从 ~46% 提升到 ~93%，耦合成为绝对瓶颈。全对制的 76.2% 收敛率实际上是在"15/15 全部达标"的严格要求下取得的——这意味着耦合测量在最佳配置下勉强够用，但远非理想。

## 三、理论对照

### 3.1 差异论中的"耦合"

差异论中的耦合（A6 流向耦合）指的是：
- 源-汇之间的**方向性**差异流
- 局域结构对差异流的**响应不对称性**
- 稳定结构之间的**功能依赖**（不是位置相邻）

### 3.2 象界中的"功能分化"

象界第八章指出，功能分化的核心是**不对称贡献**：
- 不同机制对结构稳定的贡献度不同
- 这种不对称性是结构致密化的驱动力
- 耦合应该测量"功能互补性"，而不是"信号相似度"

### 3.3 当前实现与理论的差距

| 理论概念 | 当前实现 | 差距 |
|---------|---------|------|
| 机制功能依赖 | bit_id % 6 位置分组 | 位置 ≠ 功能 |
| 功能互补性 | 方向值几何平均 | 相似度 ≠ 互补性 |
| 不对称贡献 | 对称耦合矩阵 | 丢失方向性信息 |
| 差异流方向性 | 无向耦合强度 | 丢失源-汇信息 |

## 四、重设计方向

### 4.1 短期（参数优化，不改变架构）

**目标**：在现有架构下最大化收敛率。

1. **降低耦合阈值**：从 0.30 降至 0.10-0.15
   - exp_63 已验证 0.15 比 0.30 好（21.2% vs 10.0%，修复前）
   - 但 exp_67 显示全对制在 0.30 下可达 76.2%（修复后）
   - 推荐：保持 0.30，改进测量方法

2. **增加活跃比特数**：N0=96 或 N0=120
   - 更多比特 → 每组更多信号 → 耦合测量更稳定
   - exp_67 D 配置（N0=72）表现不佳，但可能是步数不够
   - 推荐：exp_68 先验证 N0=72 + 更多步数

3. **加权耦合**：核心机制对权重更高
   - 理论依据：功能分化 → 核心机制对（如 interface_regulation ↔ self_sustaining）对结构稳定更关键
   - 实现：在 `_evaluate_coupling` 中引入权重矩阵

### 4.2 中期（算法改进）

**目标**：让耦合测量反映功能依赖而非位置相邻。

#### 方案 A：基于方向场聚类的机制识别

不再按 bit_id % 6 分组，而是：
1. 对所有活跃比特的方向值进行聚类（k-means, k=6）
2. 每个聚类代表一个功能机制
3. 计算聚类间的耦合强度

**优点**：功能分组与理论一致
**缺点**：聚类不稳定，每次运行分组可能不同

#### 方案 B：基于功能信号的耦合测量

从各 Phase 2 组件直接提取功能信号：
- `PersistentBiasMemory` → 保持深度信号
- `ReplicatePattern` → 复制保真度信号
- `CumulativeSelector` → 选择压力信号
- `FunctionalDifferentiation` → 功能分化信号
- `SelfSustainingCirculation` → 自维持信号
- `ReturnFlowChannel` → 界面调节信号

然后计算这些功能信号之间的统计依赖性（如互信息、相关系数）。

**优点**：直接测量功能耦合，与理论完全一致
**缺点**：需要重构 hierarchical_evolver 的回调管线

#### 方案 C：方向场流分析

将方向场视为差异流，计算：
1. 每个比特的**流入量**和**流出量**
2. 机制间的**净流量**（源-汇方向性）
3. 流量不对称性作为耦合强度

**优点**：与差异论 A6（流向耦合）直接对应
**缺点**：需要定义"机制边界"来计算跨机制流量

### 4.3 长期（理论对齐）

**目标**：重新定义"耦合"的工程含义，使其与差异论严格对应。

1. **分层耦合**：
   - 第一层：核心 3 对（界面↔自维持、自维持↔保持、保持↔复制）
   - 第二层：扩展 6 对（加入选择和功能分化）
   - 第三层：全部 15 对
   - 要求：第一层必须全达标，第二层多数达标，第三层无要求

2. **动态耦合阈值**：
   - 根据活跃比特数自适应调整
   - N_active ≤ 12: threshold = 0.15
   - N_active ≤ 24: threshold = 0.20
   - N_active > 24: threshold = 0.30

3. **耦合的方向性**：
   - 当前耦合矩阵是对称的（coupling[a][b] = coupling[b][a]）
   - 理论上的差异流是有方向的（源 → 汇）
   - 应该使用非对称耦合矩阵

## 六、结论

exp_61~exp_67 的实验数据揭示了一个深层问题：**当前耦合测量方法（bit_id % 6 分组 + 几何平均归一化）在结构上不反映功能依赖**。这不是参数调优能解决的，而是需要算法层面的重设计。

好消息是：即使在这个有缺陷的测量下，系统仍能达到 76.2% 的收敛率（exp_67 基线），说明底层的 Phase 2 组件功能是正确的。耦合测量的改进将进一步提升收敛率，并为 Phase 3 实验提供更可靠的结构基础。

核心认识：**位置分组是历史遗留的简化假设，不是理论要求**。下一步应该用功能信号替代位置分组，让耦合测量与差异论的功能依赖概念对齐。
