## 核心认知纠正

之前的方向是"验证公式"——手动构造状态，手动验证物理公式成立。
这是数学练习，不是涌现。

正确的方向：
  源(A1)持续注入差异 → 九公理约束下的演化 → 汇(A8)吸收差异
  → 在通量中自发形成结构 → 探测器检测涌现的物理

关键区别：
  之前：手动选状态 → 验证 Φ=-1/d_H 成立（数学练习）
  现在：系统自己演化 → 从轨迹数据中检测是否自发出现距离-作用关系（涌现）

## 理论依据

元胞自动机的教训（Wolfram NKS）：
  - 简单规则 + 足够多步 → 可以涌现复杂结构
  - 检测涌现的四类方法：
    1. 计算不可约性：行为不能被捷径预测
    2. 互信息：区域间的信息传递
    3. 熵率：有序/混沌/复杂的三分法
    4. 对称性破缺：自发对称性破缺 → 物理定律

我们的系统比元胞自动机多了九公理约束：
  - 约束本身应该产生特定的统计特征
  - 检测应该聚焦于约束导致的统计偏差
  - 需要与"无约束随机游走"对比

## 架构设计

### 1. 通量系统（已有，需增强）

当前问题：源和汇太简单——每次随机翻转固定数量的比特。
需要增强：
  - 源的位置可以动态选择（不只是随机）
  - 汇的位置可以动态选择
  - 源/汇的强度可以受公理约束调制
  - 关键：源→汇的通量应该形成稳定的流模式

### 2. 长程演化器（新）

在 HammingLattice 上跑 T=100000 步演化：
  - 每步：源注入 → 公理约束演化 → 汇吸收
  - 每 100 步采样一次状态（存 1000 个快照）
  - 记录完整轨迹用于后续分析

### 3. 涌现探测器（新）

从轨迹数据中计算以下统计量：

#### 3.1 比特翻转互信息 I(i; j)
  方法：统计位 i 翻转后，位 j 在下一步翻转的条件概率
  信号：如果 I(i;j) 随 d_H(i,j) 衰减 → 引力信号
  对比：与无约束随机游走的互信息对比

#### 3.2 汉明重量分布 P(w)
  方法：统计轨迹中各汉明重量的出现频率
  信号：如果 P(w) 在某个 w* 处有尖锐峰值 → 对称性破缺
  对比：均匀随机游走的 P(w) 应该是二项分布

#### 3.3 返回时间分布 τ
  方法：对每个位 i，计算它连续两次翻转之间的时间间隔
  信号：幂律分布 → 临界性 → 复杂涌现
  对比：泊松分布 → 无结构随机过程

#### 3.4 活跃比特聚类
  方法：基于翻转相关性对比特进行层次聚类
  信号：如果某些比特组总是同步翻转 → 形成了"粒子"
  对比：无约束情况下应该没有显著聚类

#### 3.5 DAG 方向一致性
  方法：统计每个比特的 DAG 方向（A6）是否稳定
  信号：如果某些比特组形成一致的单向流 → 时间箭头 → 弱力信号
  对比：无约束情况下方向应该随机

#### 3.6 有效自由度（A9 检测）
  方法：PCA 分析轨迹数据的主成分数
  信号：有效自由度 << N → 系统被困在低维子空间 → 涌现结构
  对比：无约束随机游走的有效自由度 ≈ N

### 4. 对照实验

关键：每个检测都要有对照。
  - 实验组：九公理约束 + 源/汇通量
  - 对照组1：无公理约束 + 相同源/汇（纯随机游走）
  - 对照组2：有公理约束 + 无源/汇（封闭系统）
  - 对照组3：有公理约束 + 源/汇但不记录（基线）

只有实验组有而对照组没有的信号，才是涌现。

## 施工顺序

批次6b：长程演化器（T=100000 步，采样记录）
批次6c：涌现探测器（6个统计量 + 对照实验）

## 关键参数

- N = 16（先小后大）
- T = 100000 步
- 采样间隔 = 100 步 → 1000 个快照
- 源强度 = 每步注入 1-3 个比特
- 汇强度 = 每步吸收 1-3 个比特
- DAG 方向 = 启用（A6 不可逆性）
- A8 对称偏好 = 启用（偏好 w=N/2）

## 预期结果

最好的情况：
  - 比特互信息随距离衰减（引力涌现）
  - 汉明重量分布有尖锐峰值（对称性破缺）
  - 活跃比特形成聚类（粒子涌现）
  - DAG 方向一致（时间箭头）

最坏的情况：
  - 所有统计量与对照组无差异 → 需要调整公理参数或增加 N
  - 系统快速收敛到均匀态 → 需要增强源/汇
  - 系统完全混沌 → 需要调整约束强度
