## A5 守恒分析

| 指标 | 值 | 评价 |
|------|-----|------|
| 总注入 | 10062 | - |
| 总吸收 | 20038 | - |
| 净通量 | **-9976** | ❌ 不平衡 |
| 净通量/注入 | -99.1% | 严重不平衡 |

**问题**：吸收是注入的 2 倍。原因是 `get_A8_sink_strength(state, n_injected)` 中 `n_injected` 是每步注入量（约 1），但 `excess * 0.5` 累积了过剩量。

**当前实现的问题**：
- 源注入强度基于 w 与 N/2 的差距（w < N/2 时注入多）
- 汇吸收强度 = 注入 + 过剩调节
- 但 w 很快达到 N/2，此时 source_strength=1，sink_strength=1+excess
- 导致吸收 > 注入

**修复方向**：源和汇应该严格平衡，excess 调节应该更温和。

## A6 DAG 分析

| 指标 | 值 | 评价 |
|------|-----|------|
| +1 (只能0→1) | 24 | ✅ 正向占多数 |
| -1 (只能1→0) | 22 | ⚠️ 负向较多 |
| 0 (双向) | 2 | ✅ 几乎全部约束 |

**观察**：
- 46/48 比特被 DAG 约束（非零方向）
- +1 和 -1 接近平衡（24 vs 22）
- 说明 A1（只允许 0→1）和汇吸收（1→0）的效果相当

**问题**：DAG 约束的效果不够强。理想情况下，+1 应该远多于 -1（因为 A1 只允许 0→1，汇吸收是例外）。

**原因**：汇吸收时 `direction[idx] = -1`（层级冻结）或 `direction[idx] = 0`（横向可逆）。当前代码中横向比特吸收后方向设为 0，导致 DAG 约束被削弱。

## A7 循环闭合分析

| 指标 | 值 | 评价 |
|------|-----|------|
| 循环状态数 | 48 | ⚠️ 稀疏 |
| 访问状态数 | 9951 | - |
| 循环比例 | 0.48% | ⚠️ 很低 |

**问题**：循环状态很少（48/9951）。原因是当前循环检测只是"状态重复"——只有当系统回到完全相同的状态时才计数。

**WorldBase 理论要求**：稳定态必须参与有向闭合循环。不是"回到相同状态"，而是"存在一条从状态出发回到自身的路径"。

**当前实现的缺陷**：
- 检测的是"精确状态重复"，不是"参与循环"
- 需要检测的是：从状态 x 出发，经过一系列演化后回到 x
- 这需要更复杂的图论算法（强连通分量）

## A9 自由度封口分析

| 指标 | 值 | 评价 |
|------|-----|------|
| 活跃比特 | 48/48 | ✅ 全部活跃 |
| 有翻转记录的比特 | 23/48 | ⚠️ 只有 48% |
| 冻结比特（从未翻转） | 25/48 | ⚠️ 52% 冻结 |
| 最大翻转次数 | 2393 | - |
| 最小翻转次数 | 1 | - |
| 平均翻转次数 | 434.8 | - |

**问题**：
- 25 个比特从未被翻转（52%）
- 翻转次数分布极不均匀（最大 2393，最小 1）
- 说明某些比特被"边缘化"——它们不参与演化

**当前实现的缺陷**：
- A9 只是追踪活跃比特，没有"封口"机制
- 多余的维度没有被冻结
- 需要实现：如果活跃比特数 > 公理要求的最小值 → 冻结多余比特
