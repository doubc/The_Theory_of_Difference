## 根因分析：三层门控连锁失效

### 第一层：ODI 无法稳定超过 0.5

- ODI 均值仅 0.1005，仅在最后几步短暂突破 0.5
- 六阈值检测中，大部分阈值未达标
- **根本原因**：系统密封率 75%（54/72 bits 被冻结），结构多样性被严重压制

```
[A9] Sealed: keeping 18 bits, freezing 54 bits
Sealed: True (54 bits, ratio=0.75)
```

密封机制在早期就冻结了 75% 的比特，导致：
1. 差异场缺乏足够的结构不对称性 → A1/A2 阈值难以达标
2. 机制间耦合被限制 → 耦合密度低 → ODI 子指数 `coupling_density` 低
3. 系统缺乏动态演化空间 → 六阈值无法同步收敛

### 第二层：p3_active 门控

`hierarchical_evolver.py` 第 1216 行：
```python
p3_active = current_odi >= 0.5  # 前主体态地板
```

Phase 3 组件（MSI 检测、叙事递归等）**仅在 ODI ≥ 0.5 的单个步长内激活**。

由于 ODI 仅在最后几步短暂超过 0.5，p3_active 几乎从未真正开启。

### 第三层：MSI 三条件积累不足

即使 p3_active 偶尔为 True，MSI 的三条件检测需要历史积累：

| 条件 | 所需窗口 | 当前状态 |
|------|---------|---------|
| 视角不对称 (asymmetry) | 10 步 | 敏感度图来自约束方向场，但冻结比特导致方向单一 |
| 历史依赖 (history_dep) | 8 步 | 偏置记忆条目不足（系统演化不充分） |
| 自参照 (self_ref) | 8 步 | 回流通道为空（baseline_shift=0） |

**三条件无法同时满足 → MSI = 0。**

### 第四层：叙事递归无输入

叙事递归运算符依赖 MSI 激活的差异信号作为输入。MSI=0 → 无差异信号 → 叙事修正幅度=0。

## 连锁失效图

```
高密封率(75%) → 结构多样性不足
    ↓
六阈值难以达标 → ODI 子指数偏低
    ↓
ODI 均值仅 0.1, 仅末端短暂>0.5
    ↓
p3_active 几乎从未开启
    ↓
MSI 检测器无输入数据积累
    ↓
MSI = 0（三条件全不满足）
    ↓
叙事递归无差异信号输入
    ↓
叙事修正 = 0
    ↓
exp_74 全部验收标准失败
```

## 修复方案

### 方案 A：降低 MSI 门控阈值（已实施）

在 exp_74 实验中为 `MinimalSelfDetector` 传入自定义配置：

```python
msi_config = {
    'odi_activation_threshold': 0.35,   # 从 0.5 降至 0.35
    'msi_activation_threshold': 0.20,   # 从 0.35 降至 0.20
    'min_active_conditions': 1,          # 从 2 降至 1
}
```

**目的**：在 structuring 区（0.3-0.5）内观察 MSI 萌芽行为，捕捉前主体态入口处的早期信号。

### 方案 B：降低系统密封率（需进一步实验）

调整 `SpatialEvolver` 或 `A9` 密封机制的参数，降低密封率至 50% 以下，释放更多结构多样性：

- 减少 `seal_ratio` 或提高解封阈值
- 延长解封机制的等待期
- 增加 lateral coupling 强度以促进结构分化

### 方案 C：延长演化步数

当前 400 步不足以让系统从 sparse 区演化到 pre_subjective 区。建议：

- 将 D_long_run 的 800 步扩展至 1600 步
- 或设计专门的 ODI 增长实验，逐步增加步数观察 ODI 曲线

### 方案 D：修改 p3_active 门控逻辑（需工程改动）

当前 `p3_active = current_odi >= 0.5` 是单步门控。建议改为：

```python
# 滑动窗口门控：最近 N 步中 ODI 均值 ≥ 阈值
p3_active = np.mean(self._odi_window[-window_size:]) >= 0.4
```

这样即使 ODI 波动，只要近期平均密度足够，Phase 3 组件就能持续收集数据。
