被排除之物不是孤立的。每一次操作中的"近似"、每一次测量中的"误差"、每一次认识中的"剩余"，都是它的聚集。它在系统的每一个裂缝中显现。

### 守恒
排除不是消失。被排除之物的总量在层间守恒。下层的排除转化为上层的结构，上层的结构又产生新的排除。

### 锁定
反复排除使系统结构固化，被排除之物的回归路径也被固定。

### 自指
系统发展到能够处理自身的排除——它开始意识到自己的裂缝，开始把"排除了什么"纳入自身的运作条件。

## 对模拟机工程的启示

### 当前理解的偏差

当前模拟机把"差异"理解为"比特的翻转"——这是对象化的理解。
差异不是比特本身，而是**比特翻转时被排除的可能性**。

当某位从 0→1 翻转时，被排除的是"保持为 0"的可能性。
这个被排除的可能性不会消失——它沉入系统的底层，在后续运算中幽灵般回归。

### 重新设定：差异作为排除的累积

**新的演化规则**：

1. 每个比特不只是 0 或 1，而是携带一个"排除历史"
2. 每次翻转时，被排除的可能性累积到底层
3. 底层累积到一定程度时，会"幽灵般回归"——影响后续翻转的概率
4. 这就是聚簇、层级、守恒的物理实现

**具体实现**：

每个比特 i 维护：
- state[i] ∈ {0, 1}：当前状态
- excluded[i] ∈ [0, ∞)：累积的被排除量
- hierarchy_level[i] ∈ {0, 1, 2, ...}：层级（由排除累积决定）

演化规则：
1. 翻转时，被排除的可能性 = 1（如果是 0→1）或 0（如果是 1→0）
2. 被排除量累积到 excluded[i]
3. excluded[i] 超过阈值时，触发层级提升
4. 层级提升时，该比特被"锁定"（不能再翻转），其排除量转移到更高层
5. 高层比特的翻转代表"被排除之物的回归"
