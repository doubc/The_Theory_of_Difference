## What was built

### 1. `engine/layer_narrative_tracker.py` — new module (810+ lines)
Three sub-components + one orchestrator:

- **PerLayerContinuityTracker**: Tracks per-layer narrative continuity using layer activity signal persistence (presence ratio) and smoothness (coefficient of variation). Each layer (MINI, INSTITUTIONAL, CIVILIZATION) has an independent Jaccard-style window.
- **PerLayerStabilityTracker**: Per-layer stability using activity level CV. Prefers CSC level_states when available.
- **PerLayerHistoryTracker**: Per-layer turning point detection using weighted second derivative of activity/ODI/MSI signals.
- **InterLayerAnalyzer**: Two-method conduction delay detection (cross-correlation with lag search + turning point matching) and Pearson correlation.
- **LayerNarrativeTracker**: Composite orchestrator that decomposes global inputs per layer each step.

### 2. `engine/hierarchical_evolver.py` — integration
- New `layer_narrative_tracker` parameter in `__init__`
- `self._last_level_states` stored after each CSC.step() call
- LNT stepped after NSE in the step callback
- Per-layer NSI/continuity/stability/history_depth written to step_results

## Key design decisions

| Decision | Rationale |
|----------|-----------|
| Separate module, not NSE subclass | Independent of NSE internals; can run in parallel |
| Activity-based continuity (not theme Jaccard) | Level distribution counts are the only per-layer signal available every step |
| CSC level_states for stability | More reliable than re-estimating for per-layer |
| Two-method conduction delay | Cross-correlation for continuous signal; turning points as ground-truth backup |
| ODI/MSI shared globally | Not re-computed per layer; per-layer differentiation comes from activity levels |
