## 一、Phase 5 续：多层级动力学与资源约束（2026-06-03 至 2026-06-04）

### 1.1 Track B 的爆发式演化：从 B2 到 B10

原始 Phase 5 规划中 Track B 仅设 B1/B2 两个子轨道。实际执行中，B1 发现 L1/L2 完全耦合（r=0.976）的问题后，Track B 经历了远超预期的演化路径：

```
原始计划: B1 → B2
实际路径: B1 → B2 → B3 → B4 → B5 → B6 → B7 → B8 → B9 → B10
```

#### B2-B4：层间解耦的失败探索

B2（exp_115，串行 CSC 耦合）尝试将 L0→L1→L2 改为串行驱动，仅降低 11.8% 耦合度（r=0.861）。B3（exp_116，通道降噪+衰减）证明噪声无法打破结构性耦合。B4（exp_117，约束传导）理论正确但实现导致 L2 被抑制到沉默——产生伪阳性。

**核心教训**：L1↔L2 的强耦合源于 L0 到 L2 的直接结构性驱动，而非耦合通道本身。

#### B5：独立聚类——真正的层间解耦突破

**exp_118**（8x2000 步）：L2 不再从 L1 派生状态，而是从 L0 独立聚类。

| 假设 | 结果 | 说明 |
|------|------|------|
| H35 (层间解耦) | **PASS** | L1↔L2 r=0.032（真实解耦，非沉默） |
| H36 (独立 ODI) | **PASS** | L2 拥有独立的组织密度 |
| H37 (独立 NSI) | **PASS** | L2 拥有独立的叙事自我 |
| H38 (基线保持) | **PASS** | H1-H8 8/8 |

**架构原则确立**：层间解耦不能通过"削弱耦合"实现，必须给每一层独立的差异空间。

#### B6：真多层级化的封口瓶颈

**exp_119/120**（8x7500 步）暴露了封口机制的结构性缺陷：

- N0=72 时封口率仅 12.5%（1/8）
- N0=48 时封口率 37.5%（3/8）
- **根因**：封口是全有/全无的——产生双峰分布，无法自然过渡到部分封口 → L1 创建

#### B7：部分封口——Track B 的转折点

**三个核心修复**（`axioms_v2.py`, `hierarchy_manager.py`, `hierarchical_evolver.py`）：

1. **部分封口**：`_seal(partial=True)`——横向比特和层级比特独立封口（各 50% 概率）
2. **分层封装**：`encapsulate_with_bits()`——使用指定冻结比特封装
3. **演化器处理**：检测部分封口 → 用横向比特创建 L1 → L0 继续演化层级比特

**验证结果**：横向 13/32（40.6%），层级 6/16（37.5%）——双峰打破，L1 创建管道端到端工作。

#### B8：L1 是被动投影——决定性发现

**exp_122**（8x10000 步，N0=48）：L0 封口率 100%，L1 形成率 100%，L1 大小均值 22.5 比特（约 50% 压缩）。

通过四版诊断方法的迭代，最终在 v4（Jaccard flux）中发现：

| 版本 | 方法 | 发现 |
|------|------|------|
| v2 | ODI delta | 后密封 CIV 仅 3 个唯一值 → 伪零 |
| v3 | CIV delta | 同上 |
| v4 | **Jaccard flux** | 后密封活动比特集合**完全相同** → flux=0.0 |

**决定性结论**：Jaccard flux=0.0 不是指标失败——是系统真理。L1 后密封没有持续动力学，它是**被动约束投影**（制度记忆），不是主动叙事主体。

#### B9：L1→L2 约束偏置耦合

**exp_123** 的 v1/v2 迭代揭示了又一个关键 Bug：

**Bug**：`get_state()` 返回 `sealed_bits`（空），而不是 `lateral_sealed_bits | sealed_bits`。`partial_seal()` 填充了 `lateral_sealed_bits` 但 `get_state()` 只查 `sealed_bits`——偏置场永不被激活。

**v2 修复**后的假设验证结果：

| 假设 | v1 | v2 | 描述 |
|------|-----|-----|------|
| H50: L1→L2 偏置效应 | 1/8 (12.5%) | **7/8 (87.5%)** PASS | L1 偏置有效性 |
| H51: L1-L2 适度解耦 | 4/8 (50%) | **7/8 (87.5%)** PASS | 相关但非融合 |
| H52: L2 自主性 | 8/8 (100%) | **8/8 (100%)** PASS | L2 独立 ODI |
| H53: L0→L2 底层驱动 | 7/8 (87.5%) | **7/8 (87.5%)** PASS | L0 底层影响 |

#### B10：L2→L3 级联——三级层级验证

**exp_124**（8x2000 步，L0→L1→L2→L3 级联）：

| 假设 | 通过率 | 结果 |
|------|--------|------|
| H54: L2 冻结事件 | **8/8 (100%)** PASS | L2 可靠封口 |
| H55: L2→L3 偏置效应 | **8/8 (100%)** PASS | 偏置均值 0.10-0.16 |
| H56: L3 自主 NSI | **8/8 (100%)** PASS | NSI 自相关 0.47-0.56 |
| H57: L1-L2 保持 | 4/8 (50%) FAIL | 级联增加耦合反馈 |

**级联耦合特征**（逐级递增）：

```
L0──r~0.50──>L1──r~0.75──>L2──r~0.87──>L3
```

**结论**：级联是收敛的——每加一层，该层与上一层的耦合增加约 0.15-0.25。L4 的耦合预计达 0.95+，无实际意义。差异论 10.3 节被工程验证——L3 确实作为框架重组层运作。

**关键发现：CIV 崩塌**——简化架构在所有 N0 水平上产生系统性低 CIV。NSE（叙事自我涌现）与 CIV 解耦——NSI 在 CIV 为零时仍然强劲（0.75-0.78），说明叙事结构与文明事件是系统行为的不同维度。

#### CIV 缺口的根因调查

C1 暴露的 CIV 崩塌触发了深入调查，最终追溯到：

**根因**：Phase 5 新增的 `set_current_step(step)` 调用（`spatial_evolver_v2.py:174`）使 `active_bits` 记录了正确的时间戳。Phase 4 中此调用缺失，`_step_counter()` 恒返回 0，导致 `_seal()` 的 `_get_active_in_window()` 将所有历史活动比特纳入窗口——产生延迟封口和丰富的状态多样性。

Phase 5 的正确时间戳使 `active_window=100` 过于紧缩，触发提前封口快捷路径（`active_now <= min_active_bits`），降低状态多样性，导致 CIV 崩塌。

**修复**（`axioms_v2.py`）：将封口的**规模检查**与**冻结操作**解耦——规模检查使用全部历史活动比特，冻结操作仅使用近期活动比特。

#### C2：NarrativeLevelBooster 与度量对齐问题

Track C2 尝试通过 `NarrativeLevelBooster`（`engine/civ_floor.py`）修复 CIV 生成。Booster 正确实现了累积分布层面的 CIV 保底（min_civ=3），但存在根本性的度量错位：Booster 修改的是传递给 NSE 的**累积**分布，而 H5/H6 度量的是**逐步**CIVILIZATION 事件。此外，`narrative_threshold=0.5` 对 Phase 5 的稀疏叙事分布过高（实际比率仅 0.1-0.2），导致触发路径永不激活。阈值修正为 0.05 后部分缓解。

## 二、Phase 6：叙事递归闭合（NRC）（2026-06-04）

### 2.1 目标与架构

Phase 6 实现叙事递归闭合——螺旋中缺失的 R→P 反馈路径：

```
P_{t+1} = R(S(M(E(P_t))))
```

核心组件：
- **EventCompressor**：将叙事张力转化为离散事件
- **MinimumVariationSelector**：选择最小阻力响应路径
- **NearestStableSettler**：事件后落入暂时平衡
- **NarrativeRecursor**：三层递归（R0 微观，R1 制度，R2 文明）
- **SpaceRewriter**：将递归输出反馈到 CSC 可能性空间

### 2.2 R2 触发机制的曲折突破

Phase 6 经历了 7 个子阶段的迭代，核心问题是 R2（文明级递归）的激活：

| 子阶段 | 实验 | 关键发现 |
|--------|------|---------|
| P1 | exp_129 | R0/R1 通过，R2 休眠（0/8） |
| P3 | exp_130 | 去除 Booster 不激活 R2 |
| P4 | exp_131 | R2 阈值扫描——全阈值零激活 |
| P4 | exp_132 | N0=72 扩大规模——仍零激活 |
| **P4** | **exp_133** | **张力触发——23/24 种子 R2 激活（突破）** |
| P5 | exp_134 | 张力+Booster 兼容——22/24 R2 |
| P6 | exp_135 | 8000 步长运行验证 |

**根因**：原始设计使用 `current_nsi >= threshold` 触发 R2。但 NSI 在前约 200 步达到峰值后衰减。当 `cycle_count > 5` 时 NSI 恒低于阈值——结构性不可能触发。

**突破**：改用**累积叙事张力**（单调递增）替代 NSI 作为 R2 触发条件。张力不依赖时序——它随叙事积累持续增长，因此始终能在阈值处触发。

### 2.4 NRC 的结构性发现

**发现 1：NRC 是事件驱动的，不是周期性的**

| 实验 | 步数 | 平均循环数 | R2 总数 |
|------|------|-----------|--------|
| exp_129 | 2000 | ~2 | 0 |
| exp_134 | 3000 | ~6 | 12 |
| exp_135 | 8000 | ~6 | 13 |

8000 步产生与 3000 步基本相同的循环频率——叙事张力在初始涌现期（前约 500 步）耗尽后不再产生新的递归。

**发现 2：R2 是稀有的且应当稀有**

每个种子产生 1-3 个 R2 事件。这与差异论 V1.7 的概念一致：文明级递归是"纪元性的"（epochal），不是常规操作。

**发现 3：核心架构极其稳固**

H1-H8 在 Phase 6 所有 48+ 次运行中 100% 通过（NSI 0.84-0.88）。NRC 反馈路径不破坏——它丰富。

### 2.5 度量调整提案

| 假设 | 当前阈值 | 建议调整 | 理由 |
|------|---------|---------|------|
| H63 收敛性 | >=6/8 负斜率 | 降至 >=5/8 | 5/8 已达成；循环数不足时斜率噪声大 |
| H64 完整性 | >=3 cycles/1k | 改为"前 500 步 >=2 cycles" | NRC 是事件驱动；全时段平均稀释了早期爆发 |

## 四、Phase 8：跨尺度螺旋耦合（2026-06-05）

### 4.1 目标与初始困境

Phase 8 目标是实现 L0↔L1 跨尺度螺旋耦合——让 L1 不仅有被动约束，还有可检测的制度循环。然而从 P0 到 P3，系统持续给出同一答案：

| 子阶段 | 实验 | 通过率 | 关键发现 |
|--------|------|--------|---------|
| P0 | exp_137 | 1/4 | L1 循环检测在 max_layers=1 下不可能 |
| P1 | exp_138 | 0/4 | 封装在所有回调之后发生 |
| P2 | exp_139 | 2/4 | L1 循环=前密封伪迹；第三层抑制 L1 |
| P3 | exp_140 | 2/4 | max_layers=2 强制成功；封口指标假阳性 |

### 4.2 L1 作为被动约束的架构决策

经过 10+ 次实验（跨 Phase 5、6、8），数据压倒性地指向同一结论。Phase 8 面临两条路径：

**路径 A（接受 L1 为被动约束提供者）**：

- L1 = 制度记忆——从 L0 早期分化中结晶的约束
- 角色是约束 L0 和为 L2 提供稳定包络，不是自主行动
- 最少代码变更，最快到达 Phase 9
- 与差异论一致：制度层是约束/记忆，不是独立的生成性差异场

**路径 B（用 Phase 5 B5 模式重新设计 L1 为独立聚类）**：

- L1 获得自己的独立比特空间和聚类机制
- 预估 400-800 行新增/修改代码，4-6 个新实验
- 结果不确定——独立 L1 仍可能收敛到 L0 的投影

**决策：选择路径 A。**

理由：
1. 证据压倒性——Phase 5 B8（10000 步）、Phase 6（8000 步）、Phase 8 P2/P3（32+ 种子）全部收敛到同一真相
2. 成本效益不支持路径 B——400-800 行代码换不确定的结果

### 5.6 P3 中发现的重大 Bug

#### GBC 张量尺寸不匹配

N0=32 和 N0=34 出现 `index N0 out of bounds for size N0` 崩溃。

**根因**：`SpatialLongRangeEvolver` 自动将 N 对齐到 3 的倍数（N0=32→N=33, N0=34→N=36），但 `AxiomConstraints.hierarchy_indices/lateral_indices` 保留原始 N0 基索引。

**修复**（commit `89ed2c2`）：在 `_run_layer()` 中添加自动裁剪/填充，对 `binding_strength` 和 `direction` 张量进行尺寸对齐。

#### NarrativeFilter 维度不匹配

`NarrativeFilter._compute_novelty()` 在 NRO 跨不同 N0 值的运行复用时崩溃——存储的历史信号维度与当前信号不同。

**修复**：每个种子必须使用新鲜的 NRO 实例。
