# Track B7: 部分封口 + 分层封装设计

## 问题回顾（exp_120 失败分析）

### 问题 1：N0=48 的双峰封口
- **现象**：8 个种子中，3 个封口（32 比特全部封口），5 个完全不封口
- **根因**：A9 封口机制是全有/全无的——所有比特统一评估绑定强度，然后冻结底部 N/3
- **问题**：绑定强度矩阵的分布决定了封口结果——种子要么形成强局部聚类（封口），要么绑定均匀扩散（不封口），中间状态不存在

### 问题 2：Layer 1 自动创建失败
- **现象**：即使 L0 封口，L1 也从未形成
- **根因**：`encapsulate_current_layer()` 使用 `constraints.sealed_bits` 作为冻结比特，但这些比特包含横向和层级比特混合。当封口不完整时，封装引擎无法正确分组
- **问题**：封装逻辑假设封口是完整的，但实际封口是部分完成的

## 解决方案

### 1. 部分封口模式（Partial Sealing）

**核心思想**：横向比特和层级比特独立封口，允许横向先封口触发 L1 创建，层级继续演化。

**实现**：
- `AxiomConstraints._seal(partial=True)`：分别对横向和层级比特排序，各冻结 50%
- `AxiomConstraints.check_A9(partial_sealing=True)`：启用部分封口模式
- `AxiomConstraints.get_sealing_status()`：返回详细的封口状态（横向/层级独立）

**封口阈值**：
- 横向封口：≥40% 横向比特被冻结
- 层级封口：≥40% 层级比特被冻结

### 2. 分层封装（Layered Encapsulation）

**核心思想**：横向封口时，只封装横向冻结比特到 L1，层级比特留在 L0 继续演化。

**实现**：
- `HierarchyManager.encapsulate_with_bits()`：使用指定的冻结/活跃比特集合执行封装
- `HierarchicalEvolver` 检测部分封口 → 调用 `encapsulate_with_bits()` 创建 L1
- L0 继续运行直到层级比特也封口

### 3. 层级延续（Layer Continuation）

**核心思想**：横向封口触发 L1 后，L0 继续运行层级比特直到它们也封口。

**实现**：
- `HierarchicalEvolver.run()` 检测到部分封口后，为 L0 运行额外步骤
- 额外步骤中只有层级比特可激活（横向比特已冻结）
- 层级封口后，L0 完全封口，可触发 L2 创建

## 实验设计（exp_121）

### 配置
- N0 = 48（exp_120 中部分封口的规模）
- partial_sealing = True
- 8 个种子 × 5000 步（+ 额外步骤）
- binding_threshold = 0.05（降低阈值，促进聚类形成）

### 假设

| 假设 | 描述 | 通过标准 |
|------|------|----------|
| H41 | 部分封口触发率 | ≥6/8 种子横向封口 |
| H42 | L1 形成率 | ≥6/8 种子创建 L1 |
| H43 | L0 层级比特最终封口 | ≥4/8 种子层级也封口 |
| H44 | L1 比特数合理 | L1 比特数在 [10, 30] 范围内 |
| H45 | 系统稳定性 | 无崩溃，所有种子运行完成 |

### 预期结果

1. **封口率提升**：部分封口模式下，横向封口率应显著提升（从 37.5% → ≥75%）
2. **L1 形成**：横向封口后 L1 应能正确创建
3. **层级延续**：部分种子中 L0 层级比特应能在额外步骤中封口
4. **双峰打破**：部分封口应打破全有/全无的双峰分布，产生中间状态

## 理论依据

- 《差异论》第二章：聚簇是差异的重新组织，不是消灭
- 《象界》第五章：封口不是全有/全无，而是自由度的逐步收缩
- 横向比特对应"现象"层面（可观察的差异分布），层级比特对应"本体"层面（差异的结构性约束）
- 横向先封口 → 现象稳定化；层级后封口 → 本体结构化
