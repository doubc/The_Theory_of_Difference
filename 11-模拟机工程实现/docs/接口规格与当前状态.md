# 接口规格与当前状态

本文档记录每个接口的规格、当前实现状态、以及已知的局限。

---

## 1. AxiomBase（公理基类）

```python
class AxiomBase(ABC):
    name: str
    category: str  # state / transition / invariant / rollout / observation / ascent_trigger

    def violation(self, state, next_state, layer, history) -> AxiomReport
```

**设计意图**：每条公理返回一个标准化的违背报告。category 决定它在 AxiomEngine 中的角色。

**当前状态**：接口已定义，九条公理已实现。

**已知局限**：
- `violation()` 的返回值量纲不统一（有的是 0-1，有的是无界实数），加权求和时可能需要归一化
- `history` 参数在某些公理中未使用，但保留以备后续需要

---

## 2. AxiomEngine（公理调度器）

```python
class AxiomEngine:
    def evaluate(self, state, next_state, layer, history) -> (total_loss, report)
    def check_ascent(self, layer, history, structures) -> bool
```

**设计意图**：统一管理所有公理的计算。`evaluate()` 返回加权总 loss 和每条公理的详细报告。`check_ascent()` 独立检查升维条件。

**当前状态**：可运行。

**已知局限**：
- loss_axioms 和 observation_axioms 的分类是硬编码的，依赖 category 字段
- `check_ascent()` 只检查 A9，如果 A9 不存在则永远返回 False

---

## 3. LayerBase（层级基类）

```python
class LayerBase(ABC):
    name: str
    stability_window: int = 16

    # 状态空间
    def initial_state(self, batch_size) -> Tensor
    def project_state(self, raw_state, mask) -> Tensor
    def valid_state(self, state) -> bool

    # 差异度量
    def measure_difference(self, state) -> Tensor
    def measure_invariant(self, state) -> Tensor
    def transition_cost(self, state, next_state) -> Tensor
    def discreteness_violation(self, state) -> Tensor
    def locality_violation(self, state, next_state) -> Tensor

    # 源汇
    def inject_difference(self, state, source_strength) -> Tensor
    def absorb_difference(self, state, sink_strength) -> Tensor

    # 稳定性
    def stability_violation(self, window) -> Tensor
    def detect_stable_structures(self, history) -> List[StableStructure]

    # 粗粒化
    def coarse_grain(self, structures) -> Optional[LayerBase]
    def measure_ascent_pressure(self, history, structures) -> float

    # 配置
    def get_axiom_weight(self, axiom_name) -> float
```

**设计意图**：所有层级共享同一接口，使公理可以跨层级复用。五元组 (S, N, T, Q, C) 分别对应状态空间、邻域、演化、守恒量、粗粒化。

**当前状态**：接口已定义，L0 已实现。

**已知局限**：
- `coarse_grain()` 在 L0 中未实现，返回 None
- `measure_ascent_pressure()` 的压力计算公式过于简单
- 没有显式的"邻域"接口——当前用 CNN 的感受野隐式定义

## 4. L0BinaryLattice（L0 二元格点）

```python
class L0BinaryLattice(LayerBase):
    def __init__(self, shape=(4,4), device="cpu", source_side="left", sink_side="right")
```

**设计意图**：最小可运行的世界层。状态空间为 [0,1] 的二维格点，左边界注入差异，右边界吸收差异。

**当前状态**：可运行（未测试）。

**已知局限**：
- `inject_difference()` 的注入位置是固定的左边界，注入概率 5% 是拍脑袋的
- `absorb_difference()` 的吸收是简单的乘法衰减，可能不够物理
- `detect_stable_structures()` 只检测"时间上稳定的区域"，没有做连通区域分析
- `coarse_grain()` 未实现
- `stability_violation()` 对全 0/全 1 的检测是全局的，没有局部化

## 5. WorldEngine（世界引擎）

```python
class WorldEngine:
    def run(self, initial_layer, max_steps, ascent_check_interval) -> Dict
    def _classify_regions(self, state, history, layer) -> RegionMap
    def _evolve_with_regions(self, state, layer, regions) -> Tensor
```

**设计意图**：核心循环。支持区域分类（死寂/爆炸/稳定/边界）和升维后自动切换新层继续运行。

**当前状态**：可运行（未测试）。

**已知局限**：
- `_classify_regions()` 的阈值（activity < 0.05, volatility > 0.8 等）是硬编码的
- `_apply_absorber()` 和 `_apply_emitter()` 的实现非常粗糙
- 区域分类是全局的（整个格点统一判定），没有空间局部化
- `_quick_pattern_persistence()` 重复了差异度量的计算，应该复用 layer 的方法
- 升维后新层的模型怎么处理？当前没有切换模型的逻辑

## 6. LocalConvModel（局部卷积模型）

```python
class LocalConvModel(nn.Module):
    def __init__(self, channels=32)
    def forward(self, state) -> Tensor
```

**设计意图**：最简单的演化模型，3×3 卷积天然满足 A3 局域性。

**当前状态**：可运行。

**已知局限**：
- 没有残差连接——模型输出和输入之间的关系不明确
- Sigmoid 输出限制在 [0,1]，可能限制了演化空间
- 没有时间记忆（每步独立预测，不知道历史）
