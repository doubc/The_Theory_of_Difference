# Task Summary: A9 Sealing Mechanism Bug Fix

**Date:** 2026-06-03 04:04 (Asia/Shanghai)
**Status:** ✅ FIX IMPLEMENTED & VERIFIED

## Problem

Phase 5 Track B6 (exp_119) and B6 Fallback (exp_120) both failed to seal Layer 0:
- exp_119: 1/8 seeds sealed (12.5%) — statistical anomaly
- exp_120: 3/8 seeds sealed (37.5%) — still far from target ≥50%
- Root cause: **not a scale problem** (N0=72 vs N0=48), but a **fundamental bug in the sealing mechanism**

## Root Cause

In `acl/axioms_v2.AxiomConstraints`:
- `active_bits` was a `Set[int]` that **only grows, never shrinks**
- Sealing condition: `len(active_bits) <= min_active_bits`
- Once the system flips enough bits (happens within ~3 steps), `active_bits` exceeds `min_active_bits` and sealing becomes **mathematically impossible**
- Evidence: exp_120 seed 42 had `active=44` at step 2500, but `min_active_bits=16` → 44 > 16, never seals

## Fix: Sliding Window for Active Bits

Replaced monotonically-growing `Set[int]` with a recency-tracking `Dict[int, int]` (bit_idx → last_active_step):

### Core Changes (`acl/axioms_v2.py`)
| Before | After |
|--------|-------|
| `active_bits: Set[int] = set()` | `active_bits: Dict[int, int] = {}` |
| `active_bits.add(flip_idx)` (only grows) | `active_bits[flip_idx] = current_step` (updates timestamp) |
| `len(self.active_bits) <= min_active_bits` | `len(active_in_window) <= min_active_bits` |
| No step tracking | `set_current_step(step)` + `_step_counter()` |

New helper methods:
- `_count_active_in_window(current_step)`: count bits active in last `active_window` steps
- `_get_active_in_window(current_step)`: return set of bits active in window
- `active_window = max(N//2, 100)`: sliding window size (default N/2 or 100)

### Evolver Wiring
Added `self.constraints.set_current_step(step)` at the start of each step loop in:
- `engine/spatial_evolver_v2.py` (`run()` method)
- `engine/long_range_evolver_v2.py` (`run()` method)
- `engine/hierarchy_manager.py` (`step_layer()` method)

### Compatibility Fixes (`engine/hierarchy_manager.py`)
Updated 3 locations that directly accessed `constraints.active_bits` (now a Dict, not a Set):
1. `encapsulate_current_layer()`: use `_get_active_in_window()` for active set
2. `_apply_cross_layer_gravity_modulation()`: use `_get_active_in_window()` for active pattern
3. `step_layer()` return dict: use `_count_active_in_window()` for active_bits count
