## 核心诊断

当前模拟机的公理实现与 WorldBase 理论存在根本性偏差：

| 理论要求 | 当前实现 | 差距 |
|----------|----------|------|
| A1：差异沿层级单调累积 | 允许任意比特翻转（包括 1→0） | ❌ 违反单调性 |
| A5：差异总量守恒 | 源注入=汇吸收（数值平衡） | ⚠️ 表面守恒，无通量约束 |
| A6：DAG不可逆 | 方向追踪存在但频繁重置 | ❌ 不可逆性不累积 |
| A7：稳定态参与有向循环 | 检测历史重复但未强制循环 | ❌ 无循环生成机制 |
| A8：偏好 w=N/2 | 权重调制但效果弱 | ⚠️ 通量压制了对称偏好 |
| A9：自由度封口 | 无实现 | ❌ 完全缺失 |
| A1'：横向涌现 | 无实现 | ❌ 完全缺失 |

| A3：有限离散 | N 固定但无离散性损失 | ✅ 基本满足 |
| A4：最小变易 | 单比特翻转 | ✅ 满足 |

## 重新设定的核心认知

**公理不是损失函数，是演化规则。**

当前实现把公理当成"损失项"——违反就惩罚。这不对。
公理应该是演化的**硬性约束**——违反的演化直接禁止，不是惩罚。

**正确理解：**
- 公理 = 演化图上的**允许边**（不是权重）
- 满足公理的演化 → 允许
- 违反公理的演化 → 禁止（不是惩罚后允许）

## 重新设计

### 核心架构：约束满足演化（CSP-based Evolution）

```
演化图 G = (X, E)
  X = {0,1}^N（状态空间）
  E = {(x,y) | y 从 x 出发满足所有公理}

每步演化：
  1. 从当前状态 x，生成所有满足 A4 的候选状态（单比特翻转邻居）
  2. 用 A1/A5/A6/A7/A8 过滤候选
  3. 从允许的候选中按 A8 权重采样
  4. 源/汇作为特殊的允许边（不来自单比特翻转，来自外部通量）
```

### 各公理的重新实现

#### A1：差异沿层级单调累积
**理论**：演化路径上汉明重量不减（w(x_{t+1}) >= w(x_t)）
**实现**：只允许 0→1 翻转（禁止 1→0）
**例外**：汇（A8）可以吸收差异（1→0），但必须在守恒约束下

#### A5：差异总量守恒
**理论**：存在守恒量 Q(x)，在所有允许演化下不变
**实现**：
- 守恒量 Q = 汉明重量 w(x)
- 内部演化（单比特翻转）不改变 w → 自动守恒
- 源注入 Δw > 0，汇吸收 Δw < 0
- **全局约束**：source_strength = sink_strength（每步严格相等）
- 通量平衡：Σ inject = Σ absorb（不是数值相等，而是路径上的守恒）

#### A6：DAG不可逆
**理论**：演化图是 DAG，不存在有向环路
**实现**：
- 每个比特记录最后一次翻转方向
- 禁止逆向翻转（direction=+1 的位不能 1→0，direction=-1 的位不能 0→1）
- **关键修正**：方向不重置！一旦某位被设为 +1，永远保持
- 源注入可以覆盖方向约束（外部力量）

#### A7：稳定态参与有向循环
**理论**：稳定态集合 S 中的每个元素都参与至少一个有向闭合循环
**实现**：
- 检测：状态 x 在历史中出现过 → 潜在稳定态
- 强制：当系统进入稳定态时，必须能找到一条回到自身的路径
- **关键修正**：A6 和 A7 的调解——局部 DAG + 全局循环
  - 局部：单步演化是 DAG（A6）
  - 全局：稳定态通过源/汇通量形成循环（A7）
  - 循环路径：x →(源注入)→ ... →(汇吸收)→ x

#### A8：对称偏好
**理论**：ρ(w) = C(N,w)/C(N,N/2)，偏好 w=N/2
**实现**：
- 不是权重调制，而是**状态空间的分层结构**
- w < N/2：优先注入（帮助达到中截面）
- w > N/2：优先吸收（防止过度累积）
- w = N/2：平衡态，注入=吸收

#### A9：自由度封口
**理论**：自由度集合 = 公理明示要求的最小集合
**实现**：
- 追踪活跃自由度（实际参与演化的比特数）
- 如果活跃自由度 > 公理要求的最小值 → 冻结多余比特
- 公理要求的最小值 = N（所有比特都是 A1-A8 要求的）
- **实际作用**：防止系统"浪费"自由度在不活跃比特上

#### A1'：横向涌现
**理论**：层级正交方向存在连续对称涌现
**实现**：
- 将 N 个比特分为两组：层级比特（A1）+ 横向比特（A1'）
- 层级比特：控制汉明重量（差异密度）
- 横向比特：控制差异的"方向"（在重量不变的情况下重新分布）
- 横向比特的演化必须保持总重量不变（A5 约束）
- **关键**：横向比特形成循环 → 涌现"粒子"

### 源/汇系统的重新设计

**当前问题**：源/汇是随机翻转固定数量比特
**重新设计**：

```
源（A1）：
  - 位置：选择当前为 0 的比特（层级比特优先）
  - 强度：基于当前 w 与 N/2 的差距
    w < N/2：强注入（帮助达到中截面）
    w > N/2：弱注入（防止过度累积）
    w = N/2：平衡注入（维持中截面）
  - 约束：只允许 0→1（A1 单调性）

汇（A8）：
  - 位置：选择当前为 1 的比特（横向比特优先）
  - 强度：与源保持平衡（A5 守恒）
  - 约束：只允许 1→0（吸收差异）
  - 特殊：汇可以"覆盖" A6 方向约束（外部力量）

通量路径：
  - 源 → 层级比特 → 横向比特 → 汇
  - 形成闭合循环（A7）
```

### 演化流程（每步）

```
1. 源注入（A1 + A8 调制）
   - 计算当前 w
   - 确定注入强度
   - 选择层级比特（0→1）

2. 内部演化（A4 + A5 + A6 + A8）
   - 生成所有单比特翻转候选
   - 过滤：A1（只允许 0→1）+ A6（不逆向）
   - 采样：A8 权重（偏好 w=N/2）
   - 执行翻转

3. 横向演化（A1' + A5）
   - 在重量不变的前提下，重新分布差异
   - 横向比特之间的翻转（保持 w 不变）
   - 形成循环模式

4. 汇吸收（A8 + A5 平衡）
   - 计算需要吸收的量（= 注入量 + 内部增量）
   - 选择横向比特（1→0）
   - 执行吸收

5. 记录轨迹
```

### 关键参数

- N = 24（层级比特 8 + 横向比特 16）
- source_strength = 动态（基于 w 与 N/2 的差距）
- sink_strength = source_strength（A5 守恒）
- coupling_strength = 0.5（比特间耦合）
- T = 50000 步

### 探测器调整

当前探测器需要调整以匹配新的演化规则：
- 互信息：检测层级比特与横向比特之间的关联
- 聚类：检测横向比特中的循环模式
- DAG一致性：应该很高（A6 严格约束）
- 有效自由度：应该降低（A9 封口）
