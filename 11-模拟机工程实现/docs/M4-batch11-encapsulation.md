# M4 批次11：分层封装 — 封口后作为整体参与新一轮九机制循环

## 工程选择声明

**选择 B：分层封装（离散模型）**

理由：
1. A9 是封口公理——明确的最小充分实现，不是连续渐变
2. 差异论九机制中的"层级"是离散的涌现步骤
3. 当前代码已有 A9 封口 + coarse_grain 基础设施，扩展成本低
4. 计算能力不支持连续模型（方案 A）
5. 分层封装对应"第一阶段→第二阶段→第三阶段"的工程路线图

## 核心设计

### 层级结构

```
第 0 层 (L0): N0 个比特，{0,1}^N0
    ↓ 演化 T0 步，A9 触发，冻结 F0 个比特
第 1 层 (L1): N1 = active_bits + encapsulated_bits
    - active_bits: L0 中未冻结的比特（直接保留）
    - encapsulated_bits: L0 冻结比特分组封装而成
    ↓ 演化 T1 步，A9 再次触发
第 2 层 (L2): N2 = ...
    ...
```

### 封装规则（核心新增）

当 A9 在第 L 层触发时：

1. **冻结比特分组**：将 sealed_bits 按绑定强度图连通分量分组
   - 用 Union-Find 算法，binding_strength > threshold 的比特对视为连通
   - 每组大小 ≥ 2 的才封装（孤立冻结比特直接丢弃）

2. **生成封装比特**：每组 → 一个高层级比特
   - 值 = 组内多数表决（majority vote）
   - 初始绑定强度 = 组内平均绑定强度
   - 层级标记 = L+1

3. **活跃比特保留**：未冻结的比特直接映射到新层
   - 保留原有值、方向、绑定强度

4. **新系统状态空间**：
   - N_{L+1} = len(active_bits) + n_encapsulated_bits
   - 状态向量 = [active_bit_values, encapsulated_bit_values]
   - 比特索引重新映射

### 跨层级交互

关键设计：高层级比特不是死的

1. **解封机制**：当封装比特所在组的所有冻结比特在低层都变为 0 时，
   封装比特可以被"解封"（重新开放为低层级自由度）
   - 触发条件：低层基底状态变化 → 重新计算封装比特值
   - 如果值变化超过阈值 → 解封

2. **基底引力**：冻结比特的分布作为"质量分布"影响高层级的源/汇权重
   - 冻结比特多的区域 = 高密度区 = 引力源
   - 高层级比特的源/汇权重受基底引力势调制

3. **层级间破缺**：高层级的 DAG 方向选择可以触发低层级某些比特重新激活
   - 当高层级比特的 DAG 方向为 -1 时，对应基底中的某些比特可以被解冻
