## **三、第一处关键问题：`LocalConvModel` 的残差写法会破坏“无变化即保持”的语义**

这是我认为现在最需要优先修的点。

当前 `LocalConvModel` 的 forward 大致是：

```python
dx = self.backbone(state)
out = state + dx

if self.use_reaction:
    reaction = self.reaction(state)
    out = out + self.reaction_scale * reaction

return torch.sigmoid(out)
```

问题在于，`sigmoid(state + dx)` 不是一个真正的残差更新。对于状态值 $0$，如果 $dx = 0$，输出是 $\sigma(0) = 0.5$
；对于状态值 $1$，如果 $dx = 0$，输出是 $\sigma(1) \approx 0.731$
。也就是说，即使模型学到“什么都不改”，输出也不会保持原状态。
这会直接影响 A4 最小变易和 A2 离散编码。模型会天然把 $0/1$ 状态推向中间值，然后 A2 又惩罚中间值，A4 又惩罚变化，结果训练很可能在几个
loss 之间拉扯，而不是学习真正的局部演化。

建议把模型改成“预测 delta”，不要在残差之后直接 sigmoid。第一版可以这样：

```python
class LocalConvModel(nn.Module):
    def __init__(self, channels: int = 32, use_reaction: bool = True, step_scale: float = 0.2):
        super().__init__()
        self.use_reaction = use_reaction
        self.step_scale = step_scale

        self.backbone = nn.Sequential(
            nn.Conv2d(1, channels, kernel_size=3, padding=1),
            nn.GELU(),
            nn.Conv2d(channels, channels, kernel_size=3, padding=1),
            nn.GELU(),
            nn.Conv2d(channels, 1, kernel_size=3, padding=1),
        )

        if use_reaction:
            self.reaction = nn.Sequential(
                nn.Conv2d(1, channels, kernel_size=1),
                nn.Tanh(),
                nn.Conv2d(channels, 1, kernel_size=1),
                nn.Tanh(),
            )
            self.reaction_scale = nn.Parameter(torch.tensor(0.05))

    def forward(self, state: torch.Tensor) -> torch.Tensor:
        delta = torch.tanh(self.backbone(state))

        if self.use_reaction:
            reaction = self.reaction(state)
            delta = delta + self.reaction_scale * reaction

        next_state = state + self.step_scale * delta
        return next_state.clamp(0.0, 1.0)
```

这个版本的语义更干净：

```text
模型输出的是变化量；
无变化时 state 保持不变；
A4 约束的就是 delta 的大小；
A2 再把状态压向离散；
A1/A8 边界条件再注入/吸收差异。
```

如果你后续想要更严格，可以把状态分成两套表示：内部用 logits，外部用 probability/state。但第一阶段不必这么复杂。现在先把
“sigmoid 残差破坏恒等映射” 修掉就很重要。

## **四、第二处关键问题：当前 A5 守恒和源/汇开放系统有张力**

`DifferenceReactor.step()` 里先让模型预测，再执行：

```python
next_state = self.layer.inject_difference(next_state)
next_state = self.layer.absorb_difference(next_state)
```

然后在 `_compute_axiom_loss()` 里计算 A5：

```python
q_now = self.layer.measure_invariant(state)
q_next = self.layer.measure_invariant(next_state)
a5_val = ((q_next - q_now) ** 2 / (q_now ** 2 + 1e-6)).mean()
```

这表示 A5
惩罚总量变化。但你的系统现在是开放系统，有左边界注入和右边界吸收。既然源/汇是理论设计的一部分，那么总量变化不一定是违背，真正该检查的是“通量平衡”。
现在的问题是：A1/A8 在推动开放流，A5 在惩罚开放流。模型可能学会抵消源汇，或者把状态压到某个折中区间，而不是形成真正的非平衡稳态。

建议把 A5 分成两个版本：

```text
A5_closed_conservation：闭合系统总量守恒。
A5_open_flux_balance：开放系统通量平衡。
```

第一版工程实现可以不复杂。让 `inject_difference()` 和 `absorb_difference()` 返回注入量与吸收量，而不是只返回状态。

例如：

```python
def apply_boundary_flow(self, state, source_strength=1.0, sink_strength=1.0):
    before = state
    after_source = self.inject_difference(before, source_strength)
    injected = self.measure_invariant(after_source) - self.measure_invariant(before)

    after_sink = self.absorb_difference(after_source, sink_strength)
    absorbed = self.measure_invariant(after_source) - self.measure_invariant(after_sink)

    return after_sink, injected, absorbed
```

然后 A5 改成：

$$
\mathcal{V}_{A5}
=
\left[
(Q_{t+1} - Q_t) - (I_t - O_t)
\right]^2
$$

代码可以是：

```python
flux_residual = (q_next - q_now) - (injected - absorbed)
a5_val = (flux_residual ** 2 / (q_now ** 2 + 1e-6)).mean()
```

这样 A5 就不再反对源/汇，而是检查源/汇是否被系统正确承接。这会更符合“差异反应堆”的定位。

## **五、第三处关键问题：`AxiomEngine` 和 `DifferenceReactor` 现在存在双轨制**

现在 `acl/axiom_base.py` 里有一个 `AxiomEngine.evaluate()`，它会遍历公理并返回总 loss 和报告。但 `DifferenceReactor`
没有直接用这个 `AxiomEngine.evaluate()` 来训练，而是在 `_compute_axiom_loss()` 里重新手写了一套可微分损失。这个选择有原因：
`axioms.py` 里的各公理 `violation()` 会把 tensor 转成 `float`
，所以它们不适合反向传播。
但这会带来一个架构风险：

```text
公理报告路径：AxiomEngine → axioms.py
训练损失路径：DifferenceReactor._compute_axiom_loss()
```

如果两条路径长期并存，它们很容易慢慢不一致。最后你看到的“公理违背报告”和实际训练的“公理约束”可能不是同一套东西。

建议下一步不要急着加新公理，而是先统一接口。可以让每条公理返回两个东西：

```python
@dataclass
class AxiomReport:
    name: str
    raw_violation: float
    weight: float
    weighted_violation: float
    metadata: dict = field(default_factory=dict)
    tensor_loss: Optional[torch.Tensor] = None
```

然后 `AxiomBase` 增加一个可微分接口：

```python
class AxiomBase(ABC):
    name: str
    category: str

    def loss(self, state, next_state, layer, history) -> torch.Tensor:
        raise NotImplementedError

    def report(self, loss_tensor, state, next_state, layer, history) -> AxiomReport:
        return AxiomReport(
            name=self.name,
            raw_violation=float(loss_tensor.detach()),
            tensor_loss=loss_tensor,
        )
```

这样训练和报告就不会分裂：

```python
loss_tensor = axiom.loss(state, next_state, layer, history)
report = axiom.report(loss_tensor, state, next_state, layer, history)
```

如果你暂时不想重构，也至少建议在 `reactor.py` 顶部写清楚：

```text
注意：当前训练损失不走 AxiomEngine.evaluate，而由 DifferenceReactor 内部可微分版本实现。
后续需要统一。
```

这是一条很重要的技术债，要明确记录。

## **六、第四处关键问题：1D 实验下 `measure_difference()` 可能返回空张量**

`L0BinaryLattice.measure_difference()` 当前假定状态有二维空间：

```python
dx = (state[:, :, :, 1:] - state[:, :, :, :-1]).abs()
dy = (state[:, :, 1:, :] - state[:, :, :-1, :]).abs()
diff = (dx[:, :, :-1, :] + dy[:, :, :, :-1]) / 2.0
return diff
```

对于 2D 网格这没问题。但 1D 实验使用的是 `shape=(1, length)`。这时高度 $H=1$，`dy` 的高度维度会变成 $0$，`dx[:, :, :-1, :]`
也会变成空，最后 `diff` 很可能是空张量。后续如果对它 `.mean()`，就可能得到
NaN。
建议改成兼容 1D/2D 的版本：

```python
def measure_difference(self, state: torch.Tensor) -> torch.Tensor:
    parts = []

    if state.shape[-1] > 1:
        dx = (state[:, :, :, 1:] - state[:, :, :, :-1]).abs()
        parts.append(dx.mean(dim=(-1, -2), keepdim=True))

    if state.shape[-2] > 1:
        dy = (state[:, :, 1:, :] - state[:, :, :-1, :]).abs()
        parts.append(dy.mean(dim=(-1, -2), keepdim=True))

    if not parts:
        return torch.zeros(state.shape[0], 1, 1, 1, device=state.device)

    return sum(parts) / len(parts)
```

如果你还需要空间分布型差异场，可以另写一个：

```python
measure_difference_field()
```

不要让同一个函数既承担“差异总量指标”，又承担“差异空间图”。这两者最好分开。

## **十一、关于 A9 升维：方向对，但压力公式还太弱**

A9 当前作为 `ascent_trigger` 是对的。`check_ascent()` 会检查 structures 是否存在，并调用 `layer.measure_ascent_pressure()`
。而 `L0BinaryLattice.measure_ascent_pressure()` 当前是：

```python
residual = ((q2 - q1) ** 2).mean().item()
density = len(structures) / max(1, self.shape[0] * self.shape[1])
return residual * density
```

这个版本可以跑，但理论意义还比较弱。它把“升维压力”理解为守恒残差乘结构密度。问题是：如果结构非常稳定，守恒残差可能很低；但这恰恰可能是可封装结构成熟的信号，而不是升维压力低。
建议把 A9 升维压力改成“当前层表达耗尽”的组合指标，而不是只看守恒残差：

$$
P_{ascent}
=
w_1 S_{stable}

+

w_2 C_{compression}

+

w_3 B_{boundary}

+

w_4 I_{interaction}

+

w_5 R_{residual}
$$

第一版可以这样写：

```python
def measure_ascent_pressure(self, history, structures) -> float:
    if len(history) < self.stability_window or not structures:
        return 0.0

    structure_density = len(structures) / max(1, self.shape[0] * self.shape[1])

    avg_lifetime = sum(s.lifetime for s in structures) / len(structures)
    lifetime_score = min(1.0, avg_lifetime / self.stability_window)

    turnover = sum(s.material_turnover for s in structures) / len(structures)
    closure_score = 1.0 / (1.0 + turnover)

    q1 = self.measure_invariant(history[-2])
    q2 = self.measure_invariant(history[-1])
    residual = ((q2 - q1) ** 2 / (q1 ** 2 + 1e-6)).mean().item()
    residual_score = min(1.0, residual)

    pressure = (
            0.35 * lifetime_score
            + 0.25 * closure_score
            + 0.20 * structure_density
            + 0.20 * residual_score
    )

    return pressure
```

这还不是最终版，但比 `residual * density` 更接近“稳定结构已经足够成为下一层单元”的意思。

### **1. 修正 `LocalConvModel`：从 sigmoid residual 改成 delta 更新**

当前模型的问题在于：

```python
return torch.sigmoid(state + dx)
```

这会导致即使 `dx = 0`，状态也不会保持不变。对一个反应堆模型来说，恒等映射应该是可学习、可表达的基本行为。否则 A4
最小变易会被模型结构本身破坏。

建议直接改成：

```python
"""models/local_conv_model.py — 局部卷积模型

3×3 卷积 + 残差 delta 更新 + 反应分支。
模型预测的是局部变化量，而不是直接预测下一状态。
这样可以保证 dx=0 时 next_state≈state，符合 A4 最小变易语义。
"""

import torch
import torch.nn as nn


class LocalConvModel(nn.Module):
    """局部卷积模型，用于差异反应堆动力学。

    结构：
    - 主干：3 层 3×3 卷积 + GELU，保证局域性；
    - delta 更新：next = state + step_scale * tanh(delta)；
    - 反应分支：1×1 卷积 + tanh，用于局部自强化；
    - clamp：保证输出仍在 [0, 1]。
    """

    def __init__(
            self,
            channels: int = 32,
            use_reaction: bool = True,
            step_scale: float = 0.2,
    ):
        super().__init__()
        self.use_reaction = use_reaction
        self.step_scale = step_scale

        self.backbone = nn.Sequential(
            nn.Conv2d(1, channels, kernel_size=3, padding=1),
            nn.GELU(),
            nn.Conv2d(channels, channels, kernel_size=3, padding=1),
            nn.GELU(),
            nn.Conv2d(channels, 1, kernel_size=3, padding=1),
        )

        if use_reaction:
            self.reaction = nn.Sequential(
                nn.Conv2d(1, channels, kernel_size=1),
                nn.Tanh(),
                nn.Conv2d(channels, 1, kernel_size=1),
                nn.Tanh(),
            )
            self.reaction_scale = nn.Parameter(torch.tensor(0.05))

    def forward(self, state: torch.Tensor) -> torch.Tensor:
        """预测下一状态。

        Args:
            state: (batch, 1, H, W)，值域 [0, 1]

        Returns:
            next_state: (batch, 1, H, W)，值域 [0, 1]
        """
        delta = torch.tanh(self.backbone(state))

        if self.use_reaction:
            reaction = self.reaction(state)
            delta = delta + self.reaction_scale * reaction

        next_state = state + self.step_scale * delta
        return next_state.clamp(0.0, 1.0)
```

这个修改会让模型结构更符合“局部最小变易”。如果模型认为当前局部状态不需要变化，它可以输出接近零的 delta，从而让 `next_state`
接近 `state`。这比原来的 sigmoid 输出更稳定。

### **2. 修正 `L0BinaryLattice.measure_difference`：兼容 1D/2D**

当前 `measure_difference()` 对 `shape=(1, length)` 的 1D 实验不安全。建议把“差异标量指标”和“差异空间场”分开。第一步先保守修复现有函数，让它永远不返回空张量。

建议替换为：

```python
def measure_difference(self, state: torch.Tensor) -> torch.Tensor:
    """度量当前层的差异强度。

    兼容：
    - 1D: shape=(1, length)
    - 2D: shape=(height, width)

    返回一个 batch 级别的差异强度张量，避免 1D 情况下产生空张量。
    """
    parts = []

    if state.shape[-1] > 1:
        dx = (state[:, :, :, 1:] - state[:, :, :, :-1]).abs()
        parts.append(dx.mean(dim=(-1, -2), keepdim=True))

    if state.shape[-2] > 1:
        dy = (state[:, :, 1:, :] - state[:, :, :-1, :]).abs()
        parts.append(dy.mean(dim=(-1, -2), keepdim=True))

    if not parts:
        return torch.zeros(
            state.shape[0],
            1,
            1,
            1,
            device=state.device,
            dtype=state.dtype,
        )

    return sum(parts) / len(parts)
```

如果你后面还需要“空间差异场”，建议另加一个函数：

```python
def measure_difference_field(self, state: torch.Tensor) -> torch.Tensor:
    """返回空间差异场，用于可视化和局部结构检测。"""
    dx_field = None
    dy_field = None

    if state.shape[-1] > 1:
        dx_field = (state[:, :, :, 1:] - state[:, :, :, :-1]).abs()

    if state.shape[-2] > 1:
        dy_field = (state[:, :, 1:, :] - state[:, :, :-1, :]).abs()

    if dx_field is None and dy_field is None:
        return torch.zeros_like(state)

    if dx_field is not None and dy_field is None:
        return torch.nn.functional.pad(dx_field, (0, 1, 0, 0))

    if dx_field is None and dy_field is not None:
        return torch.nn.functional.pad(dy_field, (0, 0, 0, 1))

    h = min(dx_field.shape[-2], dy_field.shape[-2])
    w = min(dx_field.shape[-1], dy_field.shape[-1])
    field = 0.5 * (dx_field[:, :, :h, :w] + dy_field[:, :, :h, :w])

    pad_h = state.shape[-2] - field.shape[-2]
    pad_w = state.shape[-1] - field.shape[-1]
    return torch.nn.functional.pad(field, (0, pad_w, 0, pad_h))
```

这样以后 A1/A6 的观测指标可以用 `measure_difference()`，而可视化和局部结构检测可以用 `measure_difference_field()`
。两者职责分离后，很多后续 bug 会少很多。

```python
def apply_boundary_flow(
        self,
        state: torch.Tensor,
        source_strength: float = 1.0,
        sink_strength: float = 1.0,
):
    """同时应用源端注入和汇端吸收，并返回通量统计。

    Returns:
        next_state: 应用源/汇后的状态
        injected: 源端注入总量
        absorbed: 汇端吸收总量
    """
    before = state

    after_source = self.inject_difference(
        before,
        source_strength=source_strength,
    )
    injected = self.measure_invariant(after_source) - self.measure_invariant(before)

    after_sink = self.absorb_difference(
        after_source,
        sink_strength=sink_strength,
    )
    absorbed = self.measure_invariant(after_source) - self.measure_invariant(after_sink)

    return after_sink, injected, absorbed
```

这个函数不改变已有 `inject_difference()` 和 `absorb_difference()` 的语义，只是把通量统计显式暴露出来。

### **3. 修改 A5 loss：从总量守恒改成通量平衡**

把当前 A5：

```python
q_now = self.layer.measure_invariant(state)
```

改成：

```python
q_now = self.layer.measure_invariant(state)
q_next = self.layer.measure_invariant(next_state)

if boundary_info is not None and boundary_info:
    injected = boundary_info.get("injected", torch.zeros_like(q_now))
    absorbed = boundary_info.get("absorbed", torch.zeros_like(q_now))

    expected_delta = injected - absorbed
    actual_delta = q_next - q_now

    flux_residual = actual_delta - expected_delta
    a5_val = ((flux_residual ** 2) / (q_now ** 2 + 1e-6)).mean()

    a5_metadata = {
        "mode": "open_flux_balance",
        "actual_delta": float(actual_delta.detach().mean()),
        "expected_delta": float(expected_delta.detach().mean()),
        "injected": float(injected.detach().mean()),
        "absorbed": float(absorbed.detach().mean()),
        "flux_residual": float(flux_residual.detach().mean()),
    }
else:
    a5_val = ((q_next - q_now) ** 2 / (q_now ** 2 + 1e-6)).mean()

    a5_metadata = {
        "mode": "closed_conservation",
        "conservation_residual": float(a5_val.detach()),
    }

w5 = self.layer.get_axiom_weight("A5_conservation")
a5_loss = a5_val * w5
loss_parts.append(a5_loss)

report["A5_conservation"] = AxiomReport(
    name="A5_conservation",
    raw_violation=float(a5_val.detach()),
    weight=w5,
    weighted_violation=float(a5_loss.detach()),
    metadata=a5_metadata,
)
```

这样 A5 就不再和 A1/A8 对抗，而是检查系统是否正确承接源/汇通量。


```python
@dataclass
class StructureScore:
    lifetime: float
    boundary: float
    closure: float
    turnover: float
    interaction: float
    compression: float

    @property
    def total(self) -> float:
        return (
                0.25 * self.lifetime
                + 0.20 * self.boundary
                + 0.20 * self.closure
                + 0.15 * self.turnover
                + 0.10 * self.interaction
                + 0.10 * self.compression
        )
```

```python
@dataclass
class StableStructureCandidate:
    mask: torch.Tensor
    birth_step: int
    last_seen_step: int
    source_layer: str
    signature: torch.Tensor
    score: StructureScore
```

```python
class StableStructureValidator:
    def __init__(self, min_score: float = 0.65):
        self.min_score = min_score

    def validate(self, candidate: StableStructureCandidate) -> bool:
        return candidate.score.total >= self.min_score
```

这样 M2 的任务就很清楚：

```text

```

## **二十一、M2 的稳定结构判据建议**

你后面一定会遇到一个问题：什么叫“稳定结构”？如果只看时间方差小，就会把死寂区域也当成稳定结构。这个定义太弱。

建议稳定结构至少满足五个条件。

---

### **1. 持续性：它要活得足够久**

结构不能只出现一两步。最低要求是跨越一个稳定窗口：

```text
lifetime >= stability_window
```

更强要求是：

```text
lifetime >= 2 * stability_window
```

持续性评分可以是：

$$
S_{life} = \min\left(1, \frac{lifetime}{2W}\right)
$$

其中 $W$ 是稳定窗口长度。

---

### **2. 边界性：它要有可识别边界**

如果一个结构没有边界，它可能只是背景场的一部分。边界可以用 mask 的边缘梯度来估计。

直觉上：

```text
结构内部相对一致；
结构边界与外部有差异；
结构不是全局均匀背景。
```

---

### **3. 闭合性：它要能保持模式**

闭合性不是完全静止，而是“模式保持”。也就是说，构成它的具体格点值可以变化，但整体签名要持续相似。

可以用 signature cosine similarity：

$$
S_{closure}
=
\frac{1 + \cos(sig_t, sig_{t-1})}{2}
$$

---

### **4. 物质更换：它不能只是死块**

这是差异论里非常关键的一点。一个活结构应该允许“材料换掉但模式保持”。如果完全不动，它更像死稳定；如果完全乱变，它是噪声。

所以 material turnover 应该在一个中间范围：

```text
turnover 太低：死结构；
turnover 太高：噪声；
turnover 中等：活结构。
```

评分可以用钟形函数：

$$
S_{turnover}
=
\exp\left(-\frac{(T - T_0)^2}{2\sigma^2}\right)
$$

其中 $T_0$ 是理想更换率。

---

### **5. 交互性：它要能影响周围或响应周围**

一个完全孤立、完全不影响任何东西的块，即使稳定，也未必值得升维。真正的高层单元应该能参与下一层动力学。

第一版可以用边界附近的状态变化来估计：

```text
interaction = boundary_activity / internal_activity
```

这个指标不需要一开始很精确，但必须有。因为它决定了结构是不是“可作为下一层节点”。

## **二十二、关于 L1 的建议：不要马上做物理，先做 Pattern Graph**

M2 之后的 L1 不建议直接做“粒子”“场”“力”。那会太快把理论带到现实物理解释里，容易失控。

更稳的 L1 是：

```text
L1 Pattern Graph
```

也就是把 L0 中通过验证的稳定结构封装成图节点，节点之间的关系来自空间邻近、通量交换、边界接触、共同历史等。

L1 的五元组可以这样定义：

```text
S1：结构 token 的集合；
N1：token 之间的邻接关系；
T1：token graph 的演化规则；
Q1：结构数量、连接强度、通量平衡等高层守恒量；
C1：从 L0 稳定结构到 L1 token graph 的粗粒化映射。
```

对应代码可以是：

```text
layers/L1_pattern_graph.py
```

不要一开始做复杂图神经网络。第一版可以只是一个普通 Python 图结构，甚至不用引入额外依赖。

```python
@dataclass
class PatternToken:
    token_id: int
    features: torch.Tensor
    source_structure: StableStructure
    position: torch.Tensor
    lifetime: int


@dataclass
class PatternEdge:
    source: int
    target: int
    weight: float
    relation_type: str
```

然后：

```python
class L1PatternGraph(LayerBase):
    name = "L1_pattern_graph"

    def __init__(self, tokens, edges):
        self.tokens = tokens
        self.edges = edges

    def initial_state(self, batch_size: int = 1):
        ...
```

L1 的关键不是复杂，而是要证明：

```text
L0 的稳定结构可以被压缩成 token；
token 之间存在可追踪关系；
L1 上可以继续定义差异、邻域、演化、守恒和粗粒化。
```

只要这个链路打通，递归世界引擎才真正成立。
