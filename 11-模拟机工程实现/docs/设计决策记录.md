# 差异论局部世界实验机 — 设计决策记录

本文档记录所有已做出的设计决策、背后的推理、以及尚未解决的问题。
目的是保证后续工作可以接续，不需要重新讨论已经确定的事情。

---

## 一、项目定位

**不是什么：**
- 不是物理模拟器（不直接声称复现现实物理）
- 不是大语言模型项目（不把理论文本当训练语料）
- 不是视觉演示器（不追求好看的图案）

**是什么：**
- 公理约束下的递归世界实验机
- 核心问题：九公理能否在离散世界中约束演化，生成可封装为下一层的稳定结构？
- 如果能，同一套公理能否继续约束下一层？

## 二、已确定的设计决策

### 2.1 A1 差异源：太阳模型

**决策**：设定一个内在差异源，持续 +1，类似太阳持续发光。

**推理**：差异论需要解释"世界为什么不走向热寂"。封闭系统趋向均衡，只有开放系统才能持续产生结构。差异源是系统保持远离平衡态的必要条件。

**实现**：`layer.inject_difference()`，在格点的源端（左边界）低概率注入差异。

**未解决问题**：
- 注入速率是否应随层级变化？
- 注入的分布模式（均匀？高斯？随机？）是否影响涌现？
- 多层递归时，下层的差异源是什么？是上层的"太阳"还是上层结构的副产品？

### 2.2 A6 差异流向：源-汇结构

**决策**：A6 不作为独立 loss，而是由源（A1）和汇（A8）的边界条件隐式定义方向性。

**推理**：差异从源端流向汇端，形成自然的梯度和方向性。A6 描述的是这种流动的有序性，不需要额外约束。

**实现**：源端注入 + 汇端吸收 = 自动形成从左到右的差异流。

### 2.3 A8 差异汇：对称偏好的正确理解

**决策**：A8 不是"强制对称"，而是差异的自然耗散。系统有源端（+1）和汇端（-1），结构在梯度带中间自组织。

**推理**：对称性不是目标，而是结果。对称结构恰好最能承受注入和耗散之间的张力，所以被保留下来。漩涡是对称的，不是因为水喜欢圆，而是圆形最能让流入和流出平衡。

**实现**：`layer.absorb_difference()`，在格点的汇端（右边界）吸收差异。

**与原始设计的关系**：原始施工图把 A8 写成"symmetry_preference"作为选择偏置。我们把它重新理解为物理上的耗散/汇，这是更可操作的定义。

### 2.4 A7 稳定闭合：活/死/噪声三分

**决策**：稳定不是"状态不变"，而是"模式持续"。区分三种状态：
- **活结构**：模式持续 + 物质在更换（漩涡型）→ 好
- **死结构**：模式持续 + 物质不换（冻结型）→ 惩罚
- **噪声**：模式不持续 → 失败

**推理**：差异论说"结构不是差异消失后的产物，而是差异被组织起来后的形态"。漩涡里的水分子一直在换，但漩涡一直在。所以稳定性要用"模式持续性"而非"状态不变性"来度量。

**实现**：四个子指标
- `_pattern_persistence`：差异场的时间相关性（余弦相似度）
- `_boundary_integrity`：梯度场的时间稳定性
- `_material_turnover`：状态值的平均变化量
- `_frozenness`：pattern 高 + turnover 低 → 冻结惩罚

**未解决问题**：
- pattern_signature 用什么表示？当前用差异场的余弦相似度，可能不够robust
- 边界检测用 Sobel 梯度，对离散格点可能噪声太大
- "足够持续"的阈值怎么定？当前用 stability_window=16，是拍脑袋的

### 2.5 A5 + A9 升维机制：素数模型

**决策**：
- A5（守恒）提供升维压力——守恒残差无法在当前层消解时，系统被迫寻找新自由度
- A9（最小充分实现）是升维触发器——检测当前层表达是否耗尽
- A9 对应代数里的素数：不可再约，必须升维

**推理（用户原话）**：A9 不是说不允许其他状态存在，而是到了一个限度就必须再次分层。A9 的筛选提供了升维的可能。A9 不是压缩惩罚，而是升维触发器。

**升维触发条件**：
1. A5 检测到守恒残差（流入 ≠ 流出 + 内部变化）
2. A9 检测到残差无法用当前层自由度消解
3. 同时存在可封装的稳定结构
→ 三个条件同时成立，触发升维

**实现**：
- `A5_Conservation.violation()` 追踪守恒残差
- `A9_MinimalSufficient.check_ascent()` 做升维判定
- `layer.measure_ascent_pressure()` 综合残差和结构密度计算压力

**未解决问题**：
- 升维阈值 `ascent_threshold` 怎么定？当前设 0.5，完全是拍脑袋
- 守恒残差的"不可约性"怎么检测？当前只用了残差大小 × 结构密度，没有真正检测不可约性
- 升维后新层的状态空间怎么从旧层推导？`coarse_grain()` 只有接口没有实现

### 2.6 区域分类：死寂/爆炸不是失败

**决策**：系统的一部分陷入死寂或爆炸时，不终止实验，而是将这些区域转化为新的边界条件：
- 死寂区 → 吸收体（黑洞型）
- 爆炸区 → 辐射源（超新星型）
- 两者交界 → 相边界（结构最丰富的地方）

**推理（用户原话）**：陷入死寂或爆炸的话，能否不动，新产出一系列空间继续运作，像是现实里面的黑洞或者某些宇宙学的现象。

**实现**：`WorldEngine._classify_regions()` 把状态空间分为四类区域，`_evolve_with_regions()` 按区域分流处理。

**未解决问题**：
- 吸收体和辐射源的强度怎么定？当前是固定系数
- 区域边界是否会移动？如果会，怎么追踪？
- 死寂区有没有可能"复活"？还是永久转化为吸收体？

### 2.7 公理分类：约束/观测/触发

**决策**：九条公理分三类运作：
- **约束类**（A2/A3/A4/A5/A7）：参与 loss 计算，驱动演化
- **观测类**（A1/A6/A8）：记录但不直接影响演化，用于诊断
- **触发类**（A9）：不参与训练，只做升维判定

**推理**：不是所有公理都应该写成 loss。A1（差异源）和 A8（差异汇）是边界条件，不是优化目标。A6（流向）是涌现结果，不是约束。把它们写成 loss 会和真正的约束类公理冲突。

### 2.8 模型选择：先用最小的

**决策**：第一阶段用 LocalConvModel（3×3 卷积），不急着上 Transformer。

**推理**：复杂模型会让你分不清"到底是公理有效，还是模型容量太大"。CNN 天然满足 A3 局域性。

**升级标准**：只有当"当前模型的容量瓶颈阻碍了公理约束的实现"时才升级。

## 四、尚未实现的关键模块

### 4.1 粗粒化映射 C^(l)（最高优先级）

这是整个递归架构的核心。没有它，L0→L1 的升维就无法完成。

**最小版本方案**：把 L0 划分为固定大小的 block（2×2 或 4×4），每个 block 的状态压缩为一个 L1 token。

**问题**：固定 block 切分可能切断结构的自然边界。更好的方案是基于稳定结构的 mask 来切分，但这需要先解决结构边界检测。

### 4.2 稳定结构检测器（高优先级）

当前 `detect_stable_structures()` 只是最粗糙的版本：检查哪些位置在时间窗口内标准差小。

需要升级为：
- 基于连通区域的结构切分
- 结构边界的精确定义
- 结构内部一致性的度量
- 结构之间交互关系的检测

### 4.3 层间反馈（中优先级）

当前只有自下而上（L0 涌现 → 封装 → L1），没有自上而下（L1 约束 L0 演化）。

差异论说"结构会反过来重塑差异"。这意味着 L1 形成后，应该反过来影响 L0 的演化规则或边界条件。

### 4.4 连续化过渡层（远期）

从 binary 格点到实值格点的过渡。当前 L0 只有 0/1，无法直接做物理模块（散度、旋度等需要连续场）。

## 五、设计中的张力和未解决的争论

### 5.1 规模问题

用户指出：宇宙 137 亿年、10^80 粒子才产生现在的秩序，4×4 格点期待涌现，可能性不大。

**当前态度**：项目的真正价值可能不是"造出能涌现的迷你宇宙"，而是"证明公理约束不会杀死涌现的可能性"。这已经是 meaningful 的负结果。

**但仍然需要放大规模做实验**。可能需要 64×64 或更大的格点。

### 5.2 公理翻译的保真度

差异论的公理用自然语言写成，带有哲学含义。翻译为 `state.sum()` 或 `(delta**2).mean()` 后，保留了多少理论含义？

**态度**：这是一个只能通过实验回答的问题。如果公理约束下的系统确实比无约束的更有结构，那至少说明翻译没有完全丢失理论含义。

### 5.3 A9 的"不可约性"怎么定义

A9 当前用 `ascent_pressure = conservation_residual × structure_density` 来近似。但这不是真正的"不可约性"检测。

真正的不可约性应该是：当前层的自由度组合无法消解守恒残差。这需要某种形式的"穷举测试"或"信息论度量"，目前没有好的实现方案。
