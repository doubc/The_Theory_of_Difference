## 09:14 心跳 — CounterfactualEngine 激活 Bug 修复

### 问题
Phase 3 实验子实验 D（反事实探索）失败：`counterfactual_active` 在所有 ODI 水平下均为 false。

### 根因
`DivergencePointTracker.is_significant` 使用 AND 逻辑（entropy >= 0.5 AND ratio >= 2.0）。
自动生成的随机候选方向产生均匀概率分布（ratio ~1.0），永远无法通过 ratio 阈值。
→ 从未检测到分歧点 → 从未创建反事实分支 → 引擎永远不激活。

### 修复
1. `is_significant` 改为仅使用熵检查（高熵 = 多个可行方向 = 分歧点）
2. HierarchicalEvolver 回调中传递实际 `odi_result` 而非 `None`

## 15:29 心跳 — Phase 2 六阈值"自维持稳健性"修复

### 问题
六阈值检测中，3.2 阈值（自维持稳健性）永远是瓶颈：5/6 达标，从未 6/6。
这导致前主体态收束失败，分级解封无法触发。

### 根因
`SixThresholdDetector` 的 3.2 阈值接收的是 `rebuild_success_count`，但它的值来自方向一致性（`self_sustaining` proxy），而非真实的扰动-重建测试。
→ 3.2 阈值被错误计算 → 永远无法全达标 → 收束失败 → 解封停滞

### 修复
**`pre_subjectivity_convergence.py`**:
- 先运行稳定性测试（扰动 → 重建），将实际存活率反馈给 3.2 阈值
- 建立正确的测量循环：perturb → survival rate → 3.2 threshold

**`hierarchical_evolver.py`**:
- 在 `six_threshold_detector.detect()` 之前运行真实扰动测试
- 使用实际存活计数，而非 `int(self_sustaining * 10)` 代理值
