## 17:28 心跳 — 引力场累积 Bug 修复 + Git 同步

### 发现的问题
`CrossLayerGravityModulator.clear_old_fields()` 从未被调用：
- `compute_gravity_field()` 每次调用都向 `gravity_fields[layer_id]` 追加新场
- 每次演化运行都会累积大量过时引力场
- 后果：(1) 调制结果被 stale 场污染，(2) 内存持续增长

### 修复
在 `_apply_cross_layer_gravity_modulation()` 中，计算新引力场**之前**先调用 `clear_old_fields(max_age_steps=200)`：
- 使用各层 `step_count` 之和作为全局时间戳
- 200 步阈值：足够覆盖一次完整层演化，既清除过期场又保留近期场

### 验证
- 跨层级引力集成测试：**13/13 全部通过** ✅
- 无回归

### Git
- Commit: `4944b21` fix(gravity): clear stale fields before computing to prevent accumulation
- Push: 成功到 `origin/main` (b5c93e1..4944b21)

### 结论
引力调制模块的**内存泄漏**已修复。`clear_old_fields` 之前是"写了但没人用"的防御性代码，现在正式纳入演化循环。
