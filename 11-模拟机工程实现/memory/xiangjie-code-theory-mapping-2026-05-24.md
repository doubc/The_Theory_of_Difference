## 一、核心发现

《象界》第三章末尾提出的命题——"自维持是筛选之前的第一道门槛"——与代码实现高度吻合：

```
SelectionGate 依赖 struct.connectivity_ratio
SelfMaintenanceGate 依赖 struct.material_turnover
PreSubjectiveGate 综合判定 passed_count ≥ 5/7 + connectivity > 0.6
```

**关键对应**：记忆（第IV门）与复制（第V门）在代码中都降级为 lifetime 判断（当无 history 时），这与理论的"必要前提 vs 充分条件"完全一致。

## 三、InterfaceGate 三维权重的理论依据

代码中的权重 `0.35/0.35/0.30` 对应：
- **选择性渗透**（0.35）：边界对不同维度/通道有差异化的通透性
- **方向不对称**（0.35）：外→内梯度 ≠ 内→外梯度，边界区别对待流入流出
- **时间调控**（0.30）：历史序列中边界状态与外部相关性 > 与内部相关性

**降级设计**：无 state → 几何代理；无 history → 二维权（渗透+不对称）；有 history → 三维权。
这与理论完全一致：时间调控是"充分条件"，选择性渗透和方向不对称是"必要条件"。

## 四、MemoryGate 的理论-工程偏差

理论强调："记忆 ≠ 储存，记忆 = 过去对未来施加的偏向性约束"，核心判据是**路径偏置**。

代码实现中：
- **有 history**：自相关(lag-1) + 收敛性（前半/后半方差比）
- **无 history**：降级为纯 lifetime 判断

**潜在改进点**：当降级为 lifetime 时，MemoryGate 实际上只判断了"结构是否持续存在"，而没有判断"过去是否对当前有偏置"。这在无历史数据时是合理的工程妥协，但理论上应标注为"未充分验证"而非"通过"。

## 五、ReplicationGate 的 SSIM-like 设计

最精妙的实现是 ReplicationGate 的结构相似度计算：
```
luminance  = (2*m1*m2 + C1) / (m1² + m2² + C1)   ← 均值接近
contrast   = (2*v1*v2 + C2) / (v1² + v2² + C2)   ← 方差接近
SSIM       = luminance × contrast
```

这直接对应理论："核心不是状态值的精确重复，而是**关键关系特征的结构连续性**"。
均值 ≈ 亮度相似 = 整体水平；方差 ≈ 对比度相似 = 波动模式。

## 六、理论读到的核心洞察

### 6.1 状态密度（状态密度 = 0/1 不是程度问题）

《象界》第一章："状态密度"是中层厚度，不是简单复杂化。
→ 在代码中表现为：门槛是离散的（通过/未通过），但得分是连续的 [0,1]。
这是对的：现实世界中的"象"确实有门槛效应，但门槛内外有程度的渐变。

### 6.2 无意义的功能 vs 无叙事的记忆

《象界》第八章：功能是"无意义的功能"，记忆是"无叙事的记忆"。
→ 代码中 FunctionGate 只用 connectivity_ratio，没有任何语义标注。
→ MemoryGate 只用 autocorrelation，不引用任何叙事性概念。
**这正是差异论的精髓**：低语义 ≠ 无语义，而是尚未进入高语义的语义。

### 6.3 前主体态 = 耦合达到密度的整体状态

```
passed_count ≥ 5/7 AND connectivity > 0.6
```

代码中 PreSubjectiveGate 的判定逻辑非常符合理论：不是逐项达标，而是
"诸机制在同一结构中形成耦合后的整体状态"。5/7 的阈值说明：不需要
全部通过，只需多数机制形成耦合。这是对"汇聚"而非"累积"概念的精确编码。

## 七、下一步：第二阶段锚点

下一阶段目标：**跨层级引力调制 → 解封机制 → 回流通道**

当前代码尚未实现的内容（基于《象界》第四章以后）：
1. 记忆 → 需要结构化的 history（不只是 state 列表，而是 pattern_signature 时序）
2. 复制 → 需要模板存储（当前只比较，没有"召回过去的结构模式"的能力）
3. 解封机制 → 《象界》提到"过去以结构性方式持续留在现在"，当前 xiangjie/chain.py 是检测器，但没有生成器

**核心缺口**：chain.py 是**显现链检测器**（判断结构是否达到某一门槛），
但《象界》第四章以后要求的是**显现链生成器**（让结构跨过门槛）。
这意味着第二阶段需要：在 LocalConvModel 的反应分支中，引入来自 history
的结构性约束——让"过去的关系样式"成为"当前状态更新的偏置"。
