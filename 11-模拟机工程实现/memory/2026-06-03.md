# 2026-06-03 — Phase 5 Track B7: A9 Sealing Fix

## Track B7 Complete ✅

### Problem
exp_120 (B6 fallback) achieved only 3/8 sealing rate. Root cause identified as two bugs:
1. **A9 sealing trigger too strict**: Required ALL N bits active within sliding window simultaneously. Even with sliding window fix, requiring 100% activation (48/48) was impossible — only 36-42 unique bits activated in 5000 steps.
2. **Cross-layer gravity crash**: Gravity modulator iterated over `max_layers` instead of existing layers.

### Fix
1. **Percentage-based threshold**: `sealing_activation_threshold = max(0.75*N, 30)` — sealing triggers at 75% activation, not 100%.
2. **total_unique_active**: All-time unique activation tracker (sealing trigger) + sliding window (freeze decision) separation.
3. **Gravity modulator fix**: Iterate over `hierarchy.n_layers` instead of `max_layers`.

### Results (exp_121, 8 seeds)
| Metric | exp_120 | exp_121 |
|--------|---------|---------|
| Sealing rate | 3/8 (37.5%) | **8/8 (100%)** ✅ |
| L1 formation | 0/8 (0%) | **8/8 (100%)** ✅ |
| Partial freeze | N/A | **8/8** ✅ |
| Sealing step | N/A | **16-73 (avg ~30)** |
| L0→L1 | N/A | **48→18-24** |

### Key Insight
Sealing was never a dynamics problem — the system seals in ~30 steps when the threshold is reachable. The 100% requirement was the blocker. The 75% threshold is the right balance: high enough for meaningful structure, low enough to be reachable.
