## 二、从验证实验数据中发现的新问题

重新分析 `logs/exp_validate_20260506_163319.json`（300步验证实验），发现一个被忽视的模式：

### 2.1 A2 离散编码熵的单调增长

```
A2_discrete_encoding: 0.006 → 0.085（单调增长，无平台期）
```

熵持续上升意味着**系统没有自发形成有序结构**。在九公理框架下，这对应：
- A1（差异源）持续注入 +1
- 但 A3（局域性）完全为 0（所有步数均为 0.0）
- 没有局域约束 → 注入的差异无法在局部凝聚 → 熵单调增长

**结论**：验证实验的失败不是"结构不够稳定"，而是"根本没有形成可验证的结构"。300步中只检测到1个结构，且closure=0、interaction=false——这是一个"伪结构"，只是随机涨落的瞬时聚集。

### 2.2 A3 局域性为零的深层含义

A3_locality 全部为 0，说明 `LocalConvModel` 的卷积核半径可能过小，或者边界条件导致没有有效的邻域交互。

理论对应：《差异论》公理A3要求"变化必须是局部的"。如果A3=0，意味着：
- 每个比特的变化是独立的 → 无法形成"结构"
- 没有邻域耦合 → 没有"边界"的概念
- 整个系统退化为独立比特的随机游走

**这是比"结构不稳定"更根本的问题**：系统连"结构形成的土壤"都没有。

### 2.3 spatial_diversity 的持续增长

```
spatial_diversity: 0.256 → 0.370（增长44%）
```

空间多样性持续增长，配合A2熵增长和A3为零，描绘出一个清晰的图像：
> 系统在扩散，而不是在凝聚。差异在产生，但没有在局部聚集。

## 三、理论-工程映射的新洞察

### 3.1 "凝聚条件"的缺失

《象界》第一章提到"象"的形成需要"密度达到阈值"。在代码中，这对应：
- `connectivity_ratio > 0.6`（PreSubjectiveGate）
- `boundary_closure > 0`（BoundaryGate）

但当前实验配置（grid_size=32, steps=300）下：
- 没有足够的步数让结构凝聚（300步对于32×32网格可能不够）
- 没有A3局域约束来引导凝聚
- 注入速率可能超过凝聚速率

**新假说 H3**：当 `steps / grid_size² < 1` 时（即每比特平均演化步数 < 1），系统无法形成稳定结构。当前实验 `300 / 1024 ≈ 0.29`，远低于阈值。

### 3.2 九公理的执行顺序问题

当前代码中公理的执行顺序是线性的：A1→A6→A3→A4→A7→A5→A9。

但从数据看，**A3应该在A1之后立即生效**：
```
正确顺序：A1(注入) → A3(局域约束) → A4(最小变易) → A6(流向) → ...
```

如果A3在A1之后不立即生效，注入的差异就会在"无约束窗口"中扩散。
