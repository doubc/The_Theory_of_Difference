## 一、工程进展回顾

最新提交 `9dde9e2` 实现了**回流偏置场机制**（BiasField），核心设计：

1. **BiasField 数据结构**：携带 `source_layer`、`target_layer`、`bias_vector`、`strength`、`decay`
2. **传播路径**：`propagate_bias_down()` 从高层（象界）向低层（前主体态）生成偏置向量
3. **调制机制**：`apply_bias_to_layer()` 叠加多个偏置 → 归一化到 [0,1] → 写入 `constraints.bias_profile`
4. **注入阶段**：偏置作为候选比特选择权重 → 概率 ∝ (1 + bias_strength × bias_vector[idx])
5. **吸收阶段**：偏置越小越易被吸收 → 概率 ∝ (1 + bias_strength × (1 - bias_vector[idx]))
6. **衰减保护**：每次应用后 strength × 0.95，保护低层自主性

6项端到端测试全部通过。

## 二、与象界显现链的理论对应

### 2.1 偏置场 = 显现链的"向下牵引力"

《象界》第四章的核心命题：**过去的结构性状态持续留在现在，并对当前演化施加偏向性约束**。

在代码中，这恰好对应：
- **BiasField.source_layer** = 过去的结构性状态（高层，已完成显现的"象"）
- **BiasField.bias_vector** = 该结构在低层比特上的"引力印记"
- **BiasField.strength** = 该结构对当前演化的牵引强度
- **decay()** = 牵引力随时间衰减（不是消失，是逐渐让位于新的结构）

**关键对应**：`propagate_bias_down()` 正是"象界→前主体态"的回流通道。高层已形成的结构（象）向下投射偏置，影响低层的注入/吸收选择——这正是"过去对现在的偏向性约束"的工程实现。

### 2.2 偏置调制 vs 门槛检测

`xiangjie/chain.py` 是**显现链检测器**（判断结构是否跨过门槛），而 BiasField 是**显现链生成器**（让结构更容易跨过门槛）。

二者的关系：
- **chain.py**：自下而上检测 — "当前结构达到了哪个阶段？"
- **BiasField**：自上而下调制 — "过去的结构如何帮助当前结构跨越门槛？"

例如：
- 如果高层检测到一个结构通过了 `SelfMaintenanceGate`（自维持），该结构可以作为 BiasField 向下传播
- 低层比特在注入/吸收时会受到该偏置的影响，更容易形成类似的结构模式
- 这形成了一个**正反馈循环**：过去的成功结构 → 偏置向下 → 低层更容易形成类似结构 → 新的自维持结构

### 2.3 衰减机制的理论意义

`decay()` 使用 0.95 的衰减因子，每次应用后 strength 下降 5%。这对应理论中的：

> "过去的影响不是决定性的，而是概率性的"

如果偏置不衰减：
- 低层演化会被高层结构完全锁定 → 失去自主性 → 无法产生新的差异
- 这违反了差异论的核心公理：**差异必须持续产生，不能被既有结构完全决定**

衰减机制确保了：
1. 过去的影响存在，但不绝对
2. 新的差异仍然有机会产生
3. 当新的结构形成后，它可以产生新的偏置，覆盖旧的偏置

## 三、第二阶段的关键缺口

基于本次合成分析，第二阶段（象界→前主体态回流通道）需要补充：

### 3.1 偏置场的"向上反馈"

当前只有 `propagate_bias_down()`（高层→低层），缺少 `propagate_bias_up()`（低层→高层）。

理论依据：《象界》第八章提到"前主体态是诸机制汇聚的整体状态"，这意味着低层的成功演化也应该向上影响高层的偏置场。

```python
    # 从低层提取活跃比特的模式
    # 在高层生成对应的偏置向量
    # 用于调制高层的注入/吸收
```

### 3.2 偏置场的"横向耦合"

当前偏置场只在层级之间传播，缺少同层不同结构之间的横向耦合。

理论依据：《象界》提到"多个象可以同时存在，相互影响"。

**工程建议**：
- 在同一层内，多个 BiasField 之间应该有相互作用
- 正偏置相互增强，负偏置相互抵消
- 这对应理论中的"象的共存与竞争"

### 3.3 偏置场的"记忆存储"

当前 BiasField 是瞬时的（随 decay 消失），缺少长期记忆存储。

理论依据：《象界》第四章"过去以结构性方式持续留在现在"——这意味着有些偏置应该能够长期存在。

**工程建议**：
- 引入 `PersistentBiasField`，衰减因子更小（如 0.99）
- 或者引入 `BiasMemory`，存储历史偏置场，在特定条件下重新激活
